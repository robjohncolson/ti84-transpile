# Auto-Session 138 P1: FP Category Writer Seeds + E_Domain Investigation

## Context
This is a TI-84 Plus CE ROM transpilation project. The transpiler at `scripts/transpile-ti84-rom.mjs` lifts eZ80 ROM machine code into JavaScript. It uses a seed list to discover code blocks. We need to add new seeds to cover FP (floating-point) category writer blocks that are currently missing.

## Background
- The FP dispatch system uses a "category byte" at RAM address 0xD0060E to route operations like gcd, min, max to the correct handler.
- Session 137 proved: during gcd(12,8) evaluation, the category byte is NEVER written — the FP eval engine blocks and category writers are all UNCOVERED (not in transpiled blocks).
- Manual seed injection (writing 0x28 to 0xD0060E at dispatch time) correctly routes to gcd handler at 0x06859B→0x0689DE, but then gcd hits E_Domain error (0x84) at PC=0x068D5D.

### Part B: Build probe for gcd(12,8) with new seeds

1. Loads the freshly-transpiled ROM (use same pattern as `probe-phase137-fp-category-trace.mjs` — read that file first for the exact boilerplate)
2. Sets up MEM_INIT, CreateReal for Ans, ParseInp for "gcd(12,8)" with correct tokens:
   - gcd( = [0xBB, 0x07] (two-byte token)
   - 1 = 0x31, 2 = 0x32
   - , = 0x2B
   - 8 = 0x38
   - ) = 0x11 (CORRECT close-paren, NOT 0x29)
   - end = 0x3F
3. Add write-watchpoints on 0xD0060E (category byte) to check if the newly-seeded blocks NOW write the category naturally
4. Track whether FP eval engine blocks (0x07E111, 0x07E14D, 0x07E16D, 0x07E199) get any hits
5. Track whether category writers (0x095722, 0x095765, 0x0957FF) get any hits
6. Report the final errNo, OP1 value, and category byte state at dispatch (0x0686EF)
