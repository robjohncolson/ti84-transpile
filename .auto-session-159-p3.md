# Session 159 P3 — Investigate FPDiv(1200/1200) context divergence

## Context

You are working on a TI-84 Plus CE ROM transpilation project at `C:/Users/rober/Downloads/Projects/school/ti84-transpile`. The FP gcd investigation has a critical puzzle:

- **Session 156 P2b**: FPDiv(1200,1200) called DIRECTLY at 0x07CAB9 returns 1.0 in 185 steps. CORRECT.
- **Session 158 P2**: FPDiv(1200,1200) called via BYPASS (intercept 0x07CA48→jump to 0x07CAB9) returns 1199. WRONG.

The bypass skipped the normalization function at 0x07CA48-0x07CAB8, which falls through into FPDiv at 0x07CAB9. Something in that normalization function sets up state that FPDiv needs — likely the D register, cleared bytes, or other register/RAM state.

## Task

Build a probe file `TI-84_Plus_CE/probe-phase159-fpdiv-context.mjs` that:

### Part A: Capture full CPU+memory state at FPDiv entry in DIRECT call context
- Seed OP1 = BCD 1200 (type=0x00, exp=0x83, mantissa=[12 00 00 00 00 00 00])
- Seed OP2 = BCD 1200 (same)
- Call FPDiv DIRECTLY at 0x07CAB9
- Use onBlock to capture state at the FIRST step (entry): A, F (all bits), B, C, D, E, H, L, IX, IY, SP
- Also dump: OP1 (9 bytes), OP2 (9 bytes), FPS ptr, memory at 0xD00604 (1 byte), memory at 0xD02AC8 (1 byte)
- Let it run to completion, confirm result = 1.0

### Part B: Capture full CPU+memory state at FPDiv entry in BYPASS context
- Seed OP1 = BCD 1200, OP2 = BCD 1200
- Enter at 0x07CA48 but intercept: when PC reaches 0x07CA48, redirect to 0x07CAB9 (bypass normalization)
- Capture the SAME state at FPDiv entry as Part A
- Let it run, confirm result = 1199 (or whatever wrong answer)

### Part C: Diff the two states
- Compare every register and memory value between Part A and Part B at FPDiv entry
- Print a table: register/address, value in Part A (direct), value in Part B (bypass), MATCH/DIFFER
- **This is the key deliverable**: which specific state differences cause FPDiv to produce 1199 vs 1.0?

### Part D: Isolate the causal state
- For each register/value that DIFFERS between A and B:
  - Set up the BYPASS context (Part B initial state)
  - Manually fix JUST that one register/value to match Part A
  - Run FPDiv
  - Does it now produce 1.0?
- This narrows the root cause to the specific register or memory location
