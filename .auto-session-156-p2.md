# Task: Dense FPDiv comparison — FPDiv(12,8) vs FPDiv(1200,1200)

## Context
This is a TI-84 Plus CE ROM transpilation project. The ROM is lifted to JavaScript and executed in Node.js.

FPDiv(12,8)=1.5 works correctly (202 steps, session 154). FPDiv(1200,1200) produces 0 instead of 1 (exponent underflows 0x80→0x7F, 170+ steps). We need a step-by-step comparison.

FPDiv is at JT slot 0x0201F4 → impl 0x07CAB9. The TI-84 uses 9-byte BCD floating point: [type, exponent, m1m2, m3m4, m5m6, m7m8, m9m10, m11m12, m13m14]. Exponent is biased by 0x80 (0x80=10^0, 0x81=10^1, 0x82=10^2, etc.).

OP1 is at 0xD005F8 (9 bytes). OP2 is at 0xD00603 (9 bytes).

## BCD encoding
- 12.0 = [00, 81, 12, 00, 00, 00, 00, 00, 00] (exponent 0x81 = 10^1, mantissa 1.2 × 10^1 = 12)
- 8.0 = [00, 80, 80, 00, 00, 00, 00, 00, 00] (exponent 0x80 = 10^0, mantissa 8.0 × 10^0 = 8)
- 1200.0 = [00, 83, 12, 00, 00, 00, 00, 00, 00] (exponent 0x83 = 10^3, mantissa 1.2 × 10^3 = 1200)
- 1.5 = [00, 80, 15, 00, 00, 00, 00, 00, 00] (exponent 0x80, mantissa 1.5)
- 1.0 = [00, 80, 10, 00, 00, 00, 00, 00, 00]

## Deliverables
Build a probe `TI-84_Plus_CE/probe-phase156-fpdiv-compare.mjs` that:

1. Loads the transpiled ROM and runtime
2. Runs FPDiv(12, 8) via the JT slot at 0x0201F4 — log OP1 and OP2 (first 9 bytes each) at every block execution
3. Runs FPDiv(1200, 1200) via the same JT slot — same logging
4. Compares the two traces side-by-side: at which step do they diverge? What are the OP1/OP2 values at divergence?
5. Specifically track the exponent byte (OP1[1]) through the division loop

## Runtime setup pattern
```javascript
import { readFileSync } from 'fs';
import { createExecutor } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';

const romBuf = readFileSync('TI-84_Plus_CE/ROM.rom');
const mem = new Uint8Array(16 * 1024 * 1024);
mem.set(new Uint8Array(romBuf.buffer, romBuf.byteOffset, romBuf.byteLength), 0);

const peripherals = createPeripheralBus({ timerInterrupt: false });
const { BLOCKS } = await import('./ROM.transpiled.js');
const ex = createExecutor(mem, BLOCKS, peripherals);

// Seed OP1 with dividend, OP2 with divisor
const OP1 = 0xD005F8;
const OP2 = 0xD00603;

// Example: seed 12.0 into OP1
mem.set([0x00, 0x81, 0x12, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00], OP1);
// Seed 8.0 into OP2
mem.set([0x00, 0x80, 0x80, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00], OP2);

// Set CPU state
const cpu = ex.getCPU();
cpu.pc = 0x0201F4; // JT slot for FPDiv
cpu.spx = 0xD1A86C;
cpu.adl = 1;
// Push FAKE_RET sentinel
const FAKE_RET = 0x7EEDF3;
mem[cpu.spx] = FAKE_RET & 0xFF;
mem[cpu.spx + 1] = (FAKE_RET >> 8) & 0xFF;
mem[cpu.spx + 2] = (FAKE_RET >> 16) & 0xFF;

// Use onBlock callback for logging
let steps = 0;
const trace = [];
ex.onBlock = (pc) => {
    steps++;
    const op1 = Array.from(mem.slice(OP1, OP1 + 9));
    const op2 = Array.from(mem.slice(OP2, OP2 + 9));
    trace.push({ step: steps, pc, op1, op2 });
    if (steps > 500 || pc === FAKE_RET) return true; // stop
};
ex.run();
```

Print the comparison to stdout. Focus on the exponent byte trajectory and the step where they diverge.
