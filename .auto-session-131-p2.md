# Auto-Session 131 P2 — Graph pipeline: set up graph window + test DrawCmd/GraphPars

## Context

This is a TI-84 Plus CE ROM transpilation project. The ROM (4MB eZ80 machine code) has been lifted into JavaScript. We run probes that execute transpiled ROM blocks.

Session 130 validated ILine (0x07B245) — it renders Bresenham-style lines to plotSScreen (21 non-zero bytes = 41 pixels). Session 129 validated IPoint (0x07B451) — single pixel writes. Both work with drawMode=1, seeded pixel dimensions, and coordinates < 60.

Key RAM addresses for graph rendering (from sessions 127-130):
- plotSScreen: 0xD09466 (21945 bytes, graph back-buffer, 4bpp packed format)
- pixWideP: 0xD014FE (16-bit pixel width)
- pixWide_m_2: 0xD01501 (16-bit pixel height)  
- penColor: 0xD026AE (2 bytes, 0x001F = blue works, 0x0000 = invisible)
- drawMode byte: 0xD02AC8 (A register on entry)
- modeByte: 0xD02AD4 (0=point, 1=line)
- hookflags3: 0xD000B5 bit 7 must be 0

Graph BCD real addresses (from ti84pceg.inc / phase25be-graph-investigation-report.md):
- Xmin: 0xD01E33, Xmax: 0xD01E3C, Xscl: 0xD01E45
- Ymin: 0xD01E4E, Ymax: 0xD01E57, Yscl: 0xD01E60
- graphMode: 0xD01474 (0=function)

Key JT entries:
- IPoint: 0x07B451, ILine: 0x07B245
- GraphPars: 0x09986C (reads graphMode, dispatches)
- DrawCmd: 0x05DD96

- Run with 500K step budget, track which blocks are hit
- Report: does it return? What PCs are visited? Any missing blocks?

### Test 4: DrawCmd probe
