# Auto-Session 138 P2: GraphPars X Input Slot + Y Output Location

## Context
This is a TI-84 Plus CE ROM transpilation project. The transpiler lifts eZ80 ROM into JavaScript. GraphPars (0x09986C) is the OS routine that evaluates Y-equations for graphing. Session 137 proved GraphPars runs cleanly for 27 X values but always returns Y=0.0 because tX handler at 0x07D1B4 reads X from a dedicated graph RAM slot, NOT the X variable in VAT.

## Background
- GraphPars body entry at 0x099874 works (session 136): evaluates Y1=X equation in 341 steps, errNo=0x00
- OP1 after GraphPars contains equation name [0x03, 0x10, ...] with bit 6 set (evaluated flag), NOT the computed Y
- Y is not in OP1, OP2, OP3-OP6, or FP stack (delta=0)
- tX handler at 0x07D1B4 reads X from a dedicated graph variable RAM slot
- Graph cursor variables: curGX2 at 0xD01471, curGY2 at 0xD01477
- Graph window: Xmin at 0xD01547, Xmax at 0xD01550, Ymin at 0xD01583, Ymax at 0xD0158C
- errSP must be seeded at 0xD008E0
- FPS must be separated from equation data (FPS at USERMEM+0x100 = 0xD1A981)

## Task

### Part A: Disassemble tX handler at 0x07D1B4
Create `TI-84_Plus_CE/probe-phase138-graphpars-x-source.mjs` that:

1. Read the file `TI-84_Plus_CE/probe-phase137-graphpars-loop.mjs` first for boilerplate patterns
2. Load the ROM binary directly from `TI-84_Plus_CE/ROM.rom` (4MB file)
3. Disassemble 64 bytes starting at ROM offset 0x07D1B4 using raw byte inspection:
   - Look for `LD HL,(nn)` or `LD A,(nn)` patterns that read from RAM addresses in 0xD0xxxx range
   - Common patterns: `ED 6B xx yy zz` = LD HL,(nnnn), `3A xx yy zz` = LD A,(nnnn)
   - In eZ80 ADL mode, addresses are 3 bytes (little-endian)
4. Also check these candidate X source addresses by reading ROM bytes around 0x07D1B4:
   - curGX2 = 0xD01471 (graph cursor X, BCD float, look for bytes 71 14 D0)
   - curGr = 0xD0147D (look for 7D 14 D0)
   - Xmin = 0xD01547 (look for 47 15 D0)
   - traceX = maybe near curGX2

### Part B: Find Y output location
After identifying the X source, build a test that:
1. Sets up a Y1=X equation via CreateEqu (0x082438) — use pattern from `probe-phase136-graphpars-body.mjs`
2. Seeds the identified X source address with a known BCD value (e.g., 5.0 = [0x00, 0x80, 0x50, 0x00, ...])
3. Calls GraphPars body at 0x099874 with proper setup (errSP seeded, FPS separated)
4. After GraphPars returns, scan these locations for the computed Y value:
   - curGY2 at 0xD01477 (9 bytes)
   - curGR at 0xD0147D (9 bytes)  
   - The 9 bytes at whatever address the tX handler reads from + 9 offset
   - plotSScreen at 0xD09466 (check for pixel data)
   - OP1 through OP6 (0xD005F8, each 11 bytes apart: +0, +11, +22, +33, +44, +55)
   - FP stack (check FPS pointer movement)
5. Also dump 64 bytes around the graph cursor area (0xD01470-0xD014B0) before and after GraphPars

### Part C: Full pipeline test
If X source and Y destination are found:
1. Create a loop: for X in [0, 1, 2, 5, 10], seed X at the identified slot, call GraphPars, read Y from the identified destination
2. For Y1=X, expected: Y should equal X for each value
3. Report exact BCD bytes at Y destination for each X value
