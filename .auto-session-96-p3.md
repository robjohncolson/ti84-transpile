# Auto-Session 96 P3: 0x0973C8 dual-ParseInp pattern — full function disassembly

## Context

Session 95 P3 found that 0x0973C8 is one of only 3 direct callers of ParseInp (0x099914). It has an unusual pattern:
- Writes OP1[0]=0x0E before the first ParseInp call
- Calls ParseInp TWICE (at 0x0973F8 and 0x09740A)
- Has PushErrorHandler(0x061DEF) between the two ParseInp calls
- 0x0973BA is a shared helper called by both 0x0973C8 and 0x0ACC4C (the other ParseInp caller)
- 0x0973BA calls 0x05E872 (buffer flush: BIT 2,(IY+1) check)

Type 0x0E in OP1[0] is unknown — could be a temporary/expression type, list element, or matrix element.

## Task

Create a probe `TI-84_Plus_CE/probe-phase25ak-dual-parseinp-disasm.mjs` that:

1. Reads ROM bytes and disassembles THREE function bodies:
   a. 0x0973C8 — the dual-ParseInp caller (disassemble until RET, at least 128 bytes)
   b. 0x0973BA — the shared helper (from 0x0973BA to 0x0973C7, then continue if no RET found)
   c. 0x05E872 — the buffer flush helper (at least 64 bytes)

2. For each instruction, output: `address (hex), raw bytes (hex), mnemonic, operands`

3. For 0x0973C8, specifically identify:
   - The exact instructions that write OP1[0]=0x0E (what register, what immediate value)
   - Both CALL 0x099914 sites (confirm addresses 0x0973F8 and 0x09740A)
   - The PushErrorHandler call between them (confirm 0x061DEF target)
   - What happens AFTER the second ParseInp call — does it check OP1? Does it call more functions?
   - Any PopErrorHandler or cleanup code
   - What the function returns (OP1 result? error code? flag state?)

4. For 0x0973BA, identify:
   - What setup it does before the caller proceeds to ParseInp
   - Does it set begPC/curPC/endPC? Does it initialize the token buffer?
   - What does the BIT 2,(IY+1) check in 0x05E872 do — is this the "dirty buffer" flag from session 95?

5. Cross-reference any OP1 type byte constants found. TI-OS type bytes:
   - 0x00 = Real, 0x01 = List, 0x02 = Matrix, 0x04 = String, 0x05 = Program

   - Check ti84pceg.inc (at `TI-84_Plus_CE/references/ti84pceg.inc`) for type 0x0E definition
