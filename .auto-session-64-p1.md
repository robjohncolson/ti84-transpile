# Phase 198 — Fix 42 suspicious mem.fill calls

## Context

The `mem.fill()` API in this project uses the signature `mem.fill(value, start, END)` where the third argument is an END ADDRESS, not a byte count. Phase 197 audited all 326 mem.fill calls and found 42 that pass a bare integer byte count as the third arg instead of `start + count`. Because start (cpu.sp) is a high RAM address like 0xD1A872 and end is 12 or 6, these calls silently no-op (start > end).

## Task

Fix all 42 suspicious mem.fill calls listed below. The fix is mechanical:
- `mem.fill(0xFF, cpu.sp, 12)` → `mem.fill(0xFF, cpu.sp, cpu.sp + 12)`
- `mem.fill(0xFF, cpu.sp, 6)` → `mem.fill(0xFF, cpu.sp, cpu.sp + 6)`
- `mem.fill(0xFF, cpu.sp, size)` → `mem.fill(0xFF, cpu.sp, cpu.sp + size)`
