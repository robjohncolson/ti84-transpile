## Context

This is a TI-84 Plus CE ROM transpilation project. The OS ROM (eZ80 machine code) has been lifted to JavaScript. We run "probes" — Node.js scripts that boot the OS, seed state, and trace execution.

Session 96 discovered that Chain1 (0x06CE73) gates on `ld a, (0xD0146D); or a; ret z` at 0x06CE7F-0x06CE84. When 0xD0146D==0 (default), Chain1 returns immediately and never reaches Chain2 (0x06C8B4), which is the path to ParseInp (the expression parser). This is THE critical experiment: seed 0xD0146D with a key code, run CoorMon, and see if Chain2→ParseInp fires.
