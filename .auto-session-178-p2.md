# Session 178 P2 — Trace 0x03CF7D (_GetCSC implementation) with full OS state

## Context

In the TI-84 Plus CE OS ROM transpilation project, the _GetCSC system call is accessed via jump table entry 0x02010C → JP 0x03CF7D. Previous sessions traced the handler at 0x03D184 and found it's interrupt acknowledgment code, NOT a keyboard scanner. The real keyboard scan routine is at 0x0159C0.

Session 177 found that the ISR event loop (with MBASE=0xD0) follows path: 0x000038→...→0x0019BE→...→0x001A5D→...→0x001A32→0x020108→0x0401DF→HALT. It detects keyboard IRQ (port 0x5016) and acknowledges it (port 0x500A) but never calls 0x0159C0.

**New hypothesis**: The code at 0x03CF7D might be the actual _GetCSC implementation that reads from a scan code buffer (populated by the ISR calling 0x0159C0), rather than scanning the keyboard itself. We need to trace 0x03CF7D to understand what it actually does.

## Task

Create a probe file `TI-84_Plus_CE/probe-phase178-getCSC-impl-trace.mjs` that:

1. **Statically disassembles** the code at 0x03CF7D for 64 instructions (or until HALT/RET), printing each instruction with address and hex bytes.

2. **Traces execution** starting at PC=0x03CF7D with:
   - MBASE = 0xD0
   - SP = 0xD1A87E (standard stack top)
   - IY = 0xD00080 (standard OS IY)
   - Memory initialized from ROM (0x000000-0x3FFFFF)
   - ENTER key pressed in keyboard matrix: `keyMatrix[6] = ~(1 << 0)` (group 1, bit 0 = ENTER = scan code 0x09)
   - RAM region 0xD00000-0xD1FFFF zeroed initially
   - Timer interrupt DISABLED (`createPeripheralBus({ timerInterrupt: false })`)
   - Max 200 steps (with cycle limit)

3. **Logs every memory read/write** in the 0xD00000-0xD1FFFF RAM range and every port I/O access, especially:
   - Reads from 0xE00800-0xE00FFF (keyboard MMIO)
   - Reads/writes near 0xD02AD7 (callback pointer)
   - Any reads from what might be a scan code buffer

4. **Reports the execution path** (block addresses visited) and final register state.

## Key Constants

- _GetCSC impl entry: 0x03CF7D
- Keyboard scan routine: 0x0159C0
- Standard stack: SP = 0xD1A87E
- Standard IY: 0xD00080
- MBASE: 0xD0
- Keyboard MMIO: 0xE00800-0xE00FFF
- Callback ptr: 0xD02AD7
- Timer interrupt: DISABLED
- ENTER key matrix setup: `peripherals.keyMatrix[6] = ~(1 << 0) & 0xFF` (SDK Group 1)
