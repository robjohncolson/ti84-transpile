# Task: Runtime trace 0x058241 (home-screen handler) to find ParseInp call chain

## Context
In the TI-84 Plus CE ROM transpilation project, CoorMon (0x08C331) dispatches keystrokes to the active app handler via `JP (HL)` where HL=cxMain. For the home screen, cxMain=0x058241. Session 92's static disasm showed 0x058241 has multiple RET NZ guards and calls subroutines, but ParseInp (0x099914) is NOT called directly in the first 256 bytes — it's deeper in the call chain.

## What to do

Create a probe `TI-84_Plus_CE/probe-phase25ah-homehandler-trace.mjs` that:

1. **Sets up the runtime** (import from cpu-runtime.js, peripherals.js, load ROM.rom, create transpiled module)
2. **Seeds OS state** exactly as session 92's CoorMon probe did:
   - Run MEM_INIT at 0x09DEE0 first (18 steps, sets up heap pointers)
   - Set IY = 0xD00080
   - Set cxMain: write24(0xD007CA, 0x058241)
   - Set cxCurApp: write8(0xD007E0, 0x40)
   - Seed keyboard with ENTER key: keyMatrix[1] = 0xFE, write8(0xD0052C, 0x05), write8(0xD0052D, 0x05)
   - Set kbdSCR (last scan code): write8(0xD00587, 0x09) — ENTER scan code
3. **Call the home-screen handler DIRECTLY** — don't go through CoorMon. Push a FAKE_RET sentinel (0xBEEF00) on the stack, then `cpu.PC = 0x058241` and run up to 50,000 steps
4. **Record the first 500 UNIQUE PCs visited** (use a Set for uniqueness, but record in order of first appearance)
5. **Check if any of these PCs match known entry points**:
   - 0x099914 = ParseInp
   - 0x099B81 = ParseInp internal
   - 0x08238A = CreateReal
   - 0x09AC77 = RclVarSym
   - 0x08383D = ChkFindSym
   - 0x07C77F = FPAdd
   - 0x061DB2 = JError
   - 0x061DD1 = PopErrorHandler
   - 0x03FA09 = GetCSC
   - 0x0A1B5B = PutC
   - 0x0A1799 = PutMap
6. **Track CALL instructions** — whenever a CALL is executed (PC trace shows a jump forward/backward that isn't sequential), log the source PC and target PC. This reveals the call chain.
7. **Print results**:
   - Total steps executed
   - How the run ended (FAKE_RET hit? HALT? missing_block? max_steps?)
   - First 500 unique PCs, annotated with known names
   - All CALL source→target pairs found
   - Whether ParseInp was reached, and if so, at what step number

Also create a report `TI-84_Plus_CE/phase25ah-homehandler-trace-report.md`.

## Standard probe boilerplate
```javascript
import { readFileSync } from 'fs';
import { createCPU, createMemory, loadROM } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';

const rom = readFileSync('TI-84_Plus_CE/ROM.rom');
const memory = createMemory();
loadROM(memory, rom);

// Load transpiled blocks
const transpiledModule = await import('./ROM.transpiled.js');
const blocks = transpiledModule.default || transpiledModule.blocks || transpiledModule;

const peripherals = createPeripheralBus({ timerInterrupt: false });
const cpu = createCPU(memory, blocks, peripherals);

// IY must be 0xD00080 for OS flag access
cpu.IY = 0xD00080;
```
