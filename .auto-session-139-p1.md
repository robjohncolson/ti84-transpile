# Auto-Session 139 P1: GraphPars Full Rendering Loop (Y1=X)

## Goal
Build a probe that renders Y1=X as actual pixels using the validated GraphPars pipeline from session 138. This is the first full graph rendering test.

## Background
Session 138 proved:
- tX handler at 0x07D1B4 fails for EquObj type — it doesn't push X to FPS
- **FIX**: intercept tX handler PC, push X as 9-byte BCD real to FPS (pointer at 0xD0258D), advance FPS by 9
- Y output = OP1 (0xD005F8) with type byte bit 6 set (0x40 = evaluated flag)
- Verified for 8 positive X values (0,1,2,3,5,10,0.5,3.14159)
- X=-3 causes infinite loop (likely graph window range check)

Session 136 proved:
- GraphPars body at 0x099874 works when called with correct OP1=[0x03, 0x10, 0x00...] (Y1 name)
- OS CreateEqu (0x082438) required to create proper VAT entry for the equation
- errSP at 0xD008E0 must be seeded
- FPS must be separated from equation data (FPS at USERMEM+0x100)

Session 128/131 proved:
- IPoint at 0x07B451 writes pixels when drawMode=1, needs pixWideP (0xD014FE) and pixWide_m_2 (0xD01501) seeded
- Pen color at 0xD026AE

## What to build
Create `TI-84_Plus_CE/probe-phase139-graphpars-render.mjs` that:

1. Cold boots the ROM (same pattern as existing probes — read `probe-phase138-graphpars-x-source.mjs` for the exact pattern)
2. Runs MEM_INIT (0x09DEE0) 
3. Creates Y1 equation via CreateEqu with tokens for Y1=X (token sequence: [0x58, 0x3F] = tX + end-of-expression)
4. Seeds graph window variables:
   - Xmin = -10.0, Xmax = 10.0, Ymin = -10.0, Ymax = 10.0 (standard TI-84 window)
   - These are at RAM slots — check ti84pceg.inc reference at `TI-84_Plus_CE/references/ti84pceg.inc` for addresses (search for XMin, XMax, YMin, YMax or Xmin etc.)
5. Seeds pixel dimensions: pixWideP (0xD014FE) = 320 (0x0140), pixWide_m_2 (0xD01501) = 240 (0x00F0)
6. Seeds errSP at 0xD008E0 with a valid error frame
7. For X values from -10 to 10 in steps of 0.0625 (320 columns mapped to [-10,10]):
   a. Set up OP1 = [0x03, 0x10, 0x00...] (Y1 equation name)
   b. Install a step callback that intercepts PC = 0x07D1B4 (tX handler): push X as 9-byte BCD to FPS at address mem.read24(0xD0258D), advance FPS pointer by 9
   c. Call GraphPars body at 0x099874
   d. Read Y from OP1 (strip bit 6 from type byte), decode BCD
   e. Convert (X, Y) to pixel coordinates: px = (X - Xmin) / (Xmax - Xmin) * 320, py = 240 - (Y - Ymin) / (Ymax - Ymin) * 240
   f. If pixel is in bounds (0-319, 0-239), call IPoint at 0x07B451 with drawMode=1
8. After the loop, count non-zero bytes in plotSScreen (0xD52C00, size 320*240*2 = 153,600 bytes... actually check — plotSScreen format may be different)
9. Report: how many GraphPars calls succeeded, how many pixels were rendered, any errors
