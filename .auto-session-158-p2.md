# Session 158 P2 — GCD Bypass Test: Skip 0x07CA48 Normalization, Enter FPDiv at 0x07CAB9 Directly

## Context
We're debugging gcd(12,8) in our TI-84 CE ROM transpiler. The root cause is traced to the normalization loop at 0x07CA48 which destroys the mantissa via Shl14 (0x07FB33). The gcd handler (0x068D3D) calls OP1toOP2 (0x07C747), which calls 0x07F8FA (OP1→OP2 copy) then 0x07CA48 (normalization). The normalization zeroes the mantissa.

Session 157 confirmed FPDiv at 0x07CAB9 works correctly: FPDiv(12,8)=1.5 in 202 steps.

The normalization at 0x07CA48 FALLS THROUGH into FPDiv at 0x07CAB9. So if we bypass the normalization pre-loop but still reach FPDiv, the division should work.

## Your Task
Build a probe `TI-84_Plus_CE/probe-phase158-gcd-bypass.mjs` that tests gcd(12,8) with the normalization loop intercepted.

### Approach: onBlock intercept
1. Set up the standard gcd(12,8) test:
   - Full cold boot + MEM_INIT pipeline (same as probe-phase157-gcd-07ca48-trace.mjs)
   - OP1=12.0 [00 81 12 00...], OP2=8.0 [00 81 80 00...]
   - Push 8.0 to FPS at 0xD1A881+ (FPS pointer at 0xD0258D)
   - errSP seeded, error frame seeded
   - Entry at 0x068D3D (gcd handler)

2. Use an `onBlock` callback to intercept PC=0x07CA48:
   - When PC hits 0x07CA48, **skip the normalization entirely** by changing PC to 0x07CAB9 (FPDiv entry)
   - Log: "BYPASS: skipping 0x07CA48 normalization, jumping to FPDiv at 0x07CAB9"
   - Track how many times the bypass fires

3. Run up to 5000 steps with step-by-step tracing of OP1/OP2 at key blocks:
   - 0x068D3D (gcd entry)
   - 0x068D61 (validator wrapper)
   - 0x07C747 (OP1toOP2)
   - 0x07F8FA (OP1→OP2 copy)
   - 0x07CA48 (intercepted → bypass)
   - 0x07CAB9 (FPDiv entry)
   - 0x068D82-0x068EB4 (gcd algorithm body)
   - 0x07F831 (type validator)
   - Any block that writes errNo at 0xD008DF

4. At completion, report:
   - Final OP1 value (should be 4.0 = [00 81 40 00...] if bypass works)
   - Final errNo
   - Number of bypass events
   - Total steps
   - Whether E_Domain was avoided
