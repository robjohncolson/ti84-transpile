# Session 128 P1: .SIS prefix block coverage — investigate and fix graph rendering blocker

## Context
The TI-84 CE ROM transpilation project lifts eZ80 machine code to JavaScript. The transpiler (`scripts/transpile-ti84-rom.mjs`) and decoder (`TI-84_Plus_CE/ez80-decoder.js`) already support eZ80 .SIS/.LIS/.SIL/.LIL size prefix opcodes (0x40/0x49/0x52/0x5B). The decoder recognizes them at lines 797-800, and the transpiler correctly generates MBASE-composited 16-bit addresses for .SIS-prefixed instructions.

**The problem**: Session 127's graph probe (`TI-84_Plus_CE/probe-phase127-graph-render.mjs`) found that the IPoint bounds-check at ROM address 0x07B793 uses a .SIS prefix (opcode 0x40) before `LD HL,(nn)`. At runtime, the transpiler treats 0x40 as `LD B,B` instead of .SIS prefix → reads a wrong 3-byte address instead of 2-byte + MBASE → bounds check fails → graph rendering blocked.

**Hypothesis**: The block at 0x07B793 may not have been discovered by the block walker (missing seed), OR the block was discovered starting at a different alignment where 0x40 falls inside another instruction, OR the transpiled block exists but the runtime somehow doesn't use the prefix-aware code path.
