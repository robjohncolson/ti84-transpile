# Session 140 P3 — OneVar Stat E_Memory Fix: Pre-Create Stat Result Variables

## Task
Fix OneVar's E_Memory (0x89) error by pre-creating stat result variables before calling OneVar, so it doesn't need to allocate them internally.

## Context
Session 139 P3 proved:
- Manual VAT entry construction works for L1 (list variable)
- OneVar (0x0A9325) runs 5000 steps doing real FP arithmetic on list elements when L1 exists
- Extended runs (50K steps) hit errNo=0x89 (E_Memory) when OneVar tries to create stat result variables internally
- 0/8 stat result variable slots were written

The stat result variables are stored at known addresses (from ti84pceg.inc/session 72):
- Stat var base likely around 0xD012A8 area
- Session 72 confirmed: Rcl_StatVar/Sto_StatVar with tMean=0x21 uses slot at 0xD012A8
- Each stat var is 9 bytes (BCD real)
- OneVar computes: n, meanX (mean), sumX (Σx), sumX2 (Σx²), Sx (sample std dev), sigmaX (population std dev), minX, maxX

For list data [10, 20, 30, 40, 50]:
- n = 5, mean = 30, Σx = 150, Σx² = 5500
- Sx = √(2500/4) = 25, σx = √(2500/5) = √500 ≈ 22.36
- minX = 10, maxX = 50

## Deliverables

Build `probe-phase140-onevar-prealloc.mjs` that:

### Part 1: Find Stat Variable RAM Layout
1. Read `references/ti84pceg.inc` and search for stat variable addresses:
   - Look for `statVars`, `tMean`, `tN`, `tSumX`, `tSumX2`, `tSx`, `tSigmaX`, `tMinX`, `tMaxX`
   - Map token values to stat var slot indices
   - Find the base address and stride (likely 9 bytes per slot)
2. Print the complete stat var address map

### Part 2: Pre-Create Stat Result Variables via Manual VAT Entries
Using the manual VAT entry technique from session 139:
1. Create L1 with data [10, 20, 30, 40, 50] (5 BCD reals + 2-byte count prefix)
2. For each stat result variable (n, mean, Σx, Σx², Sx, σx, minX, maxX):
   - Create a Real variable in VAT using manual entry (type=0x00, name=stat token)
   - Allocate 9 bytes of data space initialized to 0
   - OR: Pre-seed the stat variable slots at their known RAM addresses with zero values

### Part 3: Run OneVar with Pre-Created Variables
1. Call OneVar (0x0A9325) with:
   - L1 populated with [10, 20, 30, 40, 50]
   - All stat result variables pre-created
   - errSP frame seeded (use the ParseInp error-frame pattern)
   - Budget: 100K steps
2. Check errNo after completion
3. Read all stat variable slots and report values
4. If errNo=0x00 and stat values match expected, declare success

### Part 4: Alternative — errSP Error Recovery Frame
If Part 3 still fails with E_Memory:
1. Seed errSP (0xD008E0) to point to a PushErrorHandler-style frame
2. The frame should catch errors and allow OneVar to continue
3. Re-run OneVar with 200K step budget
4. Report what happens after error recovery

## VAT Entry Format (from session 139)
9 bytes stored HIGH→LOW from OPBase-1 downward:
```
+0 (highest): type byte (0x00 for Real, 0x01 for RealList)
-1: version = 0x00
-2: page = 0x00
-3: data_ptr low byte
-4: data_ptr mid byte  
-5: data_ptr high byte
-6: name byte 1
-7: name byte 2 (0x00 for single-char names)
-8: name byte 3 (0x00)
```
After each entry, decrement OPBase by 9 and update progPtr.

## List Data Format
At the data pointer address:
```
[count_lo, count_hi, elem1(9 bytes BCD), elem2(9 bytes BCD), ...]
```
For [10,20,30,40,50]: count=5, then 5 × 9-byte BCD reals.
