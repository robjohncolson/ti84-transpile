# Auto-Session 139 P3: CreateRList Investigation + Alternative List Creation

## Goal
Investigate why CreateRList (0x082398) falls through to boot code at 0x000005, and find an alternative path to create a List variable with a proper VAT entry so OneVar can find L1.

## Background

- CreateRList (0x082398) stalls at 0x0019B5 — falls through to z80 boot area
- Internal call chain from 0x082398 hits missing block at 0x000005

- All 7 stat addresses now have transpiled blocks

Session 136/138 proved:

- CreateReal works to create Real variables (Ans, letter vars)
- The pattern: set OP1 with variable name → call CreateXxx → OS allocates in VAT

### Part 2: Try CreateVar / InsertMem approach
The OS has a general-purpose variable creation routine. Try these alternatives:

2. **Manual VAT entry construction**: 
   a. After MEM_INIT, the VAT starts at progPtr (0xD3FFFF) and grows downward
   b. A VAT entry for a list is: [data_ptr_low, data_ptr_mid, data_ptr_high, type_byte=0x01, name_len, name_byte=0x01, ...]
   c. Write a VAT entry manually at progPtr-6 (or wherever), update progPtr

## Key references
- VAT structure: each entry is variable-length, stored backwards from progPtr
- Entry format (from CE documentation): 
  - Byte 0-2: data pointer (24-bit LE, points to variable data in userMem area)
  - Byte 3: type (0x00=Real, 0x01=RealList, 0x02=Matrix, 0x03=Equ, 0x04=String, 0x05=Program...)
  - Byte 4: name length (for lists: 0x01 if named L1-L6, 0x02+ for custom lists)
  - Byte 5+: name bytes
- List data format: [element_count_word (2 bytes LE), element1 (9 bytes BCD), element2 (9 bytes BCD), ...]
