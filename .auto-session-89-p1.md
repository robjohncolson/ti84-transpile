# Session 89 P1: RclVarSym probe — proper variable recall

## Goal
Build a probe that validates the RclVarSym pathway for recalling a Real variable's numeric value, independent of ParseInp.

## Background
Session 88 proved ParseInp doesn't recall variable values — it redirects the token stream to the variable's data area and re-parses BCD bytes as tokens. To actually recall a Real variable's value, the OS uses RclVarSym (JT slot at 0x020F60, impl at 0x09AC77) or similar routines. Nobody has tested RclVarSym directly yet.

## What to do
Create `TI-84_Plus_CE/probe-phase25ad-rclvarsym.mjs` that:

1. **Cold boots + MEM_INIT** (follow the exact pattern from `probe-phase25z-full-pipeline.mjs` — `coldBoot()` then call MEM_INIT at 0x09DEE0)
2. **CreateReal("A") with value 42.0** — call CreateReal at 0x08238A with OP1 set to `[0x00, 0x41, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]` (type=Real, name="A"). After CreateReal returns, DE points to the variable's data area. Write 42.0 BCD (`[0x00, 0x81, 0x42, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]`) to DE.
3. **Call RclVarSym at 0x09AC77** with OP1 re-set to `[0x00, 0x41, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]` (variable "A"). Use the same call setup pattern as other probes (push FAKE_RET=0x7FFFFE onto stack, set SP, run from 0x09AC77 in ADL mode).
4. **Check OP1 after RclVarSym returns** — if RclVarSym is the correct recall routine, OP1 should contain `[0x00, 0x81, 0x42, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]` = 42.0.

## Key addresses and constants
- MEM_INIT: 0x09DEE0
- CreateReal: 0x08238A
- RclVarSym impl: 0x09AC77 (JT slot 0x020F60)
- OP1: 0xD005F8 (9 bytes)
- errNo: 0xD008DF
- errSP: 0xD008E0
- FAKE_RET: 0x7FFFFE
- Stack top: 0xD1A87E
- IY: 0xD00080
- IX: 0xD1A860

## Reference file
Copy the structure of `TI-84_Plus_CE/probe-phase25z-full-pipeline.mjs` — it has the exact `coldBoot()`, `prepareCallState()`, `snapshotPointers()`, and memory helper patterns you need.

## Error frame setup
Set up the standard 6-byte error catch frame before calling RclVarSym:
```
write24(mem, errSP_addr, sp_value)  // or push error frame as in probe-phase25x
```
Actually, use the SIMPLE error frame from session 83 probes: just set errSP to point to a stack entry containing FAKE_RET. This way if RclVarSym hits an error (e.g., variable not found), it longjmps to FAKE_RET instead of crashing.
