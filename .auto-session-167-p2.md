# Task: Test replacing compound function 0x07C747 with simple OP1→OP2 copy in gcd

## Context
You are working on a TI-84 Plus CE ROM transpilation project. The gcd handler has a bug: InvSub computes OP2-OP1 but both operands are the same value because the compound function at 0x07C747 copies OP1 into OP2 (destroying the original OP2 value).

The compound function at 0x07C747 does: OP1→OP2 copy, then normalize OP1 (zeros it for exp >= 0x80), then negate zero, then FPAdd(0, OP2) = OP2, effectively a no-op for OP1 value. Session 165 confirmed this.

The SIMPLE OP1→OP2 copy function is at 0x07F8FA — it just does a 9-byte memory copy from OP1 to OP2 with no side effects.

**HYPOTHESIS**: If the compound function's normalization side effects (flags, B register, etc.) are what the algorithm needs (not just the OP1→OP2 copy), then replacing 0x07C747 with 0x07F8FA should produce different (possibly correct) results. If the result is the same E_Domain, then the bug is elsewhere.

## What to do
Build a probe at `TI-84_Plus_CE/probe-phase167-simple-copy-test.mjs` that:

### Part A: Control — gcd(12,8) with NO intervention
Run gcd(12,8) normally from 0x068D61 with up to 2000 steps. Record:
- Final OP1, OP2 values
- errNo (at 0xD008AF)
- Total steps
- Number of times 0x07C747 was called (add a counter hook)

### Part B: Intercept — redirect 0x07C747 calls to 0x07F8FA
Hook the block at 0x068D8B (which contains CALL 0x07C747). When execution reaches this PC, instead of running the block, manually:
1. Copy 9 bytes from OP1 (0xD005F8) to OP2 (0xD00603)
2. Skip to the next instruction after the CALL (PC = 0x068D8F, which is the next instruction after the 4-byte CALL)
3. Log each intercept with step number and OP1/OP2 values

Run gcd(12,8) with this intercept. Record same metrics as Part A.

### Part C: Intercept with OP2 preservation
The REAL fix might be that OP2 should NOT be overwritten at all. Try: when 0x068D8B is reached, simply skip the CALL entirely (set PC = 0x068D8F without doing any copy). This preserves whatever was in OP2 before the compound function call.

Run gcd(12,8) with this intercept. Record same metrics.

### Part D: Intercept with OP2 loaded from FPS (the pushed 8.0)
The FPS has OP2=8.0 pushed at the start. Try: when 0x068D8B is reached, pop 8.0 from FPS into OP2 (read 9 bytes from FPS pointer - 9, copy to OP2, but don't actually decrement the FPS pointer since we may need it again). Then skip the CALL.

Run gcd(12,8) with this intercept. Record same metrics.

## Setup pattern (copy this exactly for EACH test variant)
```javascript
import { createCPU } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';
import fs from 'fs';

const rom = new Uint8Array(fs.readFileSync('./TI-84_Plus_CE/ROM.rom'));
const bus = createPeripheralBus({ timerInterrupt: false });
const cpu = createCPU(bus);

for (let i = 0; i < rom.length; i++) cpu.mem[i] = rom[i];

const txSrc = fs.readFileSync('./TI-84_Plus_CE/ROM.transpiled.js', 'utf8');
const BLOCKS = {};
new Function('BLOCKS', txSrc)(BLOCKS);

function readOP(base) {
  return Array.from(cpu.mem.slice(base, base + 9));
}
function fmtOP(base) {
  return readOP(base).map(b => b.toString(16).padStart(2,'0')).join(' ');
}

const OP1 = 0xD005F8, OP2 = 0xD00603;

function writeFloat(addr, value) {
  for (let i = 0; i < 9; i++) cpu.mem[addr + i] = 0;
  if (value === 0) { cpu.mem[addr] = 0; cpu.mem[addr+1] = 0x80; return; }
  const sign = value < 0 ? 0x80 : 0x00;
  value = Math.abs(value);
  const exp = Math.floor(Math.log10(value)) + 1;
  cpu.mem[addr] = sign;
  cpu.mem[addr + 1] = 0x80 + exp;
  let mantissa = value / Math.pow(10, exp);
  for (let i = 2; i < 9; i++) {
    mantissa *= 100;
    const byte = Math.floor(mantissa);
    cpu.mem[addr + i] = ((Math.floor(byte / 10) & 0xF) << 4) | (byte % 10);
    mantissa -= byte;
  }
}

function setupGcd() {
  // Reset CPU state
  const freshBus = createPeripheralBus({ timerInterrupt: false });
  // Re-init is complex, so just reset key state:
  cpu.sp = 0xD1A880;
  cpu.sp -= 3;
  cpu.mem[cpu.sp] = 0xFE; cpu.mem[cpu.sp+1] = 0xFF; cpu.mem[cpu.sp+2] = 0x7F;
  
  writeFloat(OP1, 12);
  writeFloat(OP2, 8);
  
  const fpsBase = 0xD1AA00;
  cpu.mem[0xD02589] = fpsBase & 0xFF;
  cpu.mem[0xD0258A] = (fpsBase >> 8) & 0xFF;
  cpu.mem[0xD0258B] = (fpsBase >> 16) & 0xFF;
  for (let i = 0; i < 9; i++) cpu.mem[fpsBase + i] = cpu.mem[OP2 + i];
  cpu.mem[0xD02589] = (fpsBase + 9) & 0xFF;
  cpu.mem[0xD0258A] = ((fpsBase + 9) >> 8) & 0xFF;
  cpu.mem[0xD0258B] = ((fpsBase + 9) >> 16) & 0xFF;
  
  cpu.iy = 0xD00080;
  cpu.pc = 0x068D61;
  
  // Clear errNo
  cpu.mem[0xD008AF] = 0;
}
```
