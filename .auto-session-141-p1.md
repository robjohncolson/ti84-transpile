# Session 141 P1: Add FpPop seed (0x082957) and test gcd

## Context
We're transpiling the TI-84 Plus CE ROM to JavaScript. The transpiler at `scripts/transpile-ti84-rom.mjs` lifts eZ80 machine code into JS blocks. Seeds are starting addresses for the block walker.

Session 140 traced the gcd dispatch chain: `0x06859B → 0x0689DE → jump table → 0x068D3D` (real gcd handler). The FIRST instruction at 0x068D3D is `CALL 0x082957` (FpPop — pops a value from the FP stack into an OP register). **0x082957 is a MISSING BLOCK** — without it, FP stack pop doesn't execute, OP1/OP2 retain garbage, type validator at 0x07F831 fails, and E_Domain (0x84) is raised.

## Task

### Step 1: Add the seed
In `scripts/transpile-ti84-rom.mjs`, find the seed array near line 22132 (the "Phase 138 P3" stat seeds section, ending with `{ pc: 0x0aaab8, mode: 'adl' }`). AFTER that entry and BEFORE the `];` closing bracket, add:

```js
    // Phase 141 P1: FpPop seed — unblocks gcd dispatch (session 140 root cause).
    { pc: 0x082957, mode: 'adl' },  // FpPop — pops FP stack into OP register
```

### Step 2: Retranspile
Run: `node scripts/transpile-ti84-rom.mjs`
The output goes to `TI-84_Plus_CE/ROM.transpiled.js` (~214 MB). Record the new seed count and block count from stderr.

### Step 3: Run golden regression
Run: `node TI-84_Plus_CE/probe-phase99d-home-verify.mjs`
Must pass 26/26. If it fails, STOP and report.

### Step 4: Build and run a gcd probe
Create `TI-84_Plus_CE/probe-phase141-gcd-test.mjs`. This probe should:

1. Import the transpiled ROM and runtime
2. Cold boot + MEM_INIT (use the pattern from `probe-phase140-gcd-disasm.mjs` — read that file for the exact setup)
3. Seed the FP stack with two BCD real numbers: 12.0 and 8.0
   - FPS pointer is at 0xD0258D (3-byte LE pointer)
   - Each FP entry is 9 bytes: [type, exponent, mantissa×7] 
   - 12.0 BCD = [0x00, 0x81, 0x12, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]
   - 8.0 BCD = [0x00, 0x80, 0x80, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]
   - Push 12.0 first (deeper), then 8.0 (top) to the FP stack area
   - Update FPS to point past both entries
4. Set the FP category byte at 0xD0060E to 0x28 (gcd operation code)
5. Call the gcd dispatch entry at 0x06859B (or try 0x068D3D directly)
6. Run up to 50,000 steps
7. Report: errNo (at 0xD008DF), OP1 (9 bytes at 0xD005F8), steps taken, whether E_Domain (0x84) was hit

**Expected**: If FpPop now works, the gcd handler should pop both operands from FPS into OP1/OP2, validate types, and compute gcd(12,8) = 4.0. OP1 should be [0x00, 0x80, 0x40, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00].

### Step 5: Report
Print results clearly: seed/block counts, golden regression pass/fail, gcd probe results.
