# Task: Backfill phase-report labels with canonical JT names

## Context
Phase 25H-a built a jump-table cross-reference (`TI-84_Plus_CE/phase25h-a-jump-table.json`) mapping 980 ROM addresses to canonical OS names (957 named). Phase reports throughout the project scatter bare hex addresses (like `0x07C77F`, `0x08C331`, `0x0A1B5B`) without their canonical names. A one-shot script can find these and annotate them inline.

## Task
Create a script `TI-84_Plus_CE/backfill-jt-labels.mjs` that:

1. Reads `TI-84_Plus_CE/phase25h-a-jump-table.json` to build a map of `impl_address → name` (the `impl` field in each entry maps to the `name` field).
2. Finds all `phase*-report.md` files in `TI-84_Plus_CE/`.
3. For each report, scans for bare hex addresses matching the pattern `0x[0-9A-Fa-f]{6}` (case-insensitive).
4. If the address matches a JT impl address AND is NOT already followed by ` (= SomeName)` or `(SomeName)` or similar annotation, appends ` (= Name)` after the address.
5. Writes the modified file back only if changes were made.
6. Prints a summary: how many files scanned, how many modified, how many addresses annotated.

## Important details
- The JSON file structure: it's an array of objects, each with at least `slot` (hex string), `impl` (hex string), and `name` (string). Example:
  ```json
  {"slot": "0x0201C0", "impl": "0x07C77F", "name": "FPAdd"}
  ```
- Match BOTH `slot` addresses AND `impl` addresses — some reports reference the JT slot address (0x0201C0) rather than the impl address.
- The regex for already-annotated: if the hex address is immediately followed by ` (= ` or ` (` + uppercase letter, skip it (already labeled).
- Be careful with addresses that appear inside code blocks (backtick-fenced) — annotate those too, they're the most common location.
- Handle both uppercase and lowercase hex: `0x07C77F` and `0x07c77f` should both match.

## Verification
Run:

Golden regression is not affected (this only touches .md files), but run it anyway:


## Constraints
- Create ONLY `TI-84_Plus_CE/backfill-jt-labels.mjs`.
- Do NOT modify any .mjs or .js files.
- Only modify .md files that match `phase*-report.md`.
- Do NOT modify `CONTINUATION_PROMPT_CODEX.md`, `CLAUDE.md`, or any spec files.
- If a report has no matchable addresses, skip it silently.
- Use Node.js fs/path modules only (no external dependencies).
