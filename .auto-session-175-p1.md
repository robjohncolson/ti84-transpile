# Task: Fix standalone interpreter .LIL/.SIS/.SIL/.LIS prefix mode-switch handling

## Context
File: `TI-84_Plus_CE/probe-phase172-standalone-interp.mjs`

This is a standalone eZ80 interpreter used for comparing execution against the transpiled ROM. It currently matches 714/715 blocks with the transpiled output. The NEXT divergence (step 714, block 0x001AFA) is caused by the standalone interpreter ignoring `.LIL/.SIS/.SIL/.LIS` prefix bytes on branch instructions.

## Problem
The eZ80 instruction set has prefix bytes (0x40=.SIS, 0x49=.LIS, 0x52=.SIL, 0x5B=.LIL) that can precede branch instructions (JP, CALL, JR, RET, etc.). These prefixes change the execution mode of the **target** (the CPU's ADL flag after the branch). The decoder (`ez80-decoder.js`) correctly decodes these and includes a `modePrefix` field on the instruction object (e.g., `modePrefix: 'lil'`).

The transpiler at `scripts/transpile-ti84-rom.mjs` lines 248-250 computes `targetMode` from `modePrefix`:
- If `modePrefix` starts with 's' â†’ target mode is 'z80'
- If `modePrefix` starts with 'l' â†’ target mode is 'adl'
- If no prefix â†’ target mode stays the same as current mode

But in the standalone interpreter's `execStandalone()` function (around lines 1141-1222), none of the branch handlers (jp, jp-conditional, call, call-conditional, rst, jr, jr-conditional, djnz, ret, retn, reti, ret-conditional) check `instr.modePrefix` to update `cpu.madl`.

## The Fix
In `execStandalone()`, after each branch instruction sets `cpu.pc`, add logic to update `cpu.madl` if the instruction has a `modePrefix`:
- If `instr.modePrefix` exists and starts with 'l' (lil or lis), set `cpu.madl = 1` (ADL mode)
- If `instr.modePrefix` exists and starts with 's' (sis or sil), set `cpu.madl = 0` (z80 mode)
- If no prefix, leave `cpu.madl` unchanged

The cleanest approach: add a helper function right before the `return 'branch'` in each branch case, OR add a single block after all branch cases but before returning. Since all branch cases `return 'branch'`, the best approach is to add the mode-switch logic as a helper called before each `return 'branch'` in the branch instruction cases.

Suggested implementation: Add a helper at the top of `execStandalone()`:

```javascript
function applyModePrefix(cpu, instr) {
  if (instr.modePrefix) {
    cpu.madl = instr.modePrefix[0] === 'l' ? 1 : 0;
  }
}
```

Then call `applyModePrefix(cpu, instr);` before every `return 'branch';` in these cases:
- `case 'ret':` / `case 'retn':` / `case 'reti':` (line 1141-1145)
- `case 'ret-conditional':` (line 1148-1155)
- `case 'call':` (line 1157-1161)
- `case 'call-conditional':` (line 1163-1171)
- `case 'rst':` (line 1173-1177)
- `case 'jp':` (line 1179-1182)
- `case 'jp-conditional':` (line 1184-1191)
- `case 'jp-indirect':` (line 1193-1196)
- `case 'jr':` (line 1198-1201)
- `case 'jr-conditional':` (line 1203-1210)
- `case 'djnz':` (line 1212-1222)

Do NOT add it to `case 'stmix':` or `case 'rsmix':` â€” those already handle mode changes directly.
