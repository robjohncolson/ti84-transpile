# Auto-Session 153 P3: FPS stack tracking across gcd steps 1-353

## Goal
Build a probe `TI-84_Plus_CE/probe-phase153-gcd-fps-track.mjs` that runs gcd(12,8) via 0x068D3D and tracks every FPS pointer change (push/pop) for the first 353 steps. This determines WHEN the original 8.0 is popped back from FPS into OP2, and whether the pop happens before or after the division.

## Context
- At step 1, OP2=8.0. At step 18, 8.0 is pushed to FPS and OP2 is overwritten with 1.0.
- At step 353 (second TypeValidator), OP2=8.0 again — so somewhere between steps 18-353, 8.0 was popped back.
- The question: does 8.0 return to OP2 BEFORE or AFTER the division? If after, then division uses OP2=1.0 → INT(12/1)=12 → 12-12*8=-84 (wrong).
- FPS pointer is at 0xD0258D (3 bytes). Each push increments by 9, each pop decrements by 9.
