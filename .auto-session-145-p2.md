- Full entry hits E_JError at step 45 due to BIT 5 check at 0x08005A
- Scenario C2 (full entry + re-SET bit 5 via onBlock after the RES) avoids E_JError but hits E_Domain (0x84) at step 1521 with 323 unique PCs
- The full entry does `JR +6` which skips `LD B,0x00` at 0x0A9329

## Standard probe boilerplate
```js
import { readFileSync } from 'fs';
import { gunzipSync } from 'zlib';

const ROM_PATH = 'TI-84_Plus_CE/ROM.rom';
const TRANSPILED_PATH = 'TI-84_Plus_CE/ROM.transpiled.js';

let src;
try { src = readFileSync(TRANSPILED_PATH, 'utf8'); }
catch { src = gunzipSync(readFileSync(TRANSPILED_PATH + '.gz')).toString('utf8'); }

const { createExecutor } = await import('./TI-84_Plus_CE/cpu-runtime.js');
const { createPeripheralBus } = await import('./TI-84_Plus_CE/peripherals.js');

const mem = new Uint8Array(16 * 1024 * 1024);
const rom = readFileSync(ROM_PATH);
mem.set(rom, 0);

const peripherals = createPeripheralBus({ timerInterrupt: false });
const ex = createExecutor(mem, peripherals);
new Function('executor', src)(ex);
```
