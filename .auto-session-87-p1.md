# Task: Create Ans variable before ParseInp — does errNo clear?

## Context

You are working on a TI-84 Plus CE ROM transpilation project. The transpiled ROM runs in JavaScript via `cpu-runtime.js`. Session 86 proved that `errNo=0x8D` after ParseInp("2+3") is a RESIDUAL — set at step 27 when ChkFindSym fails on OP1=zeros, caught internally, never cleared. The StoAns sequence at steps 832-917 also calls ChkFindSym for Ans (OP1[0]=0xFF) and doesn't find it.

## Your task

Create a probe `TI-84_Plus_CE/probe-phase25ab-ans-variable.mjs` that:

1. Cold boots the ROM
2. Calls MEM_INIT at 0x09DEE0 (returns in ~17 steps)
3. Creates an Ans variable via CreateReal at 0x08238A with OP1 set for Ans: `[0x00, 0x72, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]` (type=0x00 for RealObj, name=0x72 for tAns per ti84pceg.inc). Budget: 50,000 steps.
4. Clears OP1 back to zeros
5. Seeds token buffer at 0xD00800 with `[0x32, 0x70, 0x33, 0x3F]` ("2+3" with end token)
6. Sets begPC/curPC (0xD02317/0xD0231A) to TOKEN_BUFFER_ADDR, endPC (0xD0231D) to TOKEN_BUFFER_ADDR + 4
7. Calls ParseInp at 0x099914. Budget: 1,500,000 steps.
8. Checks:
   - Does errNo == 0x00? (the key question)
   - Does OP1 contain 5.0? (`[0x00, 0x80, 0x50, 0x00, ...]`)
   - How many steps?
   - What's the final PC?

Also run a CONTROL (same but WITHOUT creating Ans variable) for comparison. The control should show errNo=0x8D and OP1=5.0 (918 steps) as established in session 86.

Write a markdown report to `TI-84_Plus_CE/phase25ab-ans-variable-report.md`.

## Template

Use `TI-84_Plus_CE/probe-phase25aa-empty-op1-with-var.mjs` as your template — it has the exact same structure (cold boot → MEM_INIT → CreateReal → ParseInp with control). Copy its patterns for:
- Imports (cpu-runtime.js, peripherals.js, fp-real.mjs)
- `createPeripheralBus({ timerInterrupt: false })`
- The `createExecutor` + cold boot + postInit pattern
- Error-frame seeding (errSP, FAKE_RET, ERR_CATCH_ADDR)
- Token buffer setup at 0xD00800
- Pointer snapshot helpers

## Key constants

```
MEM_INIT_ENTRY = 0x09DEE0
CREATEREAL_ENTRY = 0x08238A
PARSEINP_ENTRY = 0x099914
OP1_ADDR = 0xD005F8
ERR_NO_ADDR = 0xD008DF

BEGPC_ADDR = 0xD02317

```

The Ans variable type byte in OP1[0] should be 0x00 (RealObj). The name byte in OP1[1] should be 0x72 (tAns).

## Constraints
