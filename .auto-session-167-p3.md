# Task: Trace stack depth (PUSH/POP balance) through gcd inner body

## Context
You are working on a TI-84 Plus CE ROM transpilation project. The gcd handler inner body at 0x068D82-0x068DA1 does:
1. BIT 0,B (0x068D82)
2. PUSH AF (0x068D84) — saves Z flag (Z=1 if exp<0x82)
3. CALL 0x07F8A2 — OP1→OP4
4. CALL 0x07C747 — compound OP1→OP2 (copy+normalize+negate+FPAdd)
5. CALL 0x07F95E — OP1→OP3
6. CALL 0x07F8B6 — OP4→OP2
7. CALL 0x07C74F — InvSub (OP2-OP1)
8. CALL 0x068D20 — gcd_helper
9. CALL 0x07FD69 — exponent check
10. JR NZ -70 → 0x068D5D (loop back)

The PUSH AF at step 2 pushes 2 bytes (A + F). If any of the 7 CALLs (steps 3-9) have an unbalanced PUSH/POP, the RET from those CALLs would pop the wrong return address, OR the gcd's own POP AF would get the wrong value.

**HYPOTHESIS**: A stack imbalance inside one of the called functions could explain the bug — if a subroutine doesn't properly balance its stack, the saved AF (with the exponent flag) would be lost or corrupted.

## What to do
Build a probe at `TI-84_Plus_CE/probe-phase167-stack-balance.mjs` that:

### Part A: Track SP through the entire gcd inner body (iteration 1)
Run gcd(12,8). At every block execution, log:
- Step number
- PC (hex)
- SP value

Focus on iteration 1 (roughly steps 50-340 based on previous sessions). At each of the 7 CALL instructions (steps 3-9 in the list above), record:
- SP before the CALL
- SP at the first instruction inside the called function
- SP when execution returns (at the instruction after the CALL)

Verify that SP is the same before CALL and after RET for each of the 7 calls.

### Part B: Find the SP at PUSH AF and at any POP AF
Record the SP value right before PUSH AF at 0x068D84 (should be SP_before_push). After PUSH, SP = SP_before_push - 2 (eZ80 pushes 3 bytes in ADL mode, or 2 in Z80 mode — check which mode we're in by looking at cpu.adl).

Then track the first instruction that restores SP to SP_before_push (which would be the POP AF or equivalent). Report:
- What instruction is at that point
- What values are popped (A and F)
- Whether the F register's Z flag matches what was pushed

### Part C: Stack watermark
Track the minimum SP value reached during the inner body execution. This shows the maximum stack depth. Compare with the SP at PUSH AF to see how deep the subroutine calls go.

Report a summary table:
```
Call target    | SP before CALL | SP at entry | SP after RET | Balanced?
0x07F8A2       |                |             |              |
0x07C747       |                |             |              |
...etc
```

## Setup pattern
```javascript
import { createCPU } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';
import fs from 'fs';

const rom = new Uint8Array(fs.readFileSync('./TI-84_Plus_CE/ROM.rom'));
const bus = createPeripheralBus({ timerInterrupt: false });
const cpu = createCPU(bus);

for (let i = 0; i < rom.length; i++) cpu.mem[i] = rom[i];

const txSrc = fs.readFileSync('./TI-84_Plus_CE/ROM.transpiled.js', 'utf8');
const BLOCKS = {};
new Function('BLOCKS', txSrc)(BLOCKS);

function fmtOP(base) {
  return Array.from(cpu.mem.slice(base, base + 9)).map(b => b.toString(16).padStart(2,'0')).join(' ');
}

const OP1 = 0xD005F8, OP2 = 0xD00603;

function writeFloat(addr, value) {
  for (let i = 0; i < 9; i++) cpu.mem[addr + i] = 0;
  if (value === 0) { cpu.mem[addr] = 0; cpu.mem[addr+1] = 0x80; return; }
  const sign = value < 0 ? 0x80 : 0x00;
  value = Math.abs(value);
  const exp = Math.floor(Math.log10(value)) + 1;
  cpu.mem[addr] = sign;
  cpu.mem[addr + 1] = 0x80 + exp;
  let mantissa = value / Math.pow(10, exp);
  for (let i = 2; i < 9; i++) {
    mantissa *= 100;
    const byte = Math.floor(mantissa);
    cpu.mem[addr + i] = ((Math.floor(byte / 10) & 0xF) << 4) | (byte % 10);
    mantissa -= byte;
  }
}

cpu.sp = 0xD1A880;
cpu.sp -= 3;
cpu.mem[cpu.sp] = 0xFE; cpu.mem[cpu.sp+1] = 0xFF; cpu.mem[cpu.sp+2] = 0x7F;

writeFloat(OP1, 12);
writeFloat(OP2, 8);

const fpsBase = 0xD1AA00;
cpu.mem[0xD02589] = fpsBase & 0xFF;
cpu.mem[0xD0258A] = (fpsBase >> 8) & 0xFF;
cpu.mem[0xD0258B] = (fpsBase >> 16) & 0xFF;
for (let i = 0; i < 9; i++) cpu.mem[fpsBase + i] = cpu.mem[OP2 + i];
cpu.mem[0xD02589] = (fpsBase + 9) & 0xFF;
cpu.mem[0xD0258A] = ((fpsBase + 9) >> 8) & 0xFF;
cpu.mem[0xD0258B] = ((fpsBase + 9) >> 16) & 0xFF;

cpu.iy = 0xD00080;
cpu.pc = 0x068D61;
cpu.mem[0xD008AF] = 0;
```
