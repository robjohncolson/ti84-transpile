# Task: Disassemble 0x07F8B6 and trace its chain into FPDiv at 0x07CA48

## Context
This is a TI-84 Plus CE ROM transpilation project. The ROM is lifted to JavaScript. We're investigating why FPDiv(1200,1200) produces 0 instead of 1 during the gcd(12,8) computation.

The gcd algorithm body at 0x068D82 does:
1. AbsOP1(0x07F8A2)
2. OP1toOP2(0x07C747) — both OP1 and OP2 become 1200
3. OP1→OP3 copy(0x07F95E)
4. **CALL 0x07F8B6** ← THIS IS THE TARGET
5. OP2toOP1(0x07C74F)
6. gcd_helper(0x068D20)

0x07F8B6 is the function that chains into the FP computation at 0x07CA48 (FPDiv area). Session 154 labeled it "AbsOP2" but we need to verify.

## Deliverables
Build a probe `TI-84_Plus_CE/probe-phase156-07f8b6-disasm.mjs` that:

1. Reads the ROM binary from `TI-84_Plus_CE/ROM.rom` (4MB, raw binary)
2. Statically disassembles bytes at 0x07F8B6 through 0x07F900 (about 74 bytes) as eZ80 ADL-mode instructions
3. For each CALL target found, also disassemble the first ~20 bytes of that target
4. Specifically look for: what does 0x07F8B6 do before entering the 0x07CA48 area? Does it modify OP1/OP2? Does it set up registers?
5. Also disassemble 0x07CA48 through 0x07CA70 to see the FPDiv loop entry

Use the eZ80 decoder at `TI-84_Plus_CE/ez80-decoder.js` for disassembly:
