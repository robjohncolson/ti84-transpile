# Task: Dense trace gcd steps 171-446 to find OP1 restoration mechanism

## Context

The TI-84 CE ROM gcd handler runs an Euclidean algorithm. Session 160 P3 traced the gcd(12,8) execution and found:

- Step 58: OP1toOP2 entry, OP1=1200 (exponent bumped by gcd_helper)
- Step 62: normalization entry at 0x07CA48
- Step 161: after normalization, OP1=0 (mantissa zeroed), OP2=1200 (preserved)
- Steps 170-171: 0x07CA06 (InvOP1S = negate OP1) called, then FPAdd at 0x07C77F
- **Step 190: OP1=1200 RESTORED with full mantissa**
- Step 446: SECOND OP1toOP2 entry

The question: HOW does OP1 go from 0 (zeroed by normalization) back to 1200 between steps 171 and 190? Something in the FPAdd path or the gcd loop body restores OP1. We need a dense trace of every block in this range.
