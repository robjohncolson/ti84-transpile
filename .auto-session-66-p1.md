# Phase 25G-d — Establish real index formula for 0x09F79B scan code translation table

## Context

The ROM at `0x09F79B` contains a 228-byte scan code translation table (4 planes × 57 entries). Phase 25G-c decoded this table with an inline dictionary, but the physical-label alignment is WRONG — the table is NOT indexed by `(group<<4)|bit` raw scancode. Evidence: APPS→'8', '2' key→'D' patterns don't match.

We need to figure out the REAL index formula by instrumenting a live keystroke through the OS event loop and watching which byte of the table the OS actually reads.

## Your task

Create a probe file `TI-84_Plus_CE/probe-phase25g-d-index.mjs` that:

1. Boots the OS to HALT (5000 steps from 0x000000 in z80 mode)
2. Sets up the ISR frame (same as `probe-phase25g-eventloop.mjs` — see that file for the pattern)
3. Presses a KNOWN key — use the `1` key: `keyMatrix[3] = 0xFD` (Group 4 = SDK Group 3, bit 1 = key `1`). The raw scancode for `1` is `(3<<4)|1 = 0x31`. If the table returns `0x31` ('1' ASCII), the index formula is identity. If not, we learn the mapping.
4. **Instruments `cpu.read8`** to log every read in the range `0x09F79B..0x09F87E` (228 bytes). For each hit, log: `addr`, `addr - 0x09F79B` (= table offset), `value` (the byte read), and the current `cpu.pc`.
5. Runs the ISR from 0x000038 with keyboard IRQ active, up to 100,000 steps
6. Reports:
   - All table reads observed (address, offset, value, caller PC)
   - Whether 0x00B608 was reached
   - The inferred index formula: `offset = f(scancode)` where scancode = 0x31

## Implementation details

### Instrumenting read8

The `cpu` object from `createExecutor()` has a `read8(addr)` method. You can monkey-patch it:

```javascript
const originalRead8 = cpu.read8.bind(cpu);
const tableReads = [];
cpu.read8 = function(addr) {
  const val = originalRead8(addr);
  if (addr >= 0x09F79B && addr <= 0x09F87E) {
    tableReads.push({ addr, offset: addr - 0x09F79B, value: val, pc: cpu.pc });
  }
  return val;
};
```

### ISR setup pattern (copy from probe-phase25g-eventloop.mjs)

```javascript
import { createExecutor } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';

// Boot
const boot = executor.runFrom(0x000000, 'z80', { maxSteps: 5000, maxLoopIterations: 32 });

// Setup ISR frame
cpu.halted = false;
cpu.iff1 = 1;
cpu.iff2 = 1;
cpu.madl = true;
cpu.sp = 0xD1A87E;
cpu.push(boot.lastPc + 1); // return address

// Write callback pointer and system flag
write24(memory, 0xD02AD7, 0x0019BE); // event loop entry
memory[0xD0009B] |= 0x40; // system flag

// Press the '1' key
peripherals.keyboard.keyMatrix.fill(0xFF); // all released
peripherals.keyboard.keyMatrix[3] = 0xFD;  // Group 4 (SDK Group 3), bit 1 = '1'
peripherals.setKeyboardIRQ(true);
peripherals.write(0x5006, 0x08); // enable keyboard IRQ mask
```

### Key addresses
- `0x09F79B` — table base (228 bytes)
- `0x0019BE` — OS event loop entry
- `0x000038` — ISR entry
- `0xD0058E` — keyboard OS scan-code output
- `0x00B608` — target block (may or may not be reached)

### Keyboard matrix reference
- `keyMatrix[N]` = SDK Group(7-N)
- For key '1': SDK Group 3 = keyMatrix[4]... wait, let me be precise:
  - SDK Group 3 maps to keyMatrix[7-3] = keyMatrix[4]
  - The '1' key is in SDK Group 4 (row 4), bit 1
  - SDK Group 4 = keyMatrix[7-4] = keyMatrix[3]
  - So: `keyMatrix[3] = 0xFF ^ (1 << 1) = 0xFD`
  - Raw scancode = `(3 << 4) | 1 = 0x31`
