## Candidate Font Base and Stride

- Strongest table base candidate: `0x0040ee`.
- Shared space glyph window: `0x0040ee-0x004109` (2100 total hits across all three runs).
- Candidate glyph stride: `0x001c` (28 bytes).
- Cross-check from 'A' vs '0': `28` bytes/glyph, which matches `28`.
- Inferred addressing formula: `glyphAddr(charCode) = 0x0040ee + (charCode - 0x20) * 0x001c`.

Reasoning:

- Each run prints 26 glyphs: 25 spaces plus the seeded first character. The shared range `0x0040ee-0x004109` therefore appears 25 times per run -> `700 = 700` total reads in each experiment.
- 'A' unique glyph window: `0x00448a-0x0044a5`.
- 'B' unique glyph window: `0x0044a6-0x0044c1`.
- '0' unique glyph window: `0x0042ae-0x0042c9`.
- `0x0044a6 - 0x00448a = 0x001c` -> adjacent ASCII letters are one stride apart.
- `0x00448a - 0x0042ae = 0x01dc = 0x11 * 0x001c` -> '0' to 'A' spans exactly 17 glyph slots.
- `0x00448a - 0x0040ee = 0x039c = (0x41 - 0x20) * 0x001c` -> the table lines up perfectly with ASCII space as the base character.
