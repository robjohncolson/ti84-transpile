Phase 54.1 — Find CALC menu renderer via memory-access tracing.

CONTEXT:
- TI-84 ROM transpilation project. Phase 53 discovered the CALC menu strings (0x055d96+, 63 graph CALC strings) have a POINTER TABLE at 0x055d00 with 42+ 3-byte pointers to the strings.
- No code in ROM directly references 0x055d00 as an LD HL immediate. The pointer table is accessed via RUNTIME-COMPUTED addressing (likely base+offset or indirect via register).
- GOAL: Find the function that READS from 0x055d00-0x055dFF (the pointer table) OR from 0x055d96-0x055fff (the CALC strings themselves). That function is the CALC menu renderer.

APPROACH: Wrap cpu.read8 in a probe to log reads from the CALC string area, then run various candidate entry points and see which ones trigger reads.

EXPLICIT-INIT TEMPLATE:
- boot 0x000000 z80, then 0x08C331 adl init, then 0x0802B2 adl SetTextFgColor, snapshot RAM

YOUR TASK:
1. CREATE TI-84_Plus_CE/probe-phase54-calc-trace.mjs

2. Wrap cpu.read8 after the executor creates it:
   ```js
   const origRead8 = cpu.read8.bind(cpu);
   const calcReads = [];  // {addr, value, blockPc, step}
   let currentBlockPc = 0;
   let stepCount = 0;
   cpu.read8 = (addr) => {
     const value = origRead8(addr);
     // CALC pointer table area (0x055d00-0x055dFF) + CALC strings (0x055e00-0x055fff)
     if (addr >= 0x055d00 && addr <= 0x055fff) {
       calcReads.push({ addr, value, blockPc: currentBlockPc, step: stepCount });
     }
     return value;
   };
   ```

3. Use onBlock callback in runFrom to track current block pc + step count:
   ```js
   ex.runFrom(entry, 'adl', {
     maxSteps: 200000,
     maxLoopIterations: 5000,
     onBlock: (pc, mode) => { currentBlockPc = pc; stepCount++; }
   });
   ```

4. Candidate entry points to probe — run each and log CALC reads:
   - 0x078419 (Y=/STAT PLOT editor — Phase 36)
   - 0x081670 (STAT/MATRIX editor — Phase 31)
   - 0x0296dd (MODE screen — Phase 34)
   - 0x04e135 (CATALOG — Phase 48)
   - 0x04e1d0 (CATALOG entry)
   - 0x09ebf6 (ABOUT)
   - 0x0b9c64 (Transformation Graphing)
   - 0x074817 (Inequality Graphing)
   - 0x028944 (TEST MODE)

5. Also try: run from 0x000000 z80 boot for the full 8804 steps WITH the read8 wrapper, to see if ANY CALC string is read during boot.

6. For each entry, report:
   - Number of CALC-area reads (if any)
   - First 10 reads with: addr, value, blockPc_at_read, step_at_read
   - The set of unique blockPc values during CALC reads — these are the code locations that read the CALC strings

7. If any candidate produces CALC reads, its corresponding blockPc values are the CALC MENU RENDERER. Print them with a "CANDIDATE RENDERER" label.

8. ALSO probe by running `ex.runFrom(0x055e8b, 'adl', ...)` directly (the "CALC ZERO" string as entry) — bad idea but shows if the string byte is treated as code. And try running from the last block PC seen right before a CALC string read.

OUTPUT:
- TI-84_Plus_CE/probe-phase54-calc-trace.mjs
- Final stdout summary:
  - Entry point | CALC reads count | First read addr | First block PC at read
  - List of candidate renderer blockPc addresses
  - Verdict: Did we find the CALC menu renderer? If yes, which address.

DO NOT:

- Don't re-transpile
