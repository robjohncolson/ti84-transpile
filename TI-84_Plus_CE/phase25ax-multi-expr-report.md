# Phase 25AX - Multi-Expression Direct-Eval Probe

This report file is generated by `probe-phase25ax-multi-expr.mjs` when the probe is executed. The checked-in version documents the intended harness and acceptance rules; the probe overwrites this file with observed results.

## Harness

- Fresh runtime per expression: new `Uint8Array(0x1000000)`, fresh executor, fresh peripheral bus, and a full cold boot.
- Pipeline per case: `BOOT -> KERNEL_INIT -> POST_INIT -> MEM_INIT -> token seed -> ParseInp -> read OP1`.
- The probe copies `coldBoot`, `prepareCallState`, `seedErrFrame`, and `runCall` from `probe-phase25ag-store-recall-roundtrip.mjs`.
- Success criteria: return to `0x7FFFFE`, decode OP1 within `1e-6` of the expected value, and accept `errNo` values `0x00` or `0x8D`.

## Console Output

The probe prints one line per expression in the form:

```text
PASS 2+3 -> OP1=5 expected=5 errNo=0x8d finalPc=0x7ffffe
```
