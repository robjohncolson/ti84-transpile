## Conclusions

1. **0xD02ACC is zeroed during boot, not 0xFF**: Boot writes 0x00 to the entire
   range via bulk LDIR. The prior hypothesis that it contains 0xFF is incorrect.
2. **ROM references found**: 248 direct references in ROM.
3. **Seeding did NOT produce visible pixels**: Stage 2 writes only white pixels
   regardless of color byte values. The white-on-white issue is NOT a color-value
   problem -- the icon rendering pipeline itself needs investigation.
