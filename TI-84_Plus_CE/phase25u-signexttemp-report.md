## 0x04C9A8 - signExtTemp scratch helper

```text
0x04c9a8: f5                push af
0x04c9a9: 22 d7 2a d0       ld (0xd02ad7), hl ; signExtTemp
0x04c9ad: 3a d9 2a d0       ld a, (0xd02ad9) ; signExtFlag / high byte of signExtTemp
0x04c9b1: cb 3f             srl a
0x04c9b3: 32 d9 2a d0       ld (0xd02ad9), a ; signExtFlag / high byte of signExtTemp
0x04c9b7: cb 1c             rr h
0x04c9b9: cb 1d             rr l
0x04c9bb: 40 22 d7 2a       sis ld (0x2ad7), hl ; short store through MBASE, normally 0xd02ad7 in OS RAM
0x04c9bf: 2a d7 2a d0       ld hl, (0xd02ad7) ; signExtTemp
0x04c9c3: f1                pop af
0x04c9c4: c9                ret
```

- Stores the incoming 24-bit `HL` value to `0xd02ad7..0xd02ad9`.
- Reads `0xd02ad9` into `A`, shifts it right, writes it back, then rotates `H` and `L` right through carry.
- The `sis ld (0x2ad7), hl` at `0x04c9bb` rewrites the low 16 bits through the current `MBASE`; in OS RAM that normally aliases `0xd02ad7`.
- Reloads `HL` from `signExtTemp` and returns, so the helper is "store + 24-bit right shift + reload", not just "store and return".

## Pipeline summary

Generic signExtTemp helper family:

```text
HL -> 0x04C9A8 -> signExtTemp/signExtFlag scratch
```

Concrete InsertMem / AdjSymPtrs path:

```text
InsertMem(BC bytes)
  -> 0x04C990   BC = -BC
  -> 0x0827CA   ld (0xD02AD7), BC
  -> 0x082823   for each pointer slot: compare against DE, then add signExtTemp on the carry path
```

## Exact contract

| item | contract |
| --- | --- |
| `signExtTemp` (`0xD02AD7..0xD02AD9`) | 24-bit scratch/delta storage. `0x04C9A8` stores `HL` there and shifts the full 24-bit value right by one. `0x0827CA` stores `BC` there directly. `0x082823` reads it back as the adjustment delta. |
| `signExtFlag` (`0xD02AD9`) | In these routines it behaves as the high/extension byte of the same 24-bit scratch, not as a standalone branch flag. `0x04C9A8` uses it as the byte shifted by `srl a`, feeding carry into `rr h` and `rr l`. |
| `0x082823` predicate | The literal unsigned test is `ptr - DE`. `ret nc` skips the update when `ptr >= DE`; the delta is applied only on the borrow/carry path (`ptr < DE`). Because the InsertMem flow stores a negative delta, the add behaves like subtracting the insertion size. |

## Conclusion

`0x04C9A8` documents how the OS stores and bit-shifts the `signExtTemp` scratch pair, while the concrete InsertMem pointer-adjust path uses `0x0827CA` to copy the already-negated `BC` into `signExtTemp` and `0x082823` to add that negative delta to each slot.
