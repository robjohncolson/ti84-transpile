# Task: IPoint pixel write flags — find which IY flag combination enables actual VRAM writes

## Context
This is a TI-84 Plus CE ROM transpilation project. The ROM is lifted to JavaScript. We run the lifted code via `createExecutor()` from `cpu-runtime.js`.

IPoint (0x07B451) computes correct VRAM addresses (confirmed session 148) but writes 0 bytes to VRAM. The pixel store is gated by IY flag checks:
- BIT 0,(IY+0x3C) at 0xD000BC
- BIT 1,(IY+0x02) at 0xD00082
- BIT 2,(IY+0x2B) at 0xD000AB

Session 148 confirmed: C register = pixel Y, DE register = pixel X. VRAM formula: `addr = 0xD40000 + (239-C)*320*2 + DE*2`.

**Key known facts:**
- Session 129: IPoint with drawMode=1 DOES write pixels when pixWideP/pixWide_m_2 are seeded
- Session 144: NZ path (BIT 2 IY+43 SET) writes to LCD VRAM via `LD (BC),A` at 0x07B682
- Session 145: IY+43 SET + IY+74 CLEAR gives 10/10 IPoint OK across 320px, 10 LCD VRAM bytes written
- Session 143: IY+43 SET + IY+74 SET gives 0 pixel writes
- Session 148: Even with correct C/DE coordinates, 0 VRAM bytes written (wrong IY flag combo)

## Your task
Build `TI-84_Plus_CE/probe-phase149-ipoint-flags.mjs` that systematically tests ALL combinations of the 3 known IY flag gates to find which enables actual VRAM pixel writes.

### Deliverables
1. A probe file at `TI-84_Plus_CE/probe-phase149-ipoint-flags.mjs`
2. The probe must:
   a. Use the same boilerplate pattern as `TI-84_Plus_CE/probe-phase148-ipoint-coords.mjs` (read that file for exact import/setup)
   b. Run MEM_INIT first (entry 0x09DEE0, return sentinel 0x7FFFF6, budget 100K steps)
   c. Test a single pixel at center screen: C=120 (pixelY), DE=160 (pixelX), drawMode A=1
   d. Seed: pixWideP (0xD014FE) = 320, pixWide_m_2 (0xD01501) = 238, pen color (0xD026AE) = 0x10, hookflags3 (0xD000B5) bit 7 = 0
   e. Test ALL 8 combinations of the 3 flags:
      - BIT 0 of IY+0x3C (0xD000BC): SET vs CLEAR
      - BIT 1 of IY+0x02 (0xD00082): SET vs CLEAR
      - BIT 2 of IY+0x2B (0xD000AB): SET vs CLEAR
   f. For each combination:
      - Fresh memory state (re-run MEM_INIT or re-zero VRAM area)
      - Set the 3 flags as specified
      - Also set IY+74 (0xD000CA) BIT 2 to both SET and CLEAR (making 16 total tests)
      - Call IPoint at 0x07B451
      - After return, scan LCD VRAM (0xD40000, 153600 bytes) for any non-zero bytes
      - Also scan plotSScreen (0xD09466, 76800 bytes) for meaningful changes
      - Report: which flag combo produced VRAM writes, how many bytes, at what addresses
   g. Print a summary table of all 16 combinations and their results

### Key addresses
- IPoint entry: 0x07B451
- OP1: 0xD005F8, OP2: 0xD00603
- MEM_INIT: 0x09DEE0
- pixWideP: 0xD014FE (write 320 as 24-bit LE)
- pixWide_m_2: 0xD01501 (write 238 as 24-bit LE)
- Pen color: 0xD026AE (write 0x10)
- hookflags3: 0xD000B5 (clear bit 7)
- IY+0x3C: 0xD000BC (flag bit 0)
- IY+0x02: 0xD00082 (flag bit 1)  
- IY+0x2B: 0xD000AB (flag bit 2)
- IY+0x4A: 0xD000CA (flag bit 2, "IY+74")
- LCD VRAM base: 0xD40000 (153600 bytes for 320x240 16bpp)
- plotSScreen: 0xD09466 (76800 bytes)
- FAKE_RET: 0x7FFFFE
- errSP: 0xD008E0, errNo: 0xD008DF
- FPSbase: 0xD0258A, FPS: 0xD0258D, OPBase: 0xD02590, OPS: 0xD02593

### Setup pattern (from probe-phase148)
```javascript
// After MEM_INIT, seed allocator:
write24(mem, OPBASE_ADDR, EMPTY_VAT_ADDR);
write24(mem, OPS_ADDR, EMPTY_VAT_ADDR);
write24(mem, PTEMP_ADDR, EMPTY_VAT_ADDR);
write24(mem, PROGPTR_ADDR, EMPTY_VAT_ADDR);
write24(mem, FPSBASE_ADDR, USERMEM_ADDR);
write24(mem, FPS_ADDR, USERMEM_ADDR);
write24(mem, NEWDATA_PTR_ADDR, USERMEM_ADDR);

// Set IPoint registers:
cpu.registersF[0] = 0; // A = drawMode = 1
cpu.registers[0] = 1;  // A=1 (check register layout in probe-phase148)
// C = pixelY, DE = pixelX — check probe-phase148 for exact register mapping
```
