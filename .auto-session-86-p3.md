# Task: Trace ParseInp's Last 50 PCs Before FAKE_RET — Where Does 5.0 Go?

## Context
In the 918-step ParseInp run (no pre-created variable, OP1 starts as all zeros):
- ParseInp computes OP1=5.0 (BCD: [00 80 50 00 00 00 00 00 00])
- errNo=0x8D (ErrUndefined) — the parser tried to store/look up the result and failed
- OPBase moved -9 (one var allocated), OPS moved +3
- The run terminates by error longjmp to FAKE_RET

**Key question**: What is the parser trying to do in its LAST ~100 steps? Specifically:
- Does it call StoAns (token 0x3B, stores to Ans variable)?
- Does it call ChkFindSym or FindSym to look up a variable?
- What address does the ErrUndefined (0x8D) originate from?
- What ROM routine writes 0x8D to errNo (0xD008DF)?

## Your Task
Create a probe `TI-84_Plus_CE/probe-phase25aa-parseinp-store-trace.mjs` that:

1. Boots + MEM_INIT (no CreateReal — we want the 918-step/errNo=0x8D path)
2. Clears OP1 to all zeros
3. Sets up tokens "2+3" at 0xD00800 via begPC/curPC/endPC
4. Calls ParseInp (0x099914) with error frame
5. Captures DETAILED trace of the ENTIRE run:
   - Every PC visited (store ALL of them in an array — 918 entries is small)
   - At every step, also record: the current value of errNo (0xD008DF), and OP1 byte 0 (0xD005F8)
   - When errNo CHANGES from its previous value, log the step number and PC
   - When OP1[0] CHANGES from its previous value, log the step number and PC
6. After the run, analyze the trace to identify:
   - The exact step and PC where errNo becomes 0x8D
   - The PCs visited in the 50 steps BEFORE errNo changes (the "about to error" sequence)
   - The PCs visited in the 20 steps AFTER errNo changes (the "error recovery" sequence)
   - Whether any of these PCs match known routines:
     - 0x061D3E = ErrMemory JP target
     - 0x061DB2 = JError (longjmp entry)
     - 0x061DD1 = PopErrorHandler
     - 0x061E27 = normal-return cleanup stub
     - 0x08383D = ChkFindSym
     - 0x0846EA = FindSym
     - 0x07C77F = FPAdd
     - 0x099914 = ParseInp
7. Write report to `TI-84_Plus_CE/phase25aa-store-trace-report.md`
