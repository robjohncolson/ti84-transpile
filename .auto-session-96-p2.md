- Chain2: 0x06C8B4 — NEVER reached in session 95's 300K-step run

Session 94's static BFS assumed CoorMon→0x06CE73→0x06C8B4→0x0ACC4C→ParseInp, but the runtime trace shows Chain1 fires but Chain2 does not. Something in Chain1's function body prevents reaching Chain2.

1. Reads ROM bytes starting at 0x06CE73 and disassembles the function body using the eZ80 decoder

3. For each instruction, output: `address (hex), raw bytes (hex), mnemonic, operands`
4. Identify ALL branch targets (JR/JP/CALL destinations) and flag any conditional branches
5. Specifically look for:
   - Any reference to 0x06C8B4 (Chain2) — is it a CALL target? A JP target? A conditional branch target?

   - Any reads of RAM addresses (0xD0xxxx) that could be gate conditions

7. If 0x06C8B4 is NOT found in the disassembly, trace where Chain1 actually goes instead (follow the CALL/JP chain)
