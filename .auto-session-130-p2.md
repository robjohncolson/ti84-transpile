# Task: Graph subsystem — render a line via ILine (IPoint validated)

## Context
You are working on a TI-84 Plus CE ROM transpilation project at `C:/Users/rober/Downloads/Projects/school/ti84-transpile`. Session 129 proved IPoint works: calling IPoint at 0x07B451 with drawMode=1 and properly seeded RAM writes pixels to plotSScreen. Key RAM seeding:
- pixWideP at 0xD014FE = width (e.g., 320 = 0x0140)
- pixWide_m_2 at 0xD01501 = height (e.g., 240 = 0x00F0) (actually pixel count minus 2, but 240 works)
- Pen color at 0xD026AE (2 bytes, e.g., 0x0000 for black or 0xF800 for red)
- hookflags3 at 0xD000B5 bit 7 = 0
- drawMode in A register: 1 = normal draw, 0 = ERASE (DEC A internally), 2 = XOR
- plotSScreen buffer at 0xD09466 (21945 bytes)

Session 129 also found X bounds use 8-bit CP L comparison, so coordinates must have low byte < low byte of pixWideP (e.g., with pixWideP=320=0x0140, L=0x40, so X must be < 64 for the 8-bit check to pass).

ILine at 0x07B245: A=draw mode, mode byte 1 stored at 0xD02AD4, saves BC/DE to 0xD022D1/D2, computes deltas, calls division at 0x04C979, iteratively calls IPoint for each pixel.

   - A = 1 (drawMode = normal draw)

5. **If ILine fails**, fall back to calling IPoint in a loop (10 calls with different coordinates) to at least demonstrate multi-pixel rendering works. This gives us a baseline even if ILine's register convention isn't right.
