# Auto-Session 164 P3: Trace OP2 mutations steps 332-535 — where does OP2=12 come from?

## Context

You are working on a TI-84 Plus CE ROM transpilation project. The ROM is lifted from eZ80 machine code to JavaScript.

We are investigating the FP gcd algorithm at 0x068D3D for gcd(12,8). Session 163 found that InvSub at step 535 (in the outer-loop region) has OP2=12. This is surprising because:
- In iteration 1's body (step ~190), InvSub had OP2=1200 (from OP4→OP2 copy of saved OP1)
- But in the outer region (step 535), OP2=12 — the ORIGINAL OP1 value before gcd_helper bumped the exponent

**Critical question**: Where did OP2=12 come from? Was it popped from FPS? Was it restored via an OP register copy? Was it computed?

Session 163 noted that OP5→OP2 fires at step 570 (AFTER InvSub at step 535), so that's not the source. The 12 must come from somewhere between step 332 (end of inner loop) and step 535 (InvSub entry).

Log any hit with the step number and which copy function fired.

### Part D: Summary
