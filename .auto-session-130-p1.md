# Task: Trace upstream FP op-code dispatch for min/max

## Context

Session 129 found that when evaluating min(3,7) or max(3,7), the FP (floating-point) dispatch table at ROM address 0x0686EF receives A=0x00 instead of the raw operation code (0xDA for min, 0xDB for max). The 16 comparison entries in the dispatch chain don't match 0x00 so execution falls through to the catch-all error handler at JP 0x06677B → JError → LCD busy-wait at 0x001221.

Meanwhile, gcd(12,8) with operation code 0xD5 DOES succeed (returns via FAKE_RET in 3386 steps, but with wrong result OP1=0.588003, expected 4). There's a missing block at 0x06859B that gcd hits.

The actual operation code (0xDA/0xDB/0xD5) sits on OPS (operator stack at 0xD02593). The upstream dispatcher reads OPS and converts to some category/index byte in A before jumping to 0x0686EF.
