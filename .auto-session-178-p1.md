# Session 178 P1 — Find who writes to 0xD02AD7 (callback table setup)

## Context

In the TI-84 Plus CE OS ROM transpilation project, we've traced the ISR event loop and discovered that address 0xD02AD7 holds a callback pointer used during interrupt dispatch. The handler at 0x03D184 stores HL → (0xD02AD7) as part of interrupt acknowledgment. We need to find what code in the OS init sequence writes to this address to set up the callback table.

## Task

Create a probe file `TI-84_Plus_CE/probe-phase178-callback-table-setup.mjs` that:

1. **Searches the ROM binary** for any 3-byte sequence that encodes the address 0xD02AD7 in little-endian (bytes: `D7 2A D0`). This will find LD instructions that reference this address (LD (0xD02AD7), HL or LD (0xD02AD7), A, etc.).

2. For each match found, **decode the instruction at that location** using the decoder (`./ez80-decoder.js`) to determine what's being written there and from which register.

3. **Also search for nearby addresses** in the callback table region: 0xD02AD4-0xD02ADF (the bytes D4-DF 2A D0) to find related callback table entries.

4. **Cross-reference with the block map** — check if each matched address is within a known transpiled block (import PRELIFTED_BLOCKS from ROM.transpiled.js).

5. Print a clear report: for each reference found, show the ROM address, the decoded instruction, whether it's in a known block, and what value it would write.
