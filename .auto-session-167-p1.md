# Task: Find the POP AF corresponding to PUSH AF at 0x068D84 in the gcd handler

## Context
You are working on a TI-84 Plus CE ROM transpilation project. The gcd handler at 0x068D3D has a bug where InvSub(OP1, OP1) = 0 instead of computing a proper modulus. Session 166 found that `PUSH AF` at 0x068D84 saves the Z flag (Z=1 if exponent < 0x82, Z=0 if >= 0x82). The corresponding `POP AF` has NOT been found yet — it must be in the gcd post-body exit code, NOT in called subroutines.

## What to do
Build a probe at `TI-84_Plus_CE/probe-phase167-popaf-search.mjs` that:

### Part A: Static ROM byte scan
Read raw ROM bytes from 0x068DA3 to 0x068EF0 (the gcd handler post-body and exit code region). Search for byte 0xF1 (POP AF opcode). Report each hit with its exact address and surrounding instruction context (2-3 bytes before and after). Disassemble the surrounding bytes to confirm each 0xF1 is actually a POP AF instruction (not a data byte or part of a multi-byte instruction).

### Part B: Dynamic trace of POP AF hits
Run gcd(12,8) with the standard setup (see setup pattern below). Add hooks at each 0xF1 address found in Part A. For each hit, log:
- The step number
- The stack pointer (SP) value
- The value popped (what was on the stack at SP before POP)
- Register A and F values after POP
- Whether this is inside a subroutine (compare SP with the SP at PUSH AF entry 0x068D84)

The gcd's OWN POP AF should have SP matching the SP right after the PUSH AF at 0x068D84. Subroutine-internal POPs will have deeper (lower) SP values.

### Part C: Also check for POP AF ALIASES
The Z80/eZ80 can effectively restore flags via:
- `EX AF,AF'` (opcode 0x08) — swaps A and F with shadow registers
- `LD A,(nn)` + `PUSH AF` + `POP AF` patterns
- `EX (SP),HL` + manipulation patterns

Scan 0x068DA3-0x068EF0 for 0x08 bytes too, and check if any EX AF,AF' is used to restore the saved flags.

## Setup pattern (copy this exactly)
```javascript
import { createCPU } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';
import fs from 'fs';

const rom = new Uint8Array(fs.readFileSync('./TI-84_Plus_CE/ROM.rom'));
const bus = createPeripheralBus({ timerInterrupt: false });
const cpu = createCPU(bus);

// Load ROM
for (let i = 0; i < rom.length; i++) cpu.mem[i] = rom[i];

// Load transpiled blocks
const txSrc = fs.readFileSync('./TI-84_Plus_CE/ROM.transpiled.js', 'utf8');
const BLOCKS = {};
new Function('BLOCKS', txSrc)(BLOCKS);

// Helper to read 9-byte OP register
function readOP(base) {
  return Array.from(cpu.mem.slice(base, base + 9));
}
function fmtOP(base) {
  return readOP(base).map(b => b.toString(16).padStart(2,'0')).join(' ');
}

// OP register addresses
const OP1 = 0xD005F8, OP2 = 0xD00603, OP3 = 0xD0060E, OP4 = 0xD00619, OP5 = 0xD00624, OP6 = 0xD0062F;

// Write a float value to an OP register (TI format)
function writeFloat(addr, value) {
  // Clear 9 bytes
  for (let i = 0; i < 9; i++) cpu.mem[addr + i] = 0;
  if (value === 0) { cpu.mem[addr] = 0; cpu.mem[addr+1] = 0x80; return; }
  const sign = value < 0 ? 0x80 : 0x00;
  value = Math.abs(value);
  const exp = Math.floor(Math.log10(value)) + 1;
  cpu.mem[addr] = sign;
  cpu.mem[addr + 1] = 0x80 + exp;
  let mantissa = value / Math.pow(10, exp);
  for (let i = 2; i < 9; i++) {
    mantissa *= 100;
    const byte = Math.floor(mantissa);
    cpu.mem[addr + i] = ((Math.floor(byte / 10) & 0xF) << 4) | (byte % 10);
    mantissa -= byte;
  }
}

// Standard gcd setup: gcd(12, 8)
// Set up stack
cpu.sp = 0xD1A880;
// Push return sentinel
cpu.sp -= 3;
cpu.mem[cpu.sp] = 0xFE; cpu.mem[cpu.sp+1] = 0xFF; cpu.mem[cpu.sp+2] = 0x7F;

// Write operands
writeFloat(OP1, 12);
writeFloat(OP2, 8);

// Push OP2 to FPS (gcd handler expects it)
const fpsBase = 0xD1AA00;
cpu.mem[0xD02589] = fpsBase & 0xFF;
cpu.mem[0xD0258A] = (fpsBase >> 8) & 0xFF;
cpu.mem[0xD0258B] = (fpsBase >> 16) & 0xFF;
for (let i = 0; i < 9; i++) cpu.mem[fpsBase + i] = cpu.mem[OP2 + i];
cpu.mem[0xD02589] = (fpsBase + 9) & 0xFF;
cpu.mem[0xD0258A] = ((fpsBase + 9) >> 8) & 0xFF;
cpu.mem[0xD0258B] = ((fpsBase + 9) >> 16) & 0xFF;

// Set IY
cpu.iy = 0xD00080;

// Start at gcd inner entry
cpu.pc = 0x068D61;
```
