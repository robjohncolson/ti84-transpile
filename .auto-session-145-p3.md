## Context
TI-84 Plus CE ROM transpilation project. IPoint (0x07B451) has two BIT 2 checks:
1. **0x07B793** — BIT 2,(IY+43) at IY+0x2B=0xD000AB: routes bounds check (Z=8-bit, NZ=24-bit)
2. **0x07B6A6** — BIT 2,(IY+74) at IY+0x4A=0xD000CA: routes pixel write (NZ=skip plotSScreen, only write 16bpp LCD VRAM; Z=write BOTH plotSScreen and LCD VRAM)

But we haven't verified the actual pixel data appears in VRAM, and we haven't tested the optimal flag combination: IY+43 bit 2 SET (24-bit bounds) + IY+74 bit 2 CLEAR (write to plotSScreen too).

## Key addresses
- IPoint: 0x07B451
- Bounds check BIT 2: 0x07B793 (IY+43 = 0xD000AB)
- Pixel write BIT 2: 0x07B6A6 (IY+74 = 0xD000CA)
- plotSScreen base: 0xD09466 (76,800 bytes for 320x240 monochrome)
- 16bpp LCD VRAM: 0xD40000 to 0xD65800
- Framebuffer addr register: 0xD02A8A (3 bytes, BC loaded from here)
- Pen color: 0xD026AE
- pixWideP: 0xD014FE (2 bytes, set to 320 = 0x0140)
- pixWide_m_2: 0xD01501 (2 bytes, set to 238 = 0x00EE)
- drawMode byte: 0xD02AC8 (A register on IPoint entry, 1=draw, 0=erase, 2=XOR)
- GraphPars body: 0x099874
- tX intercept: 0x07D1B4
- OP1: 0xD005F8
- IY base: 0xD00080

## Standard probe boilerplate
```js
import { readFileSync } from 'fs';
import { gunzipSync } from 'zlib';

const ROM_PATH = 'TI-84_Plus_CE/ROM.rom';
const TRANSPILED_PATH = 'TI-84_Plus_CE/ROM.transpiled.js';

let src;
try { src = readFileSync(TRANSPILED_PATH, 'utf8'); }
catch { src = gunzipSync(readFileSync(TRANSPILED_PATH + '.gz')).toString('utf8'); }

const { createExecutor } = await import('./TI-84_Plus_CE/cpu-runtime.js');
const { createPeripheralBus } = await import('./TI-84_Plus_CE/peripherals.js');

const mem = new Uint8Array(16 * 1024 * 1024);
const rom = readFileSync(ROM_PATH);
mem.set(rom, 0);

const peripherals = createPeripheralBus({ timerInterrupt: false });
const ex = createExecutor(mem, peripherals);
new Function('executor', src)(ex);
```
