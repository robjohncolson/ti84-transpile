# Auto-Session 91 P2: Find OS StartApp / App Context Initialization Path

## Context
We're transpiling a TI-84 Plus CE ROM into JavaScript. CoorMon (the OS main event loop at 0x08C331) dispatches keys to the active app via cxCurApp (0xD007E0), but it zeroes the entire cx context range (0xD007CA-0xD007E1) during execution. We need to find the OS routine that INITIALIZES these fields for the home screen app.

From ti84pceg.inc:
- NewContext JT slot at 0x02016C → impl at 0x08C79F
- NewContext0 JT slot at 0x020170 → impl at 0x08C7AD
- cxMain = 0xD007CA (3 bytes)
- cxPPutAway = 0xD007CD (3 bytes)
- cxPutAway = 0xD007D0 (3 bytes)
- cxReDisp = 0xD007D3 (3 bytes)
- cxErrorEP = 0xD007D6 (3 bytes)
- cxSizeWind = 0xD007D9 (3 bytes)
- cxPage = 0xD007DC (3 bytes)
- cxCurApp = 0xD007E0 (2 bytes)
- cxPrev = 0xD007E2 (23-byte shadow)
- kQuit (home screen) = 0x40
- appFlags at IY+0x0D (IY=0xD00080 → appFlagsLoc=0xD0008D)

## Task
Create a probe `TI-84_Plus_CE/probe-phase25af-newcontext-disasm.mjs` that:

1. **Disassembles NewContext (0x08C79F) and NewContext0 (0x08C7AD)** — read 128 bytes of ROM starting at each address, decode each eZ80 instruction, and print the disassembly. This tells us what these routines do (likely: push current cx onto cxPrev, then populate cxMain..cxCurApp from a table or from registers).

2. **Searches ROM for all writes to 0xD007E0 (cxCurApp)**:
   - Scan ROM bytes 0x000000-0x3FFFFF for the byte pattern that encodes `LD (0xD007E0), A` or `LD (0xD007E0), HL` or similar store instructions targeting 0xD007E0. In eZ80 ADL mode:
     - `LD (nn), A` = 0x32 followed by 3-byte little-endian address (E0 07 D0)
     - `LD (nn), HL` = 0x22 followed by 3-byte LE address (E0 07 D0)
     - The 0xED prefix variants for 16/24-bit stores
   - Report every ROM address where such a store occurs.

3. **Cross-references callers of NewContext/NewContext0**:
   - Search ROM for `CALL 0x08C79F` (CD 9F C7 08) and `CALL 0x08C7AD` (CD AD C7 08)
   - Also search for JT slot calls: `CALL 0x02016C` (CD 6C 01 02) and `CALL 0x020170` (CD 70 01 02)
   - Report every caller address found.

4. **Writes a report** to `TI-84_Plus_CE/phase25af-newcontext-disasm-report.md`

For the byte-pattern search: just scan the ROM buffer for the byte sequences. No need to boot the OS or run the executor for this — it's static ROM analysis.
