# Auto-Session 169 P2: _Intgr Call at 0x068D69 — Dynamic Trace Through Loop Top

## Context

This is a TI-84 Plus CE ROM transpilation project. We're investigating why `gcd(12,8)` produces E_Domain instead of 4.0.

The gcd loop top at 0x068D61-0x068D81 calls `_Intgr` at 0x068D69 (CALL 0x0685DF). This is the `_Intgr` function that presumably computes the integer part (floor). It fires BEFORE gcd_helper in the loop iteration. If _Intgr does more than floor() — e.g., modifies mantissa representation, sets flags, or has side effects on OP2 — it could affect the inner body's behavior.

Session 163 disassembled the loop top and identified this call, but never traced OP1 through _Intgr dynamically.
