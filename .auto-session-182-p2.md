# Auto-Session 182 P2 — Home Handler Disassembly: Find Key→BufInsert Routing

## Context
Session 181 corrected that 0x058D49 and 0x058BA3 are trivial flag-clearing routines (4 and 3 instructions), NOT key classifiers. The actual key routing inside the home handler that converts a digit key code into a BufInsert call is elsewhere.

Session 129 proved kbdKey=0x90 (k2) reaches the home handler at 0x058241, but stalls on MMIO busy-wait at 0x09EFDE. The home handler dispatch table starts around 0x058693. We need to find WHERE in the home handler a digit keypress (like k2=0x90) gets routed to BufInsert (0x05E2A0).

## Task
Build a diagnostic probe `TI-84_Plus_CE/probe-phase182-home-handler-disasm.mjs` that:

1. **Statically disassembles** the home handler starting at 0x0582B8 through 0x058700 (roughly). Use the pattern from `probe-phase181-fgkbd-scan.mjs` which does static disassembly by reading ROM bytes and decoding instructions.
2. **Specifically looks for**:
   - CALL or JP instructions targeting 0x05E2A0 (BufInsert) or nearby addresses
   - CALL or JP instructions targeting addresses in the 0x058xxx or 0x059xxx range (other home handler subroutines)
   - Table lookups — CP A,imm8 / JR Z / JP Z patterns that dispatch based on key codes
   - References to kbdKey (0xD0058C) or kbdScanCode (0xD0058E)
   - Any jump table structure (sequential JP or DW entries)
3. **Also disassembles** the dispatch table area at 0x058693-0x0586D0 (session 129 identified this as a key dispatch table area).
4. **Traces execution** from 0x0582B8 with A=0x90 (k2 digit key) for up to 500 steps, logging each block visited and any CALL/JP targets.
5. **Prints a structured report** including:
   - Full disassembly listing (address, hex bytes, mnemonic)
   - All CALL/JP targets found
   - Any references to BufInsert (0x05E2A0) or edit buffer addresses (0xD00A00 area)
   - Execution trace summary (blocks visited, where it stalls)

## Files to read for patterns
- `TI-84_Plus_CE/probe-phase181-fgkbd-scan.mjs` — static disassembly pattern (the best recent example)
- `TI-84_Plus_CE/probe-phase181-key-classify.mjs` — traces 0x058D49 and 0x058BA3

- Home handler body: 0x0582B8 (after initial setup)
- Dispatch table area: 0x058693-0x0586D0
- BufInsert: 0x05E2A0
- kbdKey: 0xD0058C
- kbdScanCode: 0xD0058E
