# Session 148 P1: FP gcd — add real pop seed (0x0828FC) + re-test gcd(12,8)

## Context
This is a TI-84 Plus CE ROM transpilation project. We lift eZ80 machine code to JavaScript.

Session 147 discovered that "FpPop" at 0x082957 is **misnamed** — it always PUSHES OP2→FPS. The real pop is at **0x0828FC**:
- `LD DE,0xD00603` (OP2 address)
- `CALL 0x082912` which does `LD BC,9; LD HL,(FPS); SBC HL,BC; LD (FPS),HL; RET` (FPS-=9)
- `JP 0x07F978` (9x LDI from FPS→OP2 = copy stack top to OP2)

Also found 0x082BCE: swap OP1 ↔ top FPS entry via 0x0829F2→0x07FD3A.

The gcd call sequence (mapped in session 147):
push OP2 (0x082957) → push OP1 (0x082961) → push OP1 → inline pop→OP1 → push OP1 → pop→OP2 (via 0x0828FC at step 1425)

gcd(12,8) currently fails with E_Domain (errNo=0x84) because the real pop path at 0x0828FC is a **missing block** in the transpiler.

## Task

### Part 1: Add transpiler seed for 0x0828FC

Edit `scripts/transpile-ti84-rom.mjs`. Find the seed entries near line 22139 where `0x082957` and `0x068D3D` are already seeded. Add two new seeds right after them:

```javascript
    { pc: 0x0828FC, mode: 'adl' },  // Real FP pop — copies FPS top → OP2, decrements FPS by 9
    { pc: 0x082912, mode: 'adl' },  // FPS-=9 helper — called by real pop at 0x0828FC
```

Also add 0x082BCE if not present:
```javascript
    { pc: 0x082BCE, mode: 'adl' },  // FP swap — exchanges OP1 with FPS top entry
```

### Part 2: Re-run transpiler

```bash
node scripts/transpile-ti84-rom.mjs
```

Record the new seed count, block count, and covered byte count from the output.
