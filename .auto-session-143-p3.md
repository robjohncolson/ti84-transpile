# Session 143 P3: Graph — Check LCD VRAM (0xD40000) for Pixels After BIT 2 Set

## Context

The TI-84 ROM transpilation project lifts eZ80 ROM to JS. Session 142 found that setting BIT 2 of IY+43 (at 0xD000AB) enables the full 24-bit bounds check in IPoint, allowing 160 successful IPoint calls (vs 132 baseline). However, those 160 calls write ZERO bytes to plotSScreen (0xD09466). The hypothesis is that the NZ path writes to LCD VRAM at 0xD40000 instead.

LCD VRAM for the TI-84 CE is 76,800 bytes at 0xD40000-0xD52BFF (320×240 pixels, 1 byte per pixel in 8bpp mode, or 2 bytes in 16bpp = 153,600 bytes at 0xD40000-0xD657FF).

## Your Task

Create a NEW probe file `TI-84_Plus_CE/probe-phase143-graph-vram.mjs` based on `probe-phase142-graph-fullwidth.mjs`. Modifications:

1. **Run the same graph pipeline as Test B** from session 142: BIT 2 of IY+43 set, pixWideP=320, pixWide_m_2=238. Use Y1=X equation, loop positive X values 0 to 10 in ~133 steps through GraphPars body (0x099874) with tX intercept at 0x07D1B4, then IPoint at 0x07B451.

2. **After the render loop, scan memory regions for pixel data**:
   - plotSScreen: 0xD09466, 76,800 bytes (existing check)
   - LCD VRAM (8bpp): 0xD40000 - 0xD52BFF (76,800 bytes)
   - LCD VRAM (16bpp): 0xD40000 - 0xD657FF (153,600 bytes)
   - Any other RAM: scan 0xD00000 - 0xD60000 in 4KB chunks, report non-zero byte counts per chunk (vs pre-render baseline — take a snapshot of these ranges BEFORE the render loop)

3. **For each region**, report:
   - Total non-zero bytes (after subtracting pre-render baseline)
   - First 20 non-zero byte addresses and values
   - Whether the pattern looks like pixel data (contiguous runs, expected color values)

4. **Also trace IPoint's pixel-write instruction**: Add a write-watchpoint on the range 0xD40000-0xD52BFF during the IPoint calls. If any writes happen there, log the PC that performed the write and the address written.

## Setup Pattern

The probe should follow the same pipeline as probe-phase142-graph-fullwidth.mjs:
1. Cold boot + MEM_INIT
2. CreateEqu for Y1 = tX (token 0x58)
3. Seed graph state (errSP, IY flags, pixWideP=320, pixWide_m_2=238, pen color, BIT 2 of IY+43)
4. Loop X values: for each X, write BCD to FPS at 0x07D1B4 intercept, run GraphPars body, read Y from OP1, convert to pixel coords, call IPoint with drawMode=1
5. After loop: scan all memory regions for pixel data

- If NO pixels found anywhere: report that IPoint returns successfully but doesn't actually write
