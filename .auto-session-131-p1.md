# Auto-Session 131 P1 — Find ROM writers to 0xD0060E (FP category byte)

## Context

This is a TI-84 Plus CE ROM transpilation project. The ROM (4MB eZ80 machine code) has been lifted into JavaScript. We run probes that execute transpiled ROM blocks to trace OS behavior.

The FP (floating-point) operation dispatch table at ROM address 0x0686EF receives the operation category in register A. A is loaded from RAM slot **0xD0060E** via `LD A,(0xD0060E)` at PC=0x0663AD, then masked with `AND 0x3F`. Currently A=0x00 at dispatch time because nothing in our pipeline populates 0xD0060E.

The raw FP operation code (e.g., 0xD5 for gcd, 0xDA for min) sits on the OPS stack, NOT in A. Something in the ROM's FP evaluation chain must read the op code from OPS and derive a category byte, storing it to 0xD0060E before reaching the dispatch table.

## Task

### Part A: ROM binary search for 0xD0060E writers

Search the ROM binary (`TI-84_Plus_CE/ROM.rom`, 4MB) for all instructions that write to address 0xD0060E. The eZ80 instruction `LD (nn),A` encodes as `32 0E 06 D0` (little-endian: low byte 0x0E, mid 0x06, high-but-actually-MBASE 0xD0). Also check for `LD (nn),HL` = `22 0E 06 D0` and any 0xED-prefixed stores.

Write a probe at `TI-84_Plus_CE/probe-phase131-fp-category-writers.mjs` that:
1. Reads ROM.rom as a buffer
2. Scans for byte pattern `32 0E 06 D0` (LD (0xD0060E),A) — report all ROM offsets
3. Scans for `22 0E 06 D0` (LD (0xD0060E),HL) — report all ROM offsets  
4. For each found writer, disassemble 10 instructions before and 5 after using the ez80-decoder
5. Check if each writer address is in BLOCKS (transpiled) — if not, note it as a potential missing seed
6. Print a summary: writer addresses, surrounding context, coverage status

### Part B: Add seed at 0x06859B and test

Also in the same probe:
7. Check if 0x06859B is in BLOCKS. If not, add it as a seed by modifying the transpiler seed list.
8. The block at 0x06859B should be: `LD A,0x28; CALL 0x0689DE; RET` (part of FP handler table at 0x068580-0x0685AA)
9. Disassemble the handler table region 0x068580-0x0685AA to document all 6 entries (each loads A with a different value 0x20-0x2A before CALL 0x0689DE)
