# Auto-Session 111 P2: Enumerate Parser Flags Against PostInitState

## Context

This is a TI-84 Plus CE ROM transpilation project. We run lifted ROM code (eZ80 machine code transpiled to JavaScript) via `cpu-runtime.js` + `ROM.transpiled.js`.

ParseInp at 0x099914 evaluates tokenized expressions. Our probes work with zeroed IY flags (from MEM_INIT cold boot). But the OS normally has various flags set from prior CoorMon iterations. We need to know which parser-related flags are set/clear after MEM_INIT so we can document deterministic seed state.

## Task

Build a probe `TI-84_Plus_CE/probe-phase25az-parser-flags.mjs` that:

1. Sets up a fresh runtime (createExecutorFromFile, cold boot, kernel init, MEM_INIT — same pattern as other probes)
2. After MEM_INIT completes, reads and prints ALL of these IY-relative flag bytes:

| Name | Address | IY offset | Key bits |
|------|---------|-----------|----------|
| ParsFlag | IY+0x44 = 0xD000C4 | +0x44 | bit 5 = checked by 0x058212 |
| ParsFlag2 | IY+0x45 = 0xD000C5 | +0x45 | bit 0 = numOP1 ("result in OP1") |
| fmtFlags | IY+0x4A = 0xD000CA | +0x4A | format flags |
| numMode | IY+0x4B = 0xD000CB | +0x4B | number mode (0=normal, 1=sci, 2=eng) |
| fmtOverride | IY+0x4C = 0xD000CC | +0x4C | format override |
| onFlags | IY+0x09 = 0xD00089 | +0x09 | bit 3 = parseInput |
| graphFlags2 | IY+0x49 = 0xD000C9 | +0x49 | bit 4 = splitOverride |
| newDispF | IY+0x44 = 0xD000C4 | +0x44 | (same byte as ParsFlag) bit 6 = allowProgTokens |
| hookflags4 | IY+0x36 = 0xD000B6 | +0x36 | bit 5 = parserHookActive |
| statFlags | IY+0x09 = 0xD00089 | +0x09 | bit 6 = statsValid |
| progExecuting | IY+0x08 = 0xD00088 | +0x08 | (check all bits) |
| trigFlags | IY+0x00 = 0xD00080 | +0x00 | bit 2 = radian mode |
| IY+0x01 | 0xD00081 | +0x01 | bit 2 = dirty flag |
| IY+0x05 | 0xD00085 | +0x05 | bit 7 = scroll flag |
| IY+0x0C | 0xD0008C | +0x0C | bits 2,6,7 = display flags |
| IY+0x12 | 0xD00092 | +0x12 | bit 2 = checked in JError |
| IY+0x24 | 0xD000A4 | +0x24 | bit 4 = checked at 0x0586CC |
| IY+0x34 (0xB4) | 0xD000B4 | +0x34 | bit 4 = checked at 0x0586CC (IY+52) |
| IY+0x47 | 0xD000C7 | +0x47 | bit 7 = checked in JError |
| IY+0x49 | 0xD000C9 | +0x49 | bit 1 = checked by 0x049B12 |

3. For each byte, print:
   - The hex value
   - Each named bit's state (0 or 1)
   - Whether that bit value is expected/safe for direct-ParseInp usage

4. Also read and print the full IY flag block as a hex dump: 0xD00080 through 0xD000CF (80 bytes covering IY+0x00 through IY+0x4F)

5. Then run ParseInp("2+3") and re-read the same flag bytes AFTER ParseInp returns. Print a DIFF showing which bytes changed during ParseInp execution.

### Reference code pattern

Use the same executor setup as `TI-84_Plus_CE/probe-phase25ay-rclvar-vat-dump.mjs`:
```js
import { createPeripheralBus } from './peripherals.js';
// ... createExecutorFromFile pattern
// Cold boot: callBlock(0x000000) with maxSteps 50_000_000
// Kernel init: cpu.ix = 0xD1A860; callBlock(0x0586DA) 
// MEM_INIT: callBlock(0x09DEE0)
```

IY is always 0xD00080 after init. So IY+offset = 0xD00080 + offset.

### Success criteria
- Probe runs without errors
- All flag bytes printed with named bit breakdowns
- Before/after diff for ParseInp clearly shows which flags change
- PASS if all critical parser flags (ParsFlag, ParsFlag2, fmtFlags, numMode, hookflags4) are zero after MEM_INIT (our probes assume zero state)
