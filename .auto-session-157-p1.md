# Auto-Session 157 P1: Dense OP1/OP2 trace at 0x07CA48 entry during gcd(12,8)

## Context
This is a TI-84 Plus CE ROM transpilation project. We're investigating a gcd(12,8) bug where E_Domain is raised instead of returning 4. The gcd handler enters FP computation via address 0x07CA48 (a normalization pre-loop that shifts mantissa left + decrements exponent until exp<0x80). Session 156 proved FPDiv impl at 0x07CAB9 is correct (FPDiv(1200,1200)=1.0), so the bug is in how gcd reaches FP computation through 0x07CA48 rather than through FPDiv directly.

## Task
Build a probe `TI-84_Plus_CE/probe-phase157-gcd-07ca48-trace.mjs` that:

1. Sets up gcd(12,8) the same way as `probe-phase156-op3-track.mjs` (use it as your template — read it first).
2. Instruments execution to log OP1 (9 bytes at 0xD005F8) and OP2 (9 bytes at 0xD00603) at EVERY block entry from step 1 through step 250.
3. Specifically watches for when PC enters the range 0x07CA48-0x07CAB8 (the normalization pre-loop / gap before FPDiv impl). Log the EXACT step, OP1, OP2, and FPS pointer (3 bytes at 0xD0258D) when this happens.
4. Also log every time PC is in range 0x07C740-0x07C7FF (the OP1toOP2 / FPAdd / FPSub area that session 155 said leads to 0x07CA48).
5. Print a summary: which blocks were visited in the 0x07CA48-0x07CAB8 range, what OP1/OP2 values were at entry, and what they were after the normalization loop exits.

## Key addresses (from sessions 153-156)
- GCD handler: 0x068D3D
- GCD category byte: 0x28 (set at 0xD0060E before calling 0x068D3D)
- FpPop: 0x082957 (pops FPS into OP register)
- OP1: 0xD005F8 (9 bytes), OP2: 0xD00603 (9 bytes)
- OP3: 0xD0060E, OP4: 0xD00619, OP5: 0xD00624 (11-byte spacing per ti84pceg.inc)
- FPS pointer: 0xD0258D (3 bytes)
- errNo: 0xD008DF
- errSP: 0xD008E0
- FAKE_RET sentinel: 0x7FFFFE
- Normalization pre-loop: 0x07CA48
- FPDiv impl: 0x07CAB9
- OP1toOP2: 0x07C74B (session 155 said this leads to 0x07CA48)
- Const loader: 0x07FA86 (loads 1.0 into OP2)

## Setup pattern (from probe-phase156-op3-track.mjs)
```
1. Load ROM binary and ROM.transpiled.js
2. Create 16MB memory, createExecutor with timerInterrupt:false
3. Cold boot → kernel init → MEM_INIT (0x09DEE0 until PC hits FAKE_RET range)
4. Set up FPS/OPS/OPBase etc. to post-MEM_INIT values
5. Load OP1=12.0 BCD [00 81 12 00 00 00 00 00 00]
6. Load OP2=8.0 BCD [00 81 80 00 00 00 00 00 00]
7. Push OP2 to FPS (9-byte copy, advance FPS by 9)
8. Set FP category byte 0xD0060E = 0x28 (gcd)
9. Set errSP, push error frame
10. Execute from GCD_DIRECT_ADDR (0x068D3D) until FAKE_RET or step limit
```
