# Task: Shadow Register Trace Through gcd(12,8)

## Context
We're investigating a systemic gcd bug in the TI-84 Plus CE ROM transpiler. Every gcd(a,b) call produces E_Domain instead of the correct result. All individual components have been verified correct in isolation. The bug may be in shadow register state — EX AF,AF' or EXX could swap in wrong values. Shadow registers are accessible via `cpu._a2`, `cpu._f2`, `cpu._b2`, `cpu._c2`, `cpu._d2`, `cpu._e2`, `cpu._h2`, `cpu._l2` (discovered in session 166).

## What to build
Create `TI-84_Plus_CE/probe-phase170-shadow-reg-trace.mjs` — a probe that traces shadow registers (AF', BC', DE', HL') at every block boundary during gcd(12,8), and specifically logs every EX AF,AF' and EXX instruction.

## Reference pattern
Copy the FULL boilerplate from `TI-84_Plus_CE/probe-phase169-gcd-helper-disasm.mjs`:
- Same imports (fs, path, url, cpu-runtime, peripherals)
- Same constants (MEM_SIZE, BOOT_ENTRY, KERNEL_INIT_ENTRY, POST_INIT_ENTRY, STACK_RESET_TOP, OP1/OP2 addresses, GCD_ENTRY=0x068D3D, etc.)
- Same `createRuntime()` function
- Same `coldBoot()` function
- Same `prepareCallState()` function
- Same hex/BCD helpers

## Probe logic

1. Cold boot the system (same as all phase 165-169 probes)
2. Set up gcd(12,8): load OP1=12 (bytes: [0x00, 0x81, 0x12, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]), OP2=8 (bytes: [0x00, 0x80, 0x80, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]), set mem[0xD0060E]=0x28

3. Run gcd with step-by-step tracing. Use a loop calling the executor with maxSteps=1 repeatedly, logging after each step. Track shadow registers at each step:
   - `cpu._a2` (A')
   - `cpu._f2` (F')
   - `cpu._b2` / `cpu._c2` (BC')
   - `cpu._d2` / `cpu._e2` (DE')
   - `cpu._h2` / `cpu._l2` (HL')

4. Detect shadow register CHANGES: compare current values to previous step. When any shadow register changes, log:
   - Step number and PC
   - Which shadow register(s) changed
   - Old value → new value
   - This indicates an EX AF,AF' or EXX instruction was executed

5. After execution, print:
   - Total shadow register changes (count of EX AF,AF' and EXX)
   - Full list of changes with step/PC
   - Initial and final shadow register state
   - Error code, final OP1/OP2
   - Whether any shadow register swap brought in a value that was subsequently used in a conditional jump

## Key detail
The cpu object in cpu-runtime.js exposes shadow registers as:
- `cpu._a2`, `cpu._f2` for AF'
- `cpu._b2`, `cpu._c2` for BC'  
- `cpu._d2`, `cpu._e2` for DE'
- `cpu._h2`, `cpu._l2` for HL'

If these properties don't exist, try `cpu.a2`, `cpu.f2`, etc. Print what properties are available if neither works.

## Deliverable
A single file `TI-84_Plus_CE/probe-phase170-shadow-reg-trace.mjs` that runs with `node TI-84_Plus_CE/probe-phase170-shadow-reg-trace.mjs` and prints the complete shadow register trace.
