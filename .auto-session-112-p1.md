# Auto-Session 112 P1: Integrate Approach A round-trip into browser-shell direct-eval

## Goal
Update `TI-84_Plus_CE/browser-shell.html`'s `runDirectEval()` function to include the full Approach A round-trip after ParseInp, so that the Ans variable is correctly stored and available for future evaluations.

## What to change

In `browser-shell.html`, the `runDirectEval()` function currently:
1. Cold boots, MEM_INITs, writes tokens, runs ParseInp
2. After ParseInp returns, reads OP1, resets FPS/OPS to MEM_INIT values, returns the result

**Problem**: It does NOT create the Ans variable or store the result into it. The Approach A round-trip (validated in `probe-phase25az-rclvar-ops-fix.mjs`) proves the correct pipeline is:

1. Cold boot + MEM_INIT (already done)
2. CreateReal("Ans") to pre-allocate the Ans VAT entry (NEW)
3. Save post-CreateReal FPS/FPSbase values (NEW)
4. ParseInp (already done) 
5. Save OP1 result (already done as `val`)
6. Reset OPS to post-CreateReal OPBase, reset FPS/FPSbase to post-CreateReal values (MODIFY existing reset)
7. Re-CreateReal("Ans") to rebuild the VAT entry destroyed by OPS overflow (NEW)
8. Write saved OP1 BCD into the new Ans data slot (DE from re-CreateReal) (NEW)
9. Return result (already done)

### Helper function to run CreateReal (add near `evalWrite24`/`evalRead24`):

```javascript
function evalRunCreateReal(ex, c, mem) {
  // Set OP1 to Ans name
  for (let i = 0; i < 9; i++) mem[0xD005F8 + i] = EVAL_ANS_OP1[i];
  
  // Prepare call state
  c.halted = false; c.iff1 = 0; c.iff2 = 0;
  c.madl = 1; c.mbase = 0xD0; c._iy = 0xD00080; c.f = 0x40; c._ix = 0xD1A860;
  c.sp = SCREEN_STACK_TOP - 12;
  mem.fill(0xFF, c.sp, c.sp + 12);
  
  // Seed error frame
  c.sp -= 3;
  evalWrite24(mem, c.sp, EVAL_CREATEREAL_RET);
  const errBase = (c.sp - 6) & 0xFFFFFF;
  evalWrite24(mem, errBase, EVAL_ERR_CATCH);
  evalWrite24(mem, errBase + 3, 0);
  evalWrite24(mem, EVAL_ERRSP_ADDR, errBase);
  mem[EVAL_ERRNO_ADDR] = 0x00;
  
  // A=0x00 (Real type), HL=9 (data size)
  c.a = 0x00;
  c._hl = 0x000009;
  
  let returned = false;
  let errCaught = false;
  try {
    ex.runFrom(EVAL_CREATEREAL_ENTRY, 'adl', {
      maxSteps: 50000,
      maxLoopIterations: 8192,
      onBlock(pc) {
        const norm = pc & 0xFFFFFF;
        if (norm === EVAL_CREATEREAL_RET) throw new Error('__CR_RET__');
        if (norm === EVAL_ERR_CATCH) throw new Error('__CR_ERR__');
      },
      onMissingBlock(pc) {
        const norm = pc & 0xFFFFFF;
        if (norm === EVAL_CREATEREAL_RET) throw new Error('__CR_RET__');
        if (norm === EVAL_ERR_CATCH) throw new Error('__CR_ERR__');
      },
    });
  } catch (e) {
    if (e?.message === '__CR_RET__') returned = true;
    else if (e?.message === '__CR_ERR__') errCaught = true;
    else throw e;
  }
  
  // DE holds the data slot address after CreateReal
  const deAddr = c.de & 0xFFFFFF;
  return { returned, errCaught, deAddr };
}
```
