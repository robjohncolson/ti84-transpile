For tMean(0x21) the address was 0xD012A8. So index = (0xD012A8 - 0xD0117F) / 9 = 0x129/9 = 33. tMean=token 0x21 maps to index 33.

Actually, it's simpler to just seed 42.0 at a wide range of the stat-var table (from 0xD0117F to 0xD0117F + 50*9 = 0xD01361) and check what OP1 comes back as for each token. If the JT slot reads from the table correctly, OP1 will be 42.0.

### Approach
```javascript
// Seed the ENTIRE stat-var table with 42.0 at every 9-byte slot
const STAT_BASE = 0xD0117F;
const BCD_42 = [0x00, 0x81, 0x42, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00];
for (let i = 0; i < 50; i++) {
  const addr = STAT_BASE + i * 9;
  for (let j = 0; j < 9; j++) mem[addr + j] = BCD_42[j];
}

// Set statsValid
mem[0xD00089] |= 0x40; // bit 6

// For each token, re-run MEM_INIT, re-seed, set A=token, call 0x0204F0
```

Actually, since MEM_INIT zeroes the stat table, you need to seed AFTER MEM_INIT each time. Or just run MEM_INIT once, seed the table, then test each token in sequence (resetting OP1 between calls). The JT slot is a pure function (reads A, looks up table, loads OP1) so sequential calls should work.
