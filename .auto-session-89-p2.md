# Session 89 P2: OS event loop trace — GetCSC → key dispatch → ParseInp pathway

## Goal
Trace the OS event loop (CoorMon) with a simulated keypress to map how keystrokes flow from GetCSC through key dispatch to the expression parser.

## Background
- CoorMon (OS main coordinator) lives at 0x08C331
- GetCSC (scan code reader) lives at 0x03FA09
- The scancode table at 0x09F79B translates scan codes to k* key codes (TI-OS `_GetKey` equates)
- kbdKey is the RAM slot where the translated k* code is stored for dispatch
- Session 69 mapped GetCSC and the scan code table. Session 68 decoded the full 4-plane key mapping.
- Nobody has traced what happens AFTER GetCSC returns a key code — how does CoorMon dispatch it? Does it reach ParseInp (0x099914)?

## What to do
Create `TI-84_Plus_CE/probe-phase25ad-event-loop-trace.mjs` that:

1. **Cold boots + full OS init** (use the `coldBoot()` pattern from `probe-phase25z-full-pipeline.mjs`)
2. **Call MEM_INIT at 0x09DEE0** (17 steps)
3. **Simulate a keypress**: Find the kbdKey RAM address. From `ti84pceg.inc` (local copy at `TI-84_Plus_CE/references/ti84pceg.inc`), find `kbdKey` or `kbdScanCode` address. If not found, check `keyboard-matrix.md` or scan for references to GetCSC in the ROM. The key approach: seed the keyboard matrix so GetCSC returns kEnter (0x05) or k1 (a digit key).
4. **Run CoorMon at 0x08C331** with a moderate step budget (50K-100K steps, maxLoopIterations=1000) and **trace which PCs are visited**. Collect a full PC trace (or at least the first 5000 unique PCs).
5. **Analyze the trace**: Which PCs appear? Does the trace visit:
   - GetCSC (0x03FA09)?
   - The scancode table area (0x09F79B)?
   - ParseInp (0x099914)?
   - Any JT slot addresses (0x020100..0x0210FF)?
   - Any unknown dispatch tables?

## Key addresses
- CoorMon: 0x08C331
- GetCSC: 0x03FA09
- ParseInp: 0x099914
- Scancode table: 0x09F79B
- Keyboard matrix base: 0xF50012 (port-mapped, but check `peripherals.js` for how keyboard reads work)
- OP1: 0xD005F8
- IY: 0xD00080
- IX: 0xD1A860

Search `TI-84_Plus_CE/references/ti84pceg.inc` for `kbdKey` to find the RAM address. If it's an IY-offset (e.g., `kbdScanCode equ IY+xx`), compute absolute address as 0xD00080 + offset.
