# Task: ParseInp with Empty OP1 + Pre-Created Variable

## Context
Session 85 found that ParseInp behaves differently based on OP1 contents:
- OP1 = variable name "A" [00 41 ...] + variable exists → 253 steps, errNo=0x00, OP1 unchanged
- OP1 = all zeros + no variable → 918 steps, OP1=5.0, errNo=0x8D (ErrUndefined)

**Key question**: If we create variable "A" via CreateReal but then CLEAR OP1 to all zeros before calling ParseInp, does the parser:
(a) Compute 5.0 AND return errNo=0x00 (because "A" exists in VAT, fixing the ErrUndefined)?
(b) Compute 5.0 AND still return errNo=0x8D (because the error is about storing the result, not finding "A")?
(c) Something else entirely?

## Your Task
Create a probe `TI-84_Plus_CE/probe-phase25aa-empty-op1-with-var.mjs` that:

1. Boots + MEM_INIT + CreateReal("A") — same as `probe-phase25z-full-pipeline.mjs`
2. After CreateReal succeeds, CLEAR OP1 to all zeros: `mem.fill(0x00, 0xD005F8, 0xD005F8 + 9)`
3. Set up tokens "2+3" at 0xD00800 via begPC/curPC/endPC
4. Call ParseInp (0x099914) with the error frame
5. Capture and report:
   - errNo after ParseInp
   - OP1 (9 bytes) after ParseInp — decode as BCD float
   - Steps taken
   - All allocator pointer movements (OPBase, OPS, FPS, pTemp, progPtr, newDataPtr)
   - The last 64 PCs visited
   - Whether it returned to FAKE_RET or was caught by ERR_CATCH
6. Also run a CONTROL: same setup but WITHOUT CreateReal (just MEM_INIT → clear OP1 → ParseInp) to confirm the 918-step/5.0/0x8D baseline still holds
7. Write report to `TI-84_Plus_CE/phase25aa-empty-op1-report.md`

## Key Addresses
- ParseInp entry: 0x099914
- CreateReal: 0x08238A
- MEM_INIT: 0x09DEE0
- OP1: 0xD005F8 (9 bytes)
- errNo: 0xD008DF
- errSP: 0xD008E0
- OPBase: 0xD02590, OPS: 0xD02593, FPS: 0xD0258D, FPSbase: 0xD0258A
- pTemp: 0xD0259A, progPtr: 0xD0259D, newDataPtr: 0xD025A0
- tempMem: 0xD02587
- begPC: 0xD02317, curPC: 0xD0231A, endPC: 0xD0231D
- Token buffer: 0xD00800 (clear 0x80 bytes before writing tokens)
- Tokens for "2+3": [0x32, 0x70, 0x33, 0x3F] (4 bytes: digit2, add, digit3, newline)
- Variable "A" in OP1: [0x00, 0x41, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]
- FAKE_RET: 0x7FFFFE, ERR_CATCH: 0x7FFFFA, MEMINIT_RET: 0x7FFFF6, CREATEREAL_RET: 0x7FFFF2

## Setup Pattern
Copy the full infrastructure from `TI-84_Plus_CE/probe-phase25z-full-pipeline.mjs`:
- `coldBoot()`, `prepareCallState()`, `seedMinimalErrFrame()`, `runCall()`, `setupTokens()`, `setOp1VariableA()`

For the CONTROL run, use a separate memory/executor instance (fresh boot) since the test run modifies state.
