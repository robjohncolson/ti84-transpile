# Session 171 P3: Instruction-Level Dynamic Comparison Probe

## Task
Build `TI-84_Plus_CE/probe-phase171-instr-compare.mjs` — the main deliverable. This probe single-steps through gcd(12,8) at the instruction level within each block, executing each decoded ROM instruction through a mini-interpreter AND comparing against the transpiled block's actual register/memory changes.

## Context
We've verified every individual FP component (gcd_helper, normalization, FPDiv, FPAdd, RLD, etc.) in isolation — all correct. The bug must be a transpilation error in one of the ~230 blocks visited during gcd(12,8). This probe will find it by comparing instruction-by-instruction execution.

## Approach
For each block visited during gcd(12,8):
1. Snapshot all CPU registers + key memory regions BEFORE the block executes (via onBlock)
2. Let the transpiled block execute normally
3. Snapshot all CPU registers + key memory regions AFTER the block executes
4. Using the pre-block snapshot as starting state, decode and execute each ROM instruction through a mini-interpreter
5. Compare the mini-interpreter's final state against the transpiled block's actual output state
6. Report the first block where they diverge, with instruction-level detail

## Mini-Interpreter Requirements
The interpreter only needs to handle instructions found in the gcd path. Based on prior sessions, these include:
- **LD**: reg-reg, reg-imm, reg-mem, mem-reg, (HL)/IX+d variants
- **PUSH/POP**: 16-bit register pairs (BC, DE, HL, AF)
- **CALL/RET**: including conditional variants (CALL Z, CALL NZ, CALL C, CALL NC, RET Z, RET NZ, etc.)
- **JP/JR**: unconditional and conditional
- **ALU**: ADD, ADC, SUB, SBC, CP, AND, OR, XOR, NEG
- **Bit ops**: BIT, SET, RES
- **Shifts/Rotates**: SRL, SLA, RL, RR, RLC, RRC, RLD
- **INC/DEC**: 8-bit and 16-bit
- **Misc**: NOP, HALT, EX DE,HL, EX AF,AF', LDI, LDIR, DI, EI, IM, SCF, CCF, CPL

The eZ80 in ADL mode uses 24-bit addresses and 24-bit register pairs (BC, DE, HL, SP, IX, IY are 24-bit). The 8-bit registers (A, B, C, D, E, H, L) are still 8-bit.

## Flag Computation
The F register bits are: S(7), Z(6), Y(5), H(4), X(3), P/V(2), N(1), C(0).
- ADD/ADC/SUB/SBC/CP: set all flags based on result
- AND: H=1, N=0, C=0, P=parity
- OR/XOR: H=0, N=0, C=0, P=parity
- INC: preserves C, sets H, Z, S, P/V (overflow)
- DEC: preserves C, sets H, Z, S, P/V, N=1
- BIT: Z=!(bit), H=1, N=0, S/P copies
- RLD: shifts BCD digits between A and (HL)

```js
const peripherals = createPeripheralBus({ timerInterrupt: false });
```

```js
// 24-bit: cpu.bc (getter for b<<8|c, but in eZ80 it's 24-bit), cpu.de, cpu.hl, cpu.sp, cpu.ix, cpu.iy
// Shadow: cpu._a2, cpu._f2, cpu._b2, cpu._c2, cpu._d2, cpu._e2, cpu._h2, cpu._l2
// Flags: cpu.f (full byte). S=bit7, Z=bit6, H=bit4, PV=bit2, N=bit1, C=bit0
// PC: cpu.pc
// Memory: mem[addr] for byte read/write
```

## Key Memory Regions to Compare
After each block, compare:
- All CPU registers (A, F, B, C, D, E, H, L, IX, IY, SP, PC)
- OP1 (9 bytes at 0xD005F8)
- OP2 (9 bytes at 0xD00603)
- Stack (top 12 bytes from SP)
- errNo (1 byte at 0xD008DF)

## Output Format
For each block:
```
Block 0xXXXXXX (step N): [MATCH|DIVERGE]
  If DIVERGE:
    Register X: transpiled=0xYY, reference=0xZZ
    Instruction causing divergence: <mnemonic> <operands> at offset +N
```

Stop at the FIRST divergence and print full detail.
