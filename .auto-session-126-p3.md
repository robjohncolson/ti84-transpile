# Auto-Session 126 — Priority 4: Graph Subsystem Investigation

## Task
Investigate the TI-84 CE OS graph subsystem entry points and build a static analysis probe. Goal: map the path from GRAPH key press to pixel rendering.

## Background
The TI-84 transpilation project has:
- Full jump table named: 957/980 slots (session 69, file `TI-84_Plus_CE/phase25h-a-jump-table.json`)
- CoorMon (OS event loop) fully traced at 0x08C331
- Home handler at 0x058241, second-pass at 0x0585E9
- Keyboard scan codes mapped (session 67-68)
- LCD/VRAM pipeline working

The graph subsystem has NOT been investigated. We need to find:
1. What key code does GRAPH produce? (check `TI-84_Plus_CE/phase25g-h-scancode-table-report.md` or the scancode table)
2. What handler does CoorMon dispatch for the graph app?
3. Where is the Y= editor handler?
4. What are the graph-window setup routines?
5. Where is the pixel-plot primitive?

## What to build

### Part 1: Static analysis probe
Create `TI-84_Plus_CE/probe-phase25be-graph-investigation.mjs` that:

1. **Find GRAPH key code**: Read the scancode table at 0x09F79B in the ROM to find the k* code for the GRAPH key. The GRAPH key is in Group 1 (top row), bit position for GRAPH. Check `TI-84_Plus_CE/keyboard-matrix.md` for the physical position.

2. **Find graph app context table**: Search ROM for the graph app context table (similar to home screen's table at 0x0585D3). The home screen uses cxCurApp=0x40. Graph likely uses a different ID. Search the jump table JSON for graph-related entries.

3. **Disassemble graph-related JT entries**: From `phase25h-a-jump-table.json`, find entries with "Graph" in the name. Key ones to look for:
   - `_GraphStandard` or similar
   - `_GrBufCpy` (graph buffer copy)
   - `_SetPixel` / `_GetPixel`
   - `_DrawLine`
   - `_ILine` / `_IPoint`
   - `_ClrLCDFull` or `_ClrGraph`
   
4. **Read the jump table JSON** at `TI-84_Plus_CE/phase25h-a-jump-table.json` and extract all graph-related entries (grep for "Graph", "Plot", "Pixel", "Line", "Draw", "Point", "Window", "Zoom", "Trace", "Y=", "Func").

5. **Disassemble the first 64 bytes** of the top 5 graph-related JT implementations to understand their structure.
