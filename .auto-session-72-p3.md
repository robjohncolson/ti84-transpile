# Task: Positive-path Rcl_StatVar probe — seed stat-var table + statsValid

## Context
Session 70 (Phase 25H-b) probed `Rcl_StatVar` at JT slot 0x0204F0 (impl 0x08019F) with A=tMean=0x21 on a cold OS. Result: the slot returned cleanly with OP1 filled from an uninitialized stat-var table (all 0xFF). The key finding was that the JT slot is a 3-instruction primitive with NO statsValid guard.

This task does the **positive-path** follow-up: seed the stat-var table with a known real value, set the statsValid flag, then call Rcl_StatVar — and verify OP1 matches the seeded value.

## Research needed
The stat-var table layout needs to be determined. From session 70 and ti84pceg.inc:
- `statFlags` is at `IY + 0x09` = `0xD00089` (IY=0xD00080)
- `statsValid` is bit 6 of statFlags
- `tMean` token = `0x21`
- The JT slot calls two subroutines: `CD A5 A3 09` (0x09A3A5 = token→index) then `CD FB F9 07` (0x07F9FB = index→OP1 read)

The token→index routine at 0x09A3A5 converts the stat token (A register) to an index into the stat-var table. The index→OP1 routine at 0x07F9FB reads 9 bytes from the stat-var table at the computed offset into OP1.

To find where stat vars are stored, look at the `references/ti84pceg.inc` file in this repo for stat-related equates. The key equates to look for:
- `statVars` or `Stx` or `statMean` — the base address of the stat variable storage area
- The stat-var table likely starts near `0xD01B00`-`0xD01CFF` based on typical TI OS memory maps

## Approach
1. Read `TI-84_Plus_CE/references/ti84pceg.inc` and search for stat-var related equates to find the exact RAM address of the stat variable table (or at least `mean` / `statVars`).
2. Read the existing probe at `TI-84_Plus_CE/probe-phase25h-b-rcl-statvar.mjs` for the template.
3. Create a new probe `probe-phase25j-statvar-positive.mjs` that:
   a. Cold boots the OS (same coldBoot + postInitState pattern as the FP probes)
   b. Finds the stat-var table base address from ti84pceg.inc
   c. Seeds a known real (e.g., 42.0) into the correct offset for tMean in the stat-var table using `writeReal` from `fp-real.mjs`
   d. Sets statsValid (bit 6 of byte at 0xD00089)
   e. Sets A = 0x21 (tMean token)
   f. Calls 0x08019F (Rcl_StatVar impl) via the same FAKE_RET pattern
   g. Reads OP1 via `readReal` and asserts it equals 42.0 within tolerance 1e-6
4. If the stat-var table base can't be found in ti84pceg.inc, try a dynamic approach: run the token→index subroutine (0x09A3A5) with A=0x21 and instrument it to see what address it computes for the read.
