# Session 140 P1 — Graph Rendering in Browser-Shell

## Task
Integrate the validated GraphPars→IPoint rendering pipeline into `browser-shell.html` as a "Graph" button/mode, AND investigate the negative-X infinite loop.

## Context
Session 139 validated the full pipeline in `probe-phase139-graphpars-render.mjs`:
- Cold boot + MEM_INIT
- CreateEqu (0x082438) to build Y1=X in VAT with OP1=[0x03, 0x10, 0x00...]
- GraphPars body at 0x099874 evaluates Y for each X
- tX intercept at PC=0x07D1B4: push X as 9-byte BCD to FPS (pointer at 0xD0258D), advance FPS by 9
- Read Y from OP1 (0xD005F8) after GraphPars returns
- IPoint at 0x07B451 with drawMode=1 renders pixels to plotSScreen (0xD09466)
- 133/133 positive X values succeeded, 17 pixels written

The .SIS prefix issue (session 128) causes IPoint bounds-check to use 8-bit comparison, limiting X coordinates to <64 pixels. This is a known limitation — work around it in the browser by mapping pixel coordinates to fit within the 8-bit range, OR by directly writing to plotSScreen from JS when IPoint rejects the coordinate.

## Part A: Browser-Shell Graph Integration

Read `TI-84_Plus_CE/browser-shell.html` to understand the existing direct-eval UI structure (search for "direct-eval", "EVAL_TOKEN_MAP", "evalExpression").

Add a graph rendering feature:
1. Add a "Graph Y1=" text input and "Graph" button next to the existing eval controls
2. When clicked:
   - Tokenize the expression (reuse existing `tokenizeExpression()`)
   - Run: cold boot → MEM_INIT → CreateEqu (Y1 = tokenized expression) → loop over X values
   - For each X in the graph window (default Xmin=-10, Xmax=10, 132 columns = pixel cols 0-131):
     - Convert X to BCD using `writeReal` from fp-real.mjs (already imported in browser-shell)
     - Call GraphPars body (0x099874) with tX intercept at 0x07D1B4
     - Read Y from OP1, convert with `readReal`
     - Map (X,Y) to pixel coordinates: px = (X - Xmin) / (Xmax - Xmin) * 264, py = 132 - (Y - Ymin) / (Ymax - Ymin) * 164
   - Skip negative X values for now (they cause infinite loops — Part B)
   - Render to a canvas element (320x240 or similar) using JS canvas API
   - Show the plotSScreen contents OR the JS-computed pixel map
3. Seed required RAM before GraphPars:
   - pixWideP (0xD014FE) = 320 (0x0140)
   - pixWide_m_2 (0xD01501) = 240 (0x00F0) 
   - drawColorCode (0xD026AE) = 0x1F (blue)
   - errSP (0xD008E0) = valid error frame address
   - IY = 0xD00080
   - MBASE = 0xD0
