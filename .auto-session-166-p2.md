# Session 166 P2: Dump ALL register state at normalization loop exit

## Context
You are working on a TI-84 Plus CE ROM transpilation project at `C:/Users/rober/Downloads/Projects/school/ti84-transpile`. The project lifts eZ80 ROM machine code into JavaScript.

We are investigating why `gcd(12,8)` fails with E_Domain. The normalization function at 0x07CA48 is called during the compound function 0x07C747. For OP1=12 (exp=0x81), the normalization loop runs 2 Shl14 iterations, zeroing the mantissa, then exits.

## Your Task
Build a probe `TI-84_Plus_CE/probe-phase166-regstate-normexit.mjs` that:

1. Calls compound function 0x07C747 with OP1=12, OP2=8 (same as gcd context)
2. Watches for the normalization loop exit point(s). The loop exits when:
   - The exponent drops below 0x80 (loop condition: SUB 0x80, RET C at 0x07CA57)
   - Or via the secondary path at 0x07C9AF (LoopExit → BigShift at 0x07FAC2)
3. At the loop exit block, dump ALL register state:
   - A, B, C, D, E, H, L, F (8-bit registers)
   - IX, IY, SP (24-bit)
   - HL', DE', BC' (shadow registers if accessible via cpu._hlp, cpu._dep, cpu._bcp — check what's available)
   - OP1 (9 bytes at 0xD005F8)
   - OP2 (9 bytes at 0xD00603)
   - FPS pointer (3 bytes at 0xD0258D)
4. Also dump register state at these key points:
   - Entry to 0x07CA48 (normalization start)
   - Entry to 0x07C74F (InvOP1S, post-normalization)
   - Entry to 0x07C77F (FPAdd)
   - At FAKE_RET (final state)
5. Compare the register states at each checkpoint

## Key Addresses
