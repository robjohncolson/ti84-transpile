## BCD format for seeding graph window
TI BCD format: byte 0 = sign+exponent (0x00=positive exp 0, 0x80=positive exp 0 for 1.0, 0x81=exp 1 for 10.0). For -10.0: [0x80, 0x81, 0x10, 0x00, ...] and for 10.0: [0x00, 0x81, 0x10, 0x00, ...].

Actually the TI BCD format is:
- Byte 0: sign (bit 7) + object type (bits 6-0). For Real: 0x00 (positive) or 0x80 (negative)
- Byte 1: exponent + 0x80. So exponent 1 = 0x81, exponent 0 = 0x80
- Bytes 2-8: BCD mantissa digits (2 per byte, 14 digits total)

So 10.0 = [0x00, 0x81, 0x10, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]
And -10.0 = [0x80, 0x81, 0x10, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]

## Setup pattern
```javascript
import { createExecutor, coldBoot } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';

const mem = new Uint8Array(16 * 1024 * 1024);
const perph = createPeripheralBus(mem, { timerInterrupt: false });
const { ex, cpu } = createExecutor(mem, perph);
coldBoot(mem, ex, cpu);

// Run MEM_INIT
cpu.pc = 0x09DEE0;
cpu.sp = 0xD1A87E;
mem[0x7FFFFE] = 0x76; // HALT
mem[0x7FFFFF] = 0x76;
mem[0x800000] = 0x76;
cpu.sp -= 3;
mem[cpu.sp] = 0xFE; mem[cpu.sp+1] = 0xFF; mem[cpu.sp+2] = 0x7F;
ex.run(50000);
```
