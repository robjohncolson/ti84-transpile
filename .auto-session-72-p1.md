# Task: Harden fp-real.mjs — carry-overflow normalization + NaN/Inf handling

## Context
`TI-84_Plus_CE/fp-real.mjs` is a TI-83/84 9-byte BCD real number helper (writeReal/readReal). It has two known gaps from prior Codex review:

1. **Carry-overflow**: `writeReal` rounds to 14 significant digits via `toPrecision(14)`, but if rounding causes a carry (e.g., 9.9999999999999x → 10.000...), the first BCD digit becomes 10, which is invalid. Need a carry-fix loop after digit extraction: if any digit >= 10, carry into the next-higher digit and increment the exponent if the MSB overflows.

2. **NaN/Inf**: Currently `encodeReal(NaN)` and `encodeReal(Infinity)` silently map to zero (the `!isFinite(num)` branch). Change this to throw a clear error: `throw new Error('fp-real: cannot encode NaN/Infinity')`. Zero should still work (it's in the same branch but guarded by `num === 0`).
