# Task: Disassemble 0x05E820 — the function that calls the cx-zeroing LDDR

## Context
Session 94 found the TRUE cxCurApp zeroing site: `LDDR` at ROM 0x08321B, inside function 0x0831A4. The direct caller is `CALL 0x0831A4` at ROM address 0x05E836, inside a function starting at 0x05E820 (or nearby).

We need to understand WHY this function zeroes the cx context block (cxMain through cxCurApp at 0xD007CA-0xD007E1). Is it a "context switch" that saves/restores cx? A cleanup routine? An intentional cold-restart path?

## Deliverables

Create a probe file `TI-84_Plus_CE/probe-phase25aj-cx-zeroing-caller-disasm.mjs` that:

1. Reads the ROM binary at `TI-84_Plus_CE/ROM.rom`
2. Statically disassembles ROM bytes from 0x05E800 to 0x05E900 (256 bytes surrounding the call site at 0x05E836)
3. For each instruction, decode the eZ80 opcode and print: address, hex bytes, mnemonic
4. Identify:
   - The function boundary (look backward from 0x05E820 for a RET/JP/RETI instruction)
   - All CALL targets within this function
   - All references to cx-range addresses (0xD007CA-0xD007E1) — direct loads or stores
   - All references to IY+offset flag operations
5. Also disassemble the zeroing function itself: 0x0831A4 to 0x083230 (to understand what arguments it expects on stack/registers)
6. Search the ROM (all 4MB) for `CD A4 31 08` (little-endian CALL 0x0831A4) to find ALL callers of the zeroing function
7. Write a report to `TI-84_Plus_CE/phase25aj-cx-zeroing-caller-disasm-report.md`

## eZ80 decode approach
Import and use the decoder from `ez80-decoder.js`:
```js
import { decodeInstruction } from './ez80-decoder.js';
// Returns { mnemonic, length, bytes } or similar
```

If `decodeInstruction` doesn't exist or doesn't work for static decode, fall back to manual byte decoding. Common eZ80 opcodes:
- `C9` = RET, `CD xx xx xx` = CALL addr (3-byte address in ADL mode), `C3 xx xx xx` = JP addr
- `FD CB nn xx` = IY+offset bit operations
- `21 xx xx xx` = LD HL,imm24, `11 xx xx xx` = LD DE,imm24
- `ED B0` = LDIR, `ED B8` = LDDR
- `32 xx xx xx` = LD (addr),A
