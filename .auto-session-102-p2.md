Session 101 confirmed that 0xFFF909 is a corrupted return address caused by LDIR wraparound when the uninitialized history buffer is read. When numLastEntries=1 (at 0xD01D0B) but the history buffer at 0xD0150B is all zeros, 0x0922A2 computes BC = mem[0xD01508] - entryAddr, but 0xD01508=0x000000 → subtraction wraps to ~3M → LDIR copies garbage → stack corrupted → RET pops 0xFFF909.

The ENTER handler flow with numLastEntries>0 is:
- 0x0921CB checks numLastEntries at 0xD01D0B — if nonzero, runs history recall
- History recall reads entries from buffer at 0xD0150B
- After history recall, execution should reach 0x058693 (common tail)
- 0x058693 eventually calls ParseInp at 0x0586E3 via trampoline 0x099910
