# Session 143 P2: OneVar Full Entry — Trace E_JError in Sort Routine

## Context

The TI-84 ROM transpilation project lifts eZ80 ROM to JS. OneVar computes 1-variable statistics on a list. Entry 0x0A9325 (partial) computes 6/8 stats but skips min/max. Entry 0x0A9319 (full) enables min/max by clearing bit 5 of IY+9, but triggers E_JError at 8,250 steps (session 142).

The min/max routine at 0x0AA6FA is where the error occurs — it needs to sort/compare list elements but hits missing blocks.

## Your Task

Create a NEW probe file `TI-84_Plus_CE/probe-phase143-onevar-sort-trace.mjs` based on `probe-phase142-onevar-full.mjs`. Modifications:

1. **Add PC tracing**: Track ALL unique PCs visited during the OneVar run from 0x0A9319. Keep a `Map<pc, hitCount>`.

2. **Track missing blocks**: Use `onMissingBlock` callback to record every missing block PC and its hit count.

3. **Track CALL targets**: Monitor PCs in the range 0x0A0000-0x0AFFFF and 0x080000-0x09FFFF that are visited, to understand the subroutine chain from 0x0AA6FA.

4. **Snapshot at error**: When errNo changes from 0x00 to any non-zero value, record the PC and step number.

5. **Print last 100 PCs before termination** to see the error trail.

6. **Print all missing blocks** sorted by address.

7. **Print PC frequency for the sort/min-max range** (0x0AA000-0x0AB000) — any PCs hit in this range.

## Setup Pattern (from probe-phase142-onevar-full.mjs)

The probe must:
1. Cold boot + MEM_INIT
2. Seed allocator (USERMEM/EMPTY_VAT_ADDR pattern)
3. Create manual VAT entry for L1 with data [10,20,30,40,50]
4. Seed FPS with L1 OP1 pushed
5. Set statFlags2 bit 2 at IY+26 (0xD0009A)
6. Set A=1 (single-list mode)
7. Zero statMode at 0xD01190
8. Call OneVar at 0x0A9319 (FULL entry)
9. Use error catch frame

## Key Constants
