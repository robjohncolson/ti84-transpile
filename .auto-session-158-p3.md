# Session 158 P3 — Standalone 0x07CA48 Normalization Loop Trace with Small Values

## Context
We're debugging the FP normalization loop at 0x07CA48 in our TI-84 CE ROM transpiler. This function shifts the mantissa left (via Shl14 at 0x07FB33) and decrements the exponent (via 0x07FDF1) until exponent < 0x80. For value 1200 (exp 0x83), it runs 4 iterations and zeroes the mantissa.

The normalization loop then FALLS THROUGH into FPDiv at 0x07CAB9.

## ROM Disassembly of 0x07CA48 (already decoded by Opus)
```
0x07CA48: CD 4A FD 07   CALL 0x07FD4A    (validity check)
0x07CA4C: C8            RET Z            (return if Z)
0x07CA4D: AF            XOR A
0x07CA4E: 32 01 06 D0   LD (0xD00601), A (clear byte past OP1 mantissa)
0x07CA52: 3A F9 05 D0   LD A, (0xD005F9) (load exponent)
0x07CA56: D6 80         SUB 0x80         (subtract bias)
0x07CA58: D8            RET C            (return if exp < 0x80)
0x07CA59: FE 0F         CP 0x0F          (compare with 15)
0x07CA5B: 38 0C         JR C, +12 → 0x07CA69  (if shifts needed < 15)
0x07CA5D: 18 42         JR +66 → 0x07CAA1     (if shifts needed >= 15, special handling)

; Main shift loop (entered when A < 0x0F, i.e., 0-14 shifts needed):
0x07CA5F: 3A F9 05 D0   LD A, (0xD005F9) (reload exponent)
0x07CA63: FE 80         CP 0x80          (compare with 0x80)
0x07CA65: DA AF C9 07   JP C, 0x07C9AF   (exit if exp < 0x80)
0x07CA69: CD 33 FB 07   CALL 0x07FB33    (Shl14 — shift mantissa left 1 digit)
0x07CA6D: CD F1 FD 07   CALL 0x07FDF1    (decrement exponent)
0x07CA71: 18 EC         JR -20 → 0x07CA5F (loop)
```

## Your Task
Build a probe `TI-84_Plus_CE/probe-phase158-norm-standalone.mjs` that calls 0x07CA48 standalone (NOT through gcd) with various test values and traces every step.

### Test Cases
1. **OP1 = 1.0** (exp 0x80, mantissa [10 00 00 00 00 00 00])
   - Expected: SUB 0x80 gives A=0, RET C is NOT taken (0 has no carry), CP 0x0F → A<15 → JR to 0x07CA69
   - Then loop: CP 0x80 → 0x80 >= 0x80 → NO carry → CALL Shl14 → CALL decExp
   - After 1 shift: mantissa [00 00...], exp 0x7F → JP C to 0x07C9AF
   - **Result: 1 shift, value destroyed (mantissa = 0)**
   
2. **OP1 = 5.0** (exp 0x80, mantissa [50 00 00 00 00 00 00])
   - Same as above, 1 shift → mantissa [00 00...], exp 0x7F

3. **OP1 = 12.0** (exp 0x81, mantissa [12 00 00 00 00 00 00])
   - SUB 0x80 → A=1, no carry, CP 0x0F → A<15
   - Loop iteration 1: exp 0x81 >= 0x80 → shift → mantissa [20 00...], exp 0x80
   - Loop iteration 2: exp 0x80 >= 0x80 → shift → mantissa [00 00...], exp 0x7F
   - Exit: exp 0x7F < 0x80
   - **Result: 2 shifts, value destroyed**

4. **OP1 = 0.5** (exp 0x7F, mantissa [50 00 00 00 00 00 00])
   - SUB 0x80 → A=0xFF, carry set → RET C immediately
   - **Result: 0 shifts, value preserved**

### Implementation
1. Do a full cold boot + MEM_INIT (copy from probe-phase157 boilerplate)
2. For each test case:
   - Set OP1 to the test value
   - Set SP to a safe stack location
   - Push FAKE_RET (0x7FFFFE) onto stack as return address
   - Set PC = 0x07CA48
   - Run with step-by-step logging: at each block, print PC, A, HL, OP1 mantissa bytes, exponent
   - Log every CALL/RET event
   - Stop at FAKE_RET or after 500 steps
   - Print final OP1 value, number of shifts, and whether value was preserved

3. For test case 3 (OP1=12.0), also check: after the function completes, does it fall through to FPDiv at 0x07CAB9? The disassembly shows the exit goes to JP 0x07C9AF (not directly to FPDiv). Trace what 0x07C9AF does.

### Additional Investigation
After running all 4 tests, answer these questions:
- Does 0x07FD4A (validity check) set Z flag for any of these values? If so, RET Z would prevent the normalization entirely.
- What does 0x07C9AF (loop exit target) do? Dump the ROM bytes there and decode.
- Is there a path where the normalization loop PRESERVES the mantissa (e.g., for values with exponent exactly 0x80)?
