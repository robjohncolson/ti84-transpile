# Auto-Session 96 P1: CoorMon RAM CLEAR trace — steps 2579-3584

## Context

CoorMon (0x08C331) is the TI-84 CE OS main event loop. When seeded with cx state (cxMain=0x058241, cxCurApp=0x40, handler pointers, IY flags, error frame, tokenized "2+3" at userMem), it reaches a RAM CLEAR routine at step 3584 (PC=0x001881) that zeroes all OS RAM from 0xD00000 to 0xD13FD7. This kills the cx context we just seeded.

Session 95 P2 probe (`probe-phase25aj-coormon-parseinp-trace.mjs`) confirmed:
- cx is correctly initialized at step 2579 (via 0x08C782 LDIR)
- GetCSC fires at step 1643
- RAM CLEAR at step 3584 (PC=0x001881) wipes everything
- HomeHandler (0x058241) is NEVER dispatched
