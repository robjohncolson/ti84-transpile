# Auto-Session 112 P3: Investigate fmtFlags bit 4 and IY+0x47 bit 7

## Goal
Build a diagnostic probe to document what fmtFlags bit 4 (0x10 at 0xD000CA) and IY+0x47 bit 7 (0x80 at 0xD000C7) control. These are the ONLY non-zero IY flag bytes after MEM_INIT (found in session 111). Neither affects ParseInp("2+3") but they may matter for future features (graph rendering, error handling, number formatting).

## What to build

Create `TI-84_Plus_CE/probe-phase25ba-iy-flags.mjs` that:

### Part 1: Document the flag bits
Search the ROM for instructions that read/write these flag bytes:
- `BIT n,(IY+0x4A)` — fmtFlags is at IY+0x4A (0xD000CA = 0xD00080 + 0x4A). Look for BIT 4, SET 4, RES 4 instructions targeting IY+0x4A
- `BIT n,(IY+0x47)` — IY+0x47 = 0xD000C7. Look for BIT 7, SET 7, RES 7 instructions targeting IY+0x47

The ROM is at `TI-84_Plus_CE/ROM.rom` (4MB, raw binary). eZ80 BIT/SET/RES on (IY+d) instructions are encoded as:
- `FD CB dd xx` where dd is the displacement (0x4A for fmtFlags, 0x47 for the other) and xx encodes the bit operation:
  - BIT n: xx = 0x46 + (n * 8) — so BIT 4 = 0x66, BIT 7 = 0x7E
  - SET n: xx = 0xC6 + (n * 8) — so SET 4 = 0xE6, SET 7 = 0xFE
  - RES n: xx = 0x86 + (n * 8) — so RES 4 = 0xA6, RES 7 = 0xBE

So search for these 4-byte patterns in the ROM:
1. `FD CB 4A 66` — BIT 4,(IY+0x4A) — tests fmtFlags bit 4
2. `FD CB 4A E6` — SET 4,(IY+0x4A) — sets fmtFlags bit 4
3. `FD CB 4A A6` — RES 4,(IY+0x4A) — clears fmtFlags bit 4
4. `FD CB 47 7E` — BIT 7,(IY+0x47) — tests IY+0x47 bit 7
5. `FD CB 47 FE` — SET 7,(IY+0x47) — sets IY+0x47 bit 7
6. `FD CB 47 BE` — RES 7,(IY+0x47) — clears IY+0x47 bit 7

Also search for direct memory access patterns:
7. `LD A,(0xD000CA)` followed by AND/OR/BIT operations
8. `LD A,(0xD000C7)` followed by AND/OR/BIT operations

For each ROM hit, report the address and 8 bytes of surrounding context.
