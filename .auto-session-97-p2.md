# Auto-Session 97 — Priority 2: Trace buffer compaction cx-zeroing trigger at 0x05E3A2

## Context

This is a TI-84 Plus CE ROM transpilation project. The OS ROM (eZ80 machine code) has been lifted to JavaScript. We run "probes" — Node.js scripts that boot the OS, seed state, and trace execution.

Session 96 confirmed that cxCurApp (0xD007E0) gets zeroed NOT by RAM CLEAR but by buffer compaction: 0x05E3A2→0x05E836→0x0831A4 (LDDR sweeps through cx range as side effect) at step ~18282. We need to understand WHY this buffer compaction fires — what triggers 0x05E3A2? Possible callers include:
- 0x0973C8's buffer flush (0x05E872→0x05E820)
- The second-pass handler at 0x0585E9
- CoorMon's main loop directly
