# Task: Edit-buffer integration — CreateNumEditBuf + direct BufInsert

## Context
You are working on a TI-84 Plus CE ROM transpilation project at `C:/Users/rober/Downloads/Projects/school/ti84-transpile`. Session 129 mapped the edit buffer system:

- **Edit buffer = gap buffer** with pointers:
  - editTop = 0xD02437 (24-bit pointer)
  - editCursor = 0xD0243A (24-bit pointer)
  - editTail = 0xD0243D (24-bit pointer)
  - editBtm = 0xD02440 (24-bit pointer)
  - editSym = 0xD0244E (24-bit pointer)
  - editDat = 0xD02451 (24-bit pointer)

- **OS routines**:
  - BufClr → 0x0ADAC3 (needs pre-initialized buffer)
  - BufInsert → 0x05E2A0 (token in C/BC, calls 0x05E3D6 room check, updates editCursor)
  - BufDelete → 0x05E367
  - BufLeft → 0x05E242
  - BufRight → 0x05E27E
  - CreateNumEditBuf → 0x096E09 (likely the actual initializer that sets up the gap buffer)

- **Key codes**: k0=0x8E through k9=0x97 (these are _GetKey codes, NOT scan codes)
- **Token codes**: digit tokens 0x30-0x39 for '0'-'9', plus operator tokens like 0x70 (+), 0x71 (-), 0x82 (*), 0x83 (/)

Session 129 found that the home handler stalls at 0x09EFDE (MMIO busy-wait) before reaching BufInsert. The next step is to bypass the handler and call the buffer routines directly.

## Your Task

Build a probe `TI-84_Plus_CE/probe-phase130-edit-buffer.mjs` that:

1. **Initialize via CreateNumEditBuf (0x096E09)**: After MEM_INIT, call CreateNumEditBuf. It should set up editTop/editCursor/editTail/editBtm pointers. After the call, dump all 6 edit buffer pointers. Run with a budget of 10000 steps. If it stalls or errors, note what happened.

2. **Insert tokens via BufInsert (0x05E2A0)**: After CreateNumEditBuf, call BufInsert with C = 0x32 (token for digit '2'). Check:
   - Does editCursor advance?
   - Is the token byte present in memory between editTop and editCursor?
   
   Then insert more tokens: 0x70 ('+'), 0x33 ('3'). After all inserts, dump the buffer contents between editTop and editBtm to see "2+3" in token form.

3. **Test BufClr (0x0ADAC3)** after inserting tokens — verify it resets editCursor back to editTop.

4. **If CreateNumEditBuf fails/stalls**: Fall back to manually initializing the gap buffer. Allocate a 256-byte region in RAM (e.g., at 0xD00A00-0xD00AFF) and set:
   - editTop = 0xD00A00
   - editCursor = 0xD00A00 (initially empty)
   - editTail = 0xD00AFF (end of gap)
   - editBtm = 0xD00AFF
   Then try BufInsert directly with this manually seeded buffer.

5. **Test ParseInp reading from edit buffer**: If BufInsert works and tokens are in the buffer, set begPC/curPC to editTop and endPC to editCursor (the cursor marks end of pre-gap content). Then run ParseInp at 0x099914 to see if it can evaluate the expression from the real edit buffer instead of a manually-seeded token array. This is an exploratory test — it may not work on the first try.
