# Task: Phase 25AG — Seed cx context + NewContext(A=0x40) + CoorMon dispatch

## Goal
Build a probe that properly initializes the TI-84 CE OS context so CoorMon (the OS event loop at 0x08C331) can dispatch a keypress to ParseInp instead of falling through to the RAM CLEAR routine at 0x001881.

## Background
Session 91 found:
- RAM CLEAR at PC=0x001881 zeroes ALL OS RAM (0xD00000-0xD13FD7) because cxMain and handler pointers at 0xD007CA-0xD007E1 are all zeros — no valid dispatch targets.
- NewContext(0x08C79F) wraps NewContext0(0x08C7AD): saves context to cxPrev shadow, clears IY flags, then at 0x08C7EE loads HL=0xD007E0 and CP (HL) to compare A with current cxCurApp.
- 0x08C8CB sets cxMain=0x058241 and A=0x40 (home screen app ID).
- 0x08C782 is a bulk cx copier: `LD DE,0xD007CA; LD BC,0x15; LDIR` (21 bytes into cx range from HL source).

## What to build
Create `TI-84_Plus_CE/probe-phase25ag-cx-init-coormon.mjs` that:

1. Cold boot (same pattern as existing probes — see `probe-phase25x-meminit-then-parseinp.mjs`)
2. Run MEM_INIT at 0x09DEE0 (18 steps, sets up heap pointers)
3. **Manually seed the cx context range** at 0xD007CA-0xD007E1 with realistic values:
   - cxMain (0xD007CA, 3 bytes LE) = 0x058241 (home screen main handler)
   - cxPPutAway (0xD007CD, 3 bytes) = 0x000000
   - cxPutAway (0xD007D0, 3 bytes) = 0x000000
   - cxReDisp (0xD007D3, 3 bytes) = 0x000000
   - cxErrorEP (0xD007D6, 3 bytes) = 0x000000
   - cxSizeWind (0xD007D9, 3 bytes) = 0x000000
   - cxPage (0xD007DC, 3 bytes) = 0x000000
   - cxCurApp (0xD007E0, 1 byte) = 0x40 (home screen)
4. Also seed kbdKey (0xD0058C) = 0x05 (kEnter), kbdGetKy (0xD0058D) = 0x05, keyMatrix[1] = 0xFE (ENTER pressed in SDK group 6)
5. Set errSP (0xD008E0, 3 bytes) to a FAKE_RET sentinel (e.g. 0x7FFFFA) for error recovery
6. Push FAKE_RET (0x7FFFFE) as return address on stack
7. Run CoorMon at 0x08C331 with a 100K step budget, maxLoopIterations=8192

Track and report:
- Whether cxCurApp stays 0x40 (or gets zeroed again)
- Whether GetCSC (0x03FA09) is hit
- Whether ParseInp (0x099914) is reached
- Whether the RAM CLEAR at 0x001881 is reached
- All unique PCs visited (first 200)
- OP1 state, errNo, pointer snapshot after run
- Termination reason

Also try a SECOND scenario: Instead of manually seeding the cx fields, call NewContext(0x08C79F) with A=0x40 after MEM_INIT, then run CoorMon. Compare results.

## Key addresses
- MEM_INIT: 0x09DEE0 (RET at 0x09DF34)
- NewContext: 0x08C79F
- CoorMon: 0x08C331
- GetCSC: 0x03FA09
- ParseInp: 0x099914

- IY: 0xD00080
- IX: 0xD1A860
