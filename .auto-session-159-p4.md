# Session 159 P4 — Trace D register setup in normalization for FPDiv

## Context

You are working on a TI-84 Plus CE ROM transpilation project at `C:/Users/rober/Downloads/Projects/school/ti84-transpile`. The normalization function at 0x07CA48 runs before FPDiv at 0x07CAB9. Session 157 found that at 0x07CA73, the exponent combination code does:

```
LD A,(0xD00604)   ; load OP2 exponent byte
NEG               ; negate it  
LD D,A            ; store in D
```

The question: does FPDiv at 0x07CAB9 USE the D register? If so, what does it expect D to contain? This could explain why the bypass (skipping normalization) produces wrong results.

## Task

Build a probe file `TI-84_Plus_CE/probe-phase159-dreg-trace.mjs` that:

### Part A: Static disassembly of FPDiv entry (0x07CAB9-0x07CB20)
- Read ROM bytes from 0x07CAB9 and disassemble ~40 instructions or until first RET/JP
- Print each instruction with address, hex bytes, mnemonic
- Specifically flag ANY instruction that reads D or DE register
- Count how many instructions into FPDiv before D is first READ vs first WRITTEN (overwritten)

### Part B: D register value at FPDiv entry in the NORMAL flow (through normalization)
- Seed OP1 = BCD 1200 (type=0x00, exp=0x83, mantissa=[12 00 00 00 00 00 00])
- Seed OP2 = BCD 1200 (same)
- Enter at 0x07CA48 (normalization entry)
- Use onBlock to trace D register at EVERY block from 0x07CA73 onward
- When execution reaches 0x07CAB9 (FPDiv entry), log D, A, F, HL, BC
- Continue through FPDiv and log result

### Part C: D register value in the BYPASS case
- Same OP1/OP2 setup
- Enter directly at 0x07CAB9 (FPDiv, no normalization)
- Log D at FPDiv entry (will be whatever default/uninitialized value)
- Run to completion and log result

### Part D: D register injection test
- Enter directly at 0x07CAB9 BUT set D to the value observed in Part B
- Run to completion
- Does fixing D alone make FPDiv(1200/1200) return 1.0?

### Part E: Also check what value 0xD00604 contains
- 0xD00604 is OP2[1] = the exponent byte of OP2
- For BCD 1200, OP2 exponent = 0x83
- NEG of 0x83 = 0x7D (= -0x83 in 8-bit twos complement = 125)
- Verify this matches what D actually holds at FPDiv entry in Part B
