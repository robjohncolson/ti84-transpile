# Session 166 P1: Trace register A through normalization loop and post-loop code

## Context
You are working on a TI-84 Plus CE ROM transpilation project at `C:/Users/rober/Downloads/Projects/school/ti84-transpile`. The project lifts eZ80 ROM machine code into JavaScript.

We are investigating why `gcd(12,8)` fails with E_Domain. The root cause is narrowed to the **normalization function at 0x07CA48**. This function is called from within the compound function 0x07C747 (which does OP1→OP2 copy, then normalize, then negate, then FPAdd).

**The normalization loop** at 0x07CA48 calls Shl14 (mantissa left-shift at 0x07FB33/0x07FB50) repeatedly while decrementing the exponent. Each Shl14 iteration rotates the leading BCD digit into register A. For integer inputs (exp >= 0x80), the loop shifts until mantissa is all zeros.

**The hypothesis**: On a real TI-84, register A's value after Shl14 iterations might be READ by post-loop code (e.g., stored to memory, combined with exponent). Our transpilation might be losing A's value. Session 158 P1 confirmed Shl14 correctly produces A=leading digit, but nobody checked what happens to A AFTER the normalization loop exits.

## Your Task
Build a probe `TI-84_Plus_CE/probe-phase166-rega-norm-trace.mjs` that:

1. Sets up gcd(12,8) context: OP1=[00 81 12 00 00 00 00 00 00] (=12.0), OP2=[00 80 80 00 00 00 00 00 00] (=8.0)
2. Calls the compound function at 0x07C747 standalone (NOT the full gcd handler)
3. Logs register A at EVERY block step from entry to return, with the PC address
4. Specifically watch for: what is A immediately after the normalization loop exits? At 0x07C9AF? At 0x07CA73+? Does any instruction after the loop read/use A?

## Key Addresses
- 0x07C747: compound function entry (OP1→OP2 copy + normalize + negate + FPAdd)
- 0x07CA48: normalization function entry
- 0x07FB33/0x07FB50: Shl14 (mantissa left-shift, puts leading digit in A)
- 0x07C9AF: loop exit path (secondary normalization check)
- 0x07CA73: post-loop exponent combination
- 0x07CA9F: RET (normalization return)
- 0x07C74F: where compound function resumes after normalization (InvOP1S + FPAdd)
- 0x07C77F: FPAdd entry
- 0x7FFFFE: fake return sentinel
