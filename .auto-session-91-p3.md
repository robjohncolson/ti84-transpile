# Auto-Session 91 P3: Store+Recall Round-Trip with Correct Token Encoding

## Context
We're transpiling a TI-84 Plus CE ROM into JavaScript. Session 90 built a store-recall round-trip probe that validates the full pipeline: MEM_INIT → CreateReal("Ans") → ParseInp("2+3") → RclVarSym("Ans"). The pipeline mechanics work (all 4 stages return with errNo=0x00), BUT the token encoding was wrong.

Session 90 used tokens `[0xB0, 0x70, 0xB3, 0x3F]` — these are WRONG.
The correct TI-OS tokens for "2+3\n" are `[0x32, 0x70, 0x33, 0x3F]` where:
- 0x32 = token for digit "2"
- 0x70 = tAdd (addition)
- 0x33 = token for digit "3"
- 0x3F = tEnter (newline/enter)

With wrong tokens, ParseInp ran only 253 steps (expected ~918). With correct tokens, we expect ParseInp to run ~918 steps, produce OP1=5.0, and then RclVarSym("Ans") should return 5.0.
