# Task: Browser-shell direct ParseInp bypass — keystroke → expression evaluation

## Context
In the TI-84 Plus CE ROM transpilation project, the full parse→store→recall pipeline is VALIDATED (session 92): MEM_INIT → ParseInp("2+3") → OP1=5.0 → CreateReal("Ans") → RclVarSym("Ans") → OP1=5.0. However, CoorMon (the OS event loop) still can't dispatch keystrokes to ParseInp because cxCurApp gets zeroed at 0x04C990.

**This task bypasses CoorMon entirely**: wire the browser shell to call ParseInp directly when the user presses ENTER, then display the OP1 result on the LCD.

## What to do

### Part A: Create a helper module `TI-84_Plus_CE/direct-eval.js`

This module exports a function `evaluateExpression(cpu, memory, tokenBytes)` that:

1. Runs MEM_INIT at 0x09DEE0 (resets heap pointers — 18 steps)
2. Writes `tokenBytes` (a Uint8Array of TI-OS token bytes) to userMem at 0xD1A881
3. Sets OP1 to all zeros (9 bytes at the address cpu runtime uses — check how probes do it)
4. Sets begPC = curPC = 0xD1A881 (the token buffer start)
5. Sets endPC = 0xD1A881 + tokenBytes.length
6. Pushes FAKE_RET sentinel on stack
7. Runs ParseInp at 0x099914 for up to 5000 steps
8. Reads OP1 (9 bytes) and decodes the BCD result to a JavaScript number
9. **After ParseInp**: resets FPS to FPSbase (0xD1A881) and OPS/OPBase to 0xD3FFFF (critical — session 92 found ParseInp leaves these in post-evaluation state)
10. Runs CreateReal("Ans") at 0x08238A to store the result (214 steps)
11. Returns `{ value: <number>, steps: <number>, error: <errNo byte> }`

**Token encoding reference** (from ti84pceg.inc and session 92):
- Digits: 0x30='0', 0x31='1', ..., 0x39='9'
- Operations: 0x70='+', 0x71='-', 0x82='*', 0x83='/'
- Other: 0x3F='?' (newline/end), 0x0B='(' , 0x0C=')'
- tEnter/end: 0x3F

### Part B: Modify `TI-84_Plus_CE/browser-shell.html`

Add a minimal integration that:

1. After the existing boot sequence completes, initializes the direct-eval pipeline (runs MEM_INIT once)
2. Maintains a "current input" token buffer that maps keyboard presses to TI-OS tokens
3. When ENTER is pressed, calls `evaluateExpression()` with the accumulated tokens
4. Displays the numeric result on the LCD (use PutS or direct VRAM write — whatever the existing shell already does for text)
5. Clears the token buffer for the next expression

The keyboard mapping should cover at minimum:
- 0-9 digits
- +, -, *, / operations
- ENTER to evaluate
- CLEAR to reset

### Part C: Token mapping table

Create a mapping from the existing keyboard scan codes (already handled in browser-shell.html's keydown handler) to TI-OS token bytes. Reference `TI-84_Plus_CE/keyboard-matrix.md` for the scan code mapping and `TI-84_Plus_CE/references/ti84pceg.inc` for token equates.
