# Task: Phase 25P — ParseInp retry with OPS seeded

## Context
You are working on a TI-84 Plus CE ROM transpilation project. ParseInp (expression evaluator) at `0x099914` failed in session 73 because we seeded the wrong pointer globals. Session 74's disassembly revealed that **OPS (0xD02593)** is the critical input pointer — the helper at `0x09BEED` does `ld hl,(0xD02593)` and uses it as a read cursor. The begPC/curPC slots we previously seeded are OUTPUTS written by the helpers, not inputs.

## Goal
Write a probe `TI-84_Plus_CE/probe-phase25p-parseinp-ops.mjs` that:
1. Cold-boots the OS (same pattern as all other probes)
2. Seeds OPS (0xD02593) to point at a tokenized expression buffer
3. Calls ParseInp at 0x099914
4. Checks whether OP1 (0xD005F8) contains the evaluated result

## Tokenized expression
Use `2+3` encoded as TI tokens: `[0x32, 0x70, 0x33, 0x3F]` where:
- 0x32 = token for digit "2"
- 0x70 = token for "+"
- 0x33 = token for digit "3"  
- 0x3F = token for newline/end (tEnter)

Place these bytes at a safe RAM address like `0xD1A881` (userMem area). Then write a 3-byte LE pointer to `0xD1A881` at OPS address `0xD02593`.

## Additional setup
- Also seed `0xD00687` (asm_ram) with 0x00 (the helper at 0x099B18 uses it as scratch)
- Set IY = 0xD00080 (standard)
- The loop at 0x084711 reads OP1+1 (0xD005F9) as a byte compare — pre-clear OP1 to 0x00 bytes so it doesn't match a stale type byte
- Use max_steps = 500000 (ParseInp may run longer than simple FP calls)
- Use maxLoopIterations = 8192

## Template
Use `TI-84_Plus_CE/probe-phase25o-fpmult.mjs` as your template. Key differences:
- Entry point is 0x099914 (not an FP routine)
- You need to write the tokenized buffer to RAM and set OPS pointer
- Import `readReal` from `./fp-real.mjs` to read the result from OP1
- Expected result: 5.0 (2+3)
- Track visited PCs in an array (last 50) so we can see where it gets stuck if it loops

## Deliverables
1. `TI-84_Plus_CE/probe-phase25p-parseinp-ops.mjs` — the probe
2. `TI-84_Plus_CE/phase25p-parseinp-ops-report.md` — written by the probe

## Constraints
- Do NOT modify any existing files
- Do NOT touch cpu-runtime.js, peripherals.js, ez80-decoder.js, or the transpiler
- The probe must be runnable with `node TI-84_Plus_CE/probe-phase25p-parseinp-ops.mjs`
- Use `import { createExecutor } from './cpu-runtime.js'` and `import { createPeripheralBus } from './peripherals.js'`

- `mem.fill(val, start, END)` — third arg is END not length!
- Timer IRQ must be disabled: `createPeripheralBus({ timerInterrupt: false })`

## Success criterion
If ParseInp evaluates "2+3" and OP1 = 5.0, that's a PASS. If it loops forever or crashes, capture the last 50 PCs visited and report as informative FAIL. Either outcome is valuable.
