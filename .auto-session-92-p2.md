# Task: Phase 25AG — Store+Recall Round-Trip (RESTRUCTURED pipeline, no pre-created Ans)

## Goal
Build a probe that validates the full store+recall pipeline: MEM_INIT → ParseInp("2+3")=5.0 → StoAns(5.0) → RclVarSym("Ans")=5.0, WITHOUT pre-creating the Ans variable before ParseInp.

## Background
Session 91 proved: creating Ans BEFORE ParseInp causes token-stream-redirect — OP1=zeros maps to Ans, ParseInp finds it via ChkFindSym and redirects the token stream to Ans data area (all zeros → 0.0). The pipeline design was wrong.

The CORRECT pipeline, per sessions 84-88:
1. MEM_INIT (0x09DEE0) — 18 steps, initializes heap pointers
2. ParseInp (0x099914) with OP1=zeros and tokens "2+3" → returns OP1=5.0 in ~918 steps (errNo=0x8D is a residual, harmless)
3. StoAns — ParseInp's own StoAns sequence (at step ~870) tries to store 5.0 into Ans. When Ans doesn't exist, StoAns's ChkFindSym fails (49-step failure path). When Ans exists, it stores successfully (session 88 confirmed: Ans data = [00 80 50 00 00 00 00 00 00] = 5.0).
4. After ParseInp returns with OP1=5.0, call RclVarSym(0x09AC77) with OP1 set to Ans variable name [0x72, 0x00, ...] to recall 5.0.

The PROBLEM: ParseInp's internal StoAns can't create Ans because ChkFindSym fails (the variable doesn't exist yet and ParseInp doesn't create it). So after ParseInp, we need to either:
- (a) Call CreateReal("Ans") to create it, then store 5.0 into it manually, then RclVarSym
- (b) Or: after ParseInp returns OP1=5.0, manually call StoAns as a separate step (via the StoAns address at 0x099A9B or through the StoAns sequence)

Session 89 confirmed: RclVarSym works perfectly — MEM_INIT → CreateReal("A") → write 42.0 → RclVarSym("A") returns 42.0 in 43 steps.

## What to build
Create `TI-84_Plus_CE/probe-phase25ag-store-recall-roundtrip.mjs` that:

### Stage 1: MEM_INIT
- Cold boot + MEM_INIT at 0x09DEE0

### Stage 2: ParseInp("2+3")
- Set OP1 = all zeros (NOT a variable name)
- Write tokens [0x32, 0x70, 0x33, 0x3F] ("2+3\n") at 0xD00800
- Set begPC/curPC (0xD02317/0xD0231A) = 0xD00800, endPC (0xD0231D) = 0xD00804
- Leave OPS at MEM_INIT default (0xD3FFFF)
- Set errSP to point at a 6-byte error catch frame: [FAKE_ERR_CATCH(3 bytes), prevErrSP=0(3 bytes)] pushed on stack
- Push FAKE_RET as ParseInp's return address
- Run ParseInp at 0x099914 with 1.5M step budget
- Verify OP1=5.0 (should be ~918 steps)
- Snapshot the DE register value (points to where StoAns tried to store)

### Stage 3: CreateReal("Ans") + store 5.0
- Set OP1 = [0x00, 0x72, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00] (Ans variable name, type=0x00 Real, name=0x72="Ans")
- Call CreateReal at 0x08238A
- Read DE after return — it points to the 9-byte data area
- Write the BCD value for 5.0: [0x00, 0x80, 0x50, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00] at DE

### Stage 4: RclVarSym("Ans")
- Set OP1 = [0x00, 0x72, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00] (Ans)
- Call RclVarSym at 0x09AC77
- Read OP1 — should be [0x00, 0x80, 0x50, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00] = 5.0

Report:
- Each stage's step count, errNo, OP1 state, pointer snapshot
- Whether the full round-trip (parse 2+3 → store 5.0 → recall 5.0) succeeds
- Use `readReal` from `fp-real.mjs` to decode BCD values
