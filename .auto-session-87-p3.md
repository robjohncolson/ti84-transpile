# Task: Disassemble 0x03E1B4 error-writing routine

## Context

TI-84 Plus CE ROM transpilation project. Session 86 identified 0x03E1B4 as the routine that JError (0x061DB2) dispatches to for persisting errNo before longjmp. The call sequence is: JError stores errNo at 0xD008DF → calls 0x03E1B4 → 0x03E1B4 does interrupt-safe cleanup → clears IY-offset flag bits → performs `ld sp, (errSP)` longjmp.

Session 78 partially described 0x03E1B4 as "interrupt-safe cleanup wrapper (saves A to scratch 0xD00542, checks `ld a,i / jp pe`, DI, calls 0x03E187, restores EI state)".

## Your task

Create a probe `TI-84_Plus_CE/probe-phase25ab-error-routine-disasm.mjs` that performs STATIC DISASSEMBLY of the ROM bytes starting at 0x03E1B4, continuing through to the next RET or unconditional jump (likely ~0x03E1D4 or so based on session 86 estimates). Also disassemble the subroutine at 0x03E187 that it calls.

The probe should:

1. Read the ROM file directly (fs.readFileSync)
2. Use the instruction decoder from `TI-84_Plus_CE/ez80-decoder.js` if available, OR manually decode the bytes by reading raw hex and mapping to eZ80 instructions
3. Print a formatted disassembly listing with:
   - Address
   - Raw hex bytes
   - Decoded instruction mnemonic
   - Comments for known RAM addresses (e.g., 0xD00542 = scratch, 0xD008DF = errNo, 0xD008E0 = errSP)
4. Cover range 0x03E180..0x03E200 (to capture both 0x03E187 and 0x03E1B4 with context)
5. Identify what state 0x03E1B4 modifies beyond errNo (IY flags? other RAM? interrupt state?)

Write a markdown report to `TI-84_Plus_CE/phase25ab-error-routine-disasm-report.md` with the full disassembly listing and analysis.

## ROM file location

`TI-84_Plus_CE/ROM.rom` — raw 4MB binary, byte at address N is at file offset N.

## eZ80 decoder

Import from `TI-84_Plus_CE/ez80-decoder.js`:
```js
import { decodeInstruction } from './ez80-decoder.js';
```
Call: `decodeInstruction(romBytes, offset)` — returns an object with `.mnemonic`, `.length`, etc. If that doesn't work, just read raw bytes and output a hex dump.

## Known RAM addresses for annotation
- 0xD005F8 = OP1
- 0xD00542 = scratch temporary
- 0xD008DF = errNo
- 0xD008E0 = errSP
- 0xD00080 = IY base (so IY+N = 0xD00080+N)
