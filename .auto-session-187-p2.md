# Auto-Session 187 P2 — Static Disassembly of Inline Code Between Home Handler CALLs

## Context

The TI-84 Plus CE home handler at 0x058460 has 6 known CALL sites that process a keypress. Session 186 tested all 6 individually and found **NONE writes to TOKEN_STAGING (0xD0230E)**. The revised hypothesis is that TOKEN_STAGING is populated by **inline code between the CALL sites**, not by any called function. The inline code likely stores ConvKeyToTok's output (register E) to 0xD0230E.

Session 185 disassembled the home handler from 0x058460-0x0584B0 and found the 6 CALL sites:
1. CALL 0x06EDFE
2. CALL 0x083623
3. CALL 0x083764
4. CALL 0x0800EC
5. CALL 0x0A223A
6. CALL 0x058C65

The key question: where between (or around) these CALLs does the ROM write the token value to TOKEN_STAGING (0xD0230E)?

## Task

Build a probe `TI-84_Plus_CE/probe-phase187-inline-token-staging.mjs` that:

1. **Static disassembly**: Read ROM.rom bytes from 0x058460 to 0x058700 (the full home handler range from session 182's disassembly). Use `ez80-decoder.js` to decode every instruction. For each instruction, check if it:
   - References address 0xD0230E (TOKEN_STAGING) in any operand (immediate, displacement, or memory address)
   - References address 0xD02300-0xD02310 range (nearby TOKEN area)
   - Is an LDIR/LDI that might copy to/from this area
   - Is a LD (nn),A or LD (nn),r where nn could be TOKEN_STAGING
   - Contains the byte sequence `0E 23 D0` (little-endian 0xD0230E) anywhere in the instruction encoding

2. **Extended static scan**: Also scan 0x058700-0x058D00 (the broader home handler exit code area) for the same patterns

3. **ConvKeyToTok output trace**: After ConvKeyToTok (0x05C52C) returns, register E contains the token. Scan the ROM for instructions that store E to memory after a CALL to 0x05C52C — look for `LD (HL),E`, `LD (nn),E` (if that exists), or `LD A,E; LD (nn),A` patterns within 20 bytes after any CALL 0x05C52C site.

4. **Report**: List every instruction that references TOKEN_STAGING or could plausibly write to it, with PC address, hex bytes, and decoded mnemonic.
