# Auto-Session 169 P3: ConstLoader at 0x07FA74 — Context-Dependency Check

## Context

This is a TI-84 Plus CE ROM transpilation project. We're investigating why `gcd(12,8)` produces E_Domain instead of 4.0.

The type validator at 0x068B78 calls ConstLoader at 0x07FA74 to load OP2=1.0. Session 146 disassembled 0x07FA74 and found it loads a fixed constant (type=0x00, exp=0x80, mantissa=0x10 = 1.0 BCD). But the question remains: is this truly context-independent, or could the value loaded depend on register state at entry?

Session 146 showed the routine uses `LD A,0x10` (hardcoded mantissa byte) and `LD (HL),0x00; INC HL; LD (HL),0x80; INC HL; LD (HL),A` to write the constant. This looks hardcoded. But we should verify by:
1. Checking if HL at entry always points to OP2
2. Checking if different entry points (0x07FA95, 0x07FA9A, 0x07FA9F) load different values
3. Testing with different register states to confirm no context sensitivity
