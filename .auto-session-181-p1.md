# Auto-Session 181 P1 — Disassemble Foreground Keyboard Scan at 0x0159C0

## Goal
Write a Node.js probe (`TI-84_Plus_CE/probe-phase181-fgkbd-scan.mjs`) that disassembles the foreground keyboard scan routine at 0x0159C0 and traces how it reads the keyboard matrix, translates scan codes, and populates kbdKey (0xD0058C).

## Context
- Session 178 confirmed `_GetCSC` (0x03FA09) is the **ISR dispatcher**, NOT the keyboard scanner.
- The **foreground** keyboard scan is at 0x0159C0 — this is what the OS event loop calls to poll keys.
- The scan-to-keycode translation table is at 0x09F79B (confirmed session 129).
- kbdScanCode = 0xD00587, kbdKey = 0xD0058C, kbdGetKy = 0xD0058D.
- IY = 0xD00080 (so IY+0x507 = 0xD00587, IY+0x50C = 0xD0058C).
- The keyboard matrix MMIO is at ports 0xA00-0xA0F (see `peripherals.js`).

## Deliverables
Write `TI-84_Plus_CE/probe-phase181-fgkbd-scan.mjs` that:

1. **Disassembles** the function at 0x0159C0 using the eZ80 decoder. Read bytes from ROM, decode each instruction, print address + mnemonic + bytes. Continue for ~200 instructions or until you hit a RET at the same call depth.

2. **Traces execution** with a seeded keyboard matrix: set `keyMatrix[1] = 0xFE` (ENTER pressed, SDK group 6), then call 0x0159C0 with:
   - CPU: `madl=1, mbase=0xD0, iy=0xD00080, ix=0xD1A860, f=0x40`
   - SP at 0xD1A87E - 12
   - Push FAKE_RET=0x7FFFFE on stack
   - maxSteps: 50000, maxLoopIterations: 8192
   
3. **Logs** the values of kbdScanCode (0xD00587) and kbdKey (0xD0058C) before and after the call.

4. **Generates a report** at `TI-84_Plus_CE/phase181-fgkbd-scan-report.md` with the disassembly, trace results, and any findings about how 0x0159C0 interacts with the scan-to-keycode table at 0x09F79B.

## Technical Pattern (copy this exactly)
```javascript
import fs from 'node:fs';
import path from 'node:path';
import { fileURLToPath, pathToFileURL } from 'node:url';
import { createExecutor } from './cpu-runtime.js';
import { createPeripheralBus } from './peripherals.js';
import { decodeInstruction as decodeEz80 } from './ez80-decoder.js';

const __dirname = path.dirname(fileURLToPath(import.meta.url));
const romBytes = fs.readFileSync(path.join(__dirname, 'ROM.rom'));
const romModule = await import(pathToFileURL(path.join(__dirname, 'ROM.transpiled.js')).href);
const BLOCKS = romModule.PRELIFTED_BLOCKS;

const MEM_SIZE = 0x1000000;
const mem = new Uint8Array(MEM_SIZE);
mem.set(romBytes);
const perph = createPeripheralBus({ pllDelay: 2, timerInterrupt: false });
const ex = createExecutor(BLOCKS, mem, { peripherals: perph });
const c = ex.cpu;
```

For keyboard matrix seeding, use `perph.keyMatrix[1] = 0xFE` (ENTER = bit 0 clear in SDK group 6 = keyMatrix[1]).

The `decodeEz80(romBytes, offset, isADL)` function returns `{ mnemonic, operands, size }`. Call it with `isADL = true` for ADL mode.
