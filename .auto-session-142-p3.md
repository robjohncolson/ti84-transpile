# Auto-Session 142 P3 — Graph 17-Pixel Investigation (IY+43 bit 2)

## Context
The TI-84 ROM transpilation project lifts eZ80 machine code into JavaScript. GraphPars (graph rendering) evaluates Y=X for 133 X values but only writes 17 pixels to plotSScreen. Session 141 confirmed that the .SIS prefix is decoded correctly and the 8-bit CP at IPoint is intentional OS behavior.

Session 141 P3 found: At the IPoint bounds-check (0x07B793), `BIT 2,(IY+43)` determines the path:
- Z path (bit clear): Uses 8-bit `CP L` comparison — limits X coordinates to 0-255 (8-bit range), rejecting most valid pixel columns
- NZ path (bit set): Uses full 24-bit compare via `LD HL,320; CALL 0x04C979` — supports full 320-pixel width

IY = 0xD00080, so IY+43 = IY+0x2B = 0xD000AB.

   - IY+43 = 0xD000AB (IY=0xD00080, offset 0x2B)

3. **Runs the full 133-column rendering loop** with the bit set
4. **Compares pixel counts**: Report how many non-zero bytes in plotSScreen with vs without the bit set
