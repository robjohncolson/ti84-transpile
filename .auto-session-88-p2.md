# Task: ParseInp with BOTH Ans variable AND real variable "A" + stored 42.0

## Context
TI-84 Plus CE ROM transpilation project. We lift eZ80 machine code into JavaScript. Runtime at `TI-84_Plus_CE/cpu-runtime.js`, peripherals at `TI-84_Plus_CE/peripherals.js`, FP decode at `TI-84_Plus_CE/fp-real.mjs`.

## Background
- Session 87 showed: (a) Creating Ans alone doesn't clear errNo=0x8D (902 steps), (b) Variable recall of "A" returns 0.0 not 42.0 (710 steps, errNo=0x00)
- Session 86 showed: errNo=0x8D is set at step 27 by ChkFindSym failing on OP1=zeros, before parsing begins
- The initial ChkFindSym at ParseInp entry checks OP1's contents. If OP1 is zeros → type byte 0x00 → lookup fails → errNo=0x8D residual

## What to build
Create probe: `TI-84_Plus_CE/probe-phase25ac-both-vars.mjs`

The probe should run THREE scenarios against the same boot state:

**Scenario A**: MEM_INIT → CreateReal("A") with 42.0 → CreateReal(Ans) → set OP1=[00 41 ...] (variable A name) → ParseInp("2+3")
- Tests: does having BOTH variables change behavior vs session 87's single-variable tests?

**Scenario B**: MEM_INIT → CreateReal("A") with 42.0 → CreateReal(Ans) → clear OP1 to zeros → ParseInp("2+3")
- Tests: with both variables created, does errNo clear? (OP1=zeros triggers ChkFindSym fail at step 27; Ans exists for StoAns at step ~878)

**Scenario C** (control): MEM_INIT → ParseInp("2+3") with OP1=zeros, no variables created
- Expected: 918 steps, errNo=0x8D, OP1=5.0 (matches all prior sessions)

Each scenario needs a FRESH runtime (new Uint8Array, new executor). Do NOT reuse memory between scenarios.

For each scenario, report:
- Step count, errNo, OP1 bytes + decoded value
- Whether ParseInp returned to FAKE_RET or hit ERR_CATCH
- Pointer snapshots (OPBase, OPS, FPS, newDataPtr) before/after ParseInp

**CreateReal setup for Ans variable:**
- Ans in TI-OS has OP1 type byte = 0x00, name byte = 0x72 (tAns token)
- So OP1 for CreateReal(Ans) = `[0x00, 0x72, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]`
- After CreateReal returns, DE points to the new 9-byte data slot. You can leave it zeros (Ans=0.0) or write a value.

**CreateReal setup for variable "A":**
- OP1 = `[0x00, 0x41, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]`
- After CreateReal, write 42.0 BCD `[0x00, 0x81, 0x42, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00]` at DE

**Key addresses (same as all Phase 25 probes):**
- MEM_INIT_ENTRY = 0x09DEE0
- CREATEREAL_ENTRY = 0x08238A
- PARSEINP_ENTRY = 0x099914
- OP1_ADDR = 0xD005F8 (9 bytes)
- ERR_NO_ADDR = 0xD008DF
- ERR_SP_ADDR = 0xD008E0
- TOKEN_BUFFER_ADDR = 0xD00800
- BEGPC/CURPC/ENDPC = 0xD02317/0xD0231A/0xD0231D
- TEMPMEM/FPSBASE/FPS = 0xD02587/0xD0258A/0xD0258D
- OPBASE/OPS = 0xD02590/0xD02593
- PTEMP/PROGPTR = 0xD0259A/0xD0259D
- NEWDATA_PTR = 0xD025A0
- FAKE_RET = 0x7FFFFE, ERR_CATCH_ADDR = 0x7FFFFA

**Boot sequence** (copy from `probe-phase25ab-parseinp-recall.mjs`):
- Cold boot: runFrom(0x000000, 'z80', maxSteps 20000, maxLoopIterations 32)
- Reset: halted=false, iff1/iff2=0, sp=0xD1A87E-3, fill 0xFF at sp..sp+3
- Kernel init: runFrom(0x08C331, 'adl', maxSteps 100000, maxLoopIterations 10000)
- Set: mbase=0xD0, iy=0xD00080, hl=0, halted=false, iff1/iff2=0, sp=0xD1A87E-3, fill 0xFF
- Post init: runFrom(0x0802B2, 'adl', maxSteps 100, maxLoopIterations 32)

**Before each OS call**: halted=false, iff1/iff2=0, madl=1, mbase=0xD0, iy=0xD00080, f=0x40, ix=0xD1A860, sp=STACK_RESET_TOP-12, fill 0xFF at sp for 12 bytes. Then push return address (3 bytes) and set up minimal error frame.

**Minimal error frame**: push FAKE_RET on stack, write 6-byte frame below: [00 00 00 | ERR_CATCH_ADDR], set errSP to frame base, clear errNo.

**Timer IRQ**: MUST disable: `createPeripheralBus({ timerInterrupt: false })`
**Token bytes for "2+3"**: `[0x32, 0x70, 0x33, 0x3F]`
**mem.fill** uses (val, start, END) — third arg is END not length
