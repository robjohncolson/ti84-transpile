# Auto-Session 187 P3 — STAT Key Dispatch Lookup in Home Handler

## Context

The TI-84 Plus CE home handler processes keypresses via a dispatch table. We need to find how the STAT key (scan code 0xA1 per keyboard-matrix.md) routes from the home handler into the STAT menu system. Sessions 139-146 validated stat primitives (1-Var Stats, etc.) but the menu UI path has never been traced.

The home handler at 0x058460 has been partially disassembled (session 182). There's a JP (HL) indirect dispatch table at 0x0585D3. The STAT key should have an entry in this dispatch table or in a key-to-handler lookup table nearby.

## Task

Build a probe `TI-84_Plus_CE/probe-phase187-stat-key-dispatch.mjs` that:

1. **Find the key dispatch table**: The home handler uses JP (HL) at 0x0585D3 for indirect dispatch. Scan the ROM around 0x058500-0x058D00 for tables of 3-byte addresses (ADL pointers). A dispatch table would be a sequence of 3-byte little-endian addresses, each pointing to handler code. Look for patterns where consecutive 3-byte values all point into the 0x05xxxx-0x0Axxxx range.

2. **Scan code mapping**: The TI-OS converts physical scan codes into key codes before dispatch. Read `TI-84_Plus_CE/keyboard-matrix.md` for the STAT key's scan code. Search for the STAT key code value in the home handler ROM bytes (0x058000-0x059000). Look for CP instructions comparing against the STAT key code, or table entries indexed by key code.

3. **Static disassembly of dispatch mechanism**: Starting from 0x0585D3 (JP (HL)), trace backward to find how HL is loaded. This likely involves:
   - A table base address loaded into HL
   - A key code used as index (shifted/multiplied)
   - HL = table_base + key_code * entry_size
   
   Disassemble 0x058580-0x0585E0 to find this pattern.

4. **Look for STAT handler address**: Once the dispatch table structure is understood, compute where the STAT key entry would be and read the handler address from the ROM.

5. **Verify handler**: If a STAT handler address is found, disassemble the first 10-20 instructions at that address to confirm it's STAT-related (look for menu setup calls, string references like "STAT", or calls to DispMenu).
