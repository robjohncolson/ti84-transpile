# Session 141 P2: OneVar minX/maxX investigation

## Context
We're transpiling the TI-84 Plus CE ROM to JavaScript. Session 140 got 6/8 OneVar stat results correct for L1=[10,20,30,40,50]: n=5, x̄=30, Σx=150, Σx²=5500, Sx=15.8114, σx=14.1421 — all correct. But **minX and maxX were NOT computed** (stat var slots at 0xD011C7 and 0xD011D0 remain zero).

The stat var address map (11 bytes per slot):
- n = 0xD01191
- xMean = 0xD0119A
- sumX = 0xD011A3
- sumX2 = 0xD011AC
- Sx = 0xD011B5
- sigmaX = 0xD011BE
- minX = 0xD011C7
- maxX = 0xD011D0

## Task

### Step 1: Read the existing OneVar probe
Read `TI-84_Plus_CE/probe-phase140-onevar-prealloc.mjs` in full. Understand the setup: MEM_INIT, manual VAT entry for L1, list data layout, statFlags2 manipulation, OneVar entry point, step budget.

### Step 2: Build an investigation probe
Create `TI-84_Plus_CE/probe-phase141-onevar-minmax.mjs` that:

1. Uses the exact same setup as `probe-phase140-onevar-prealloc.mjs` (copy the MEM_INIT, VAT, list data, statFlags2, FPS seeding, A=1 for single-list)
2. Runs OneVar with a **100,000 step budget** (increased from the 50K in session 140) to give it more room
3. After execution, dumps:
   - errNo at 0xD008DF
   - All 8 stat var slots (n through maxX) with BCD-to-decimal conversion
   - The minX slot (11 bytes at 0xD011C7) and maxX slot (11 bytes at 0xD011D0) raw hex
   - Whether any new memory regions were written near the stat vars (scan 0xD011D0-0xD01200 for non-zero)
4. Also adds a write-watchpoint on 0xD011C7 and 0xD011D0 during execution: intercept `write8` calls to these addresses and log the PC + step number when they're written. This tells us if OneVar writes minX/maxX at all, and if so, at which step and from which PC.
   
   To add a write-watchpoint, wrap the memory's write8 function:
   ```js
   const origWrite8 = mem.write8 || ((addr, val) => { mem[addr] = val; });
   const watchLog = [];
   // Monkey-patch the executor's write8 or the mem array
   // The executor calls mem[addr] = val directly, so we need to use a Proxy or 
   // check after each step. Simpler approach: after each step, check if the 
   // watched bytes changed from their previous values.
   ```
   
   Actually, the simplest approach: snapshot the minX/maxX slots before each step, run one step, compare after. Log any changes with the step number and PC.

5. If minX/maxX are STILL zero after 100K steps, also trace the last 1000 PCs to see where OneVar got stuck or exited

### Step 3: Analyze and report
The key question: does OneVar compute minX/maxX at all, or does it rely on a separate routine (like SortA at JT slot 0x020674)?

On real TI-84, `1-Var Stats L1` computes all 8 values including min/max. The OS may:
- (a) Sort the list first, then read first/last elements
- (b) Scan the list for min/max in a separate pass
- (c) Call a subroutine (SortA or similar) internally

If the probe shows minX/maxX are never written, the next step is to check if OneVar calls SortA or a min/max scanning routine, and whether those routines have transpiled blocks.

Print results clearly.
