# Task: Additional FP primitives — probe FPMult, FPDiv, and TenX

## Context

We are reverse-engineering the TI-84 Plus CE OS ROM. Sessions 71-73 proved the FP engine works for FPAdd, FPSub, FPSquare, SqRoot, Sin, Cos, Tan, LnX, CpOP1OP2. Now we want to extend coverage to three more FP primitives.

Jump table entries (from `phase25h-a-jump-table.json`):
- **FPMult**: slot 0x0201D8, impl `0x07C8B7` — OP1 = OP1 * OP2
- **FPDiv**: slot 0x0201F4, impl `0x07CAB9` — OP1 = OP1 / OP2
- **TenX**: slot 0x02021C, impl `0x07E219` — OP1 = 10^OP1

## Your Task

Create THREE probe files, one for each primitive, following the exact pattern of `TI-84_Plus_CE/probe-phase25l-fpsub.mjs`:

### 1. `TI-84_Plus_CE/probe-phase25o-fpmult.mjs`
- Test: OP1=6.0 * OP2=7.0 → expect 42.0
- Entry: 0x07C8B7
- Both OP1 (0xD005F8) and OP2 (0xD00603) seeded via `writeReal()`

### 2. `TI-84_Plus_CE/probe-phase25o-fpdiv.mjs`
- Test: OP1=84.0 / OP2=2.0 → expect 42.0
- Entry: 0x07CAB9
- Both OP1 and OP2 seeded

### 3. `TI-84_Plus_CE/probe-phase25o-tenx.mjs`
- Test: OP1=3.0 → expect 10^3 = 1000.0
- Entry: 0x07E219
- Only OP1 seeded (unary operation)

Cold boot sequence:
```js
function coldBoot(executor, cpu, mem) {
  const result = executor.runFrom(BOOT_ENTRY, BOOT_MODE, {
    maxSteps: BOOT_MAX_STEPS, maxLoopIterations: BOOT_MAX_LOOP_ITER,
  });
  cpu.halted = false; cpu.iff1 = 0; cpu.iff2 = 0;
  cpu.sp = STACK_RESET_TOP - 3;
  mem.fill(0xFF, cpu.sp, cpu.sp + 3);
  executor.runFrom(KERNEL_INIT_ENTRY, 'adl', { maxSteps: 100000, maxLoopIterations: 10000 });
  cpu.mbase = 0xD0; cpu._iy = 0xD00080; cpu._hl = 0;
  cpu.halted = false; cpu.iff1 = 0; cpu.iff2 = 0;
  cpu.sp = STACK_RESET_TOP - 3;
  mem.fill(0xFF, cpu.sp, cpu.sp + 3);
  executor.runFrom(POST_INIT_ENTRY, 'adl', { maxSteps: 100, maxLoopIterations: 32 });
  return result;
}

function postInitState(cpu, mem) {
  cpu.halted = false; cpu.iff1 = 0; cpu.iff2 = 0;
  cpu.madl = 1; cpu._iy = 0xD00080; cpu.f = 0x40;
  cpu._ix = 0xD1A860;
  cpu.sp = STACK_RESET_TOP - 12;
  mem.fill(0xFF, cpu.sp, cpu.sp + 12);
}
```

CRITICAL: `mem.fill(val, start, END)` NOT `mem.fill(val, start, length)`. Always `start + N` for the third argument.
