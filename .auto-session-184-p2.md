# Auto-Session 184 P2: Trace Key Classifier 0x07F7BD with Digit Key

## Context

This is a TI-84 Plus CE ROM transpilation project. The home handler at 0x0582B8 processes keypresses. At 0x0584A7 it calls `0x07F7BD` — the key classifier — which returns a type value in register A. The handler then dispatches:
- CP 0x06 → JP 0x061D42
- CP 0x05 → special handling
- Otherwise → falls through to character insertion (BufInsert path)

We need to know what value 0x07F7BD returns for a digit key (e.g., key '2' = scan code 0x9A from keyboard-matrix.md) so we can confirm digit keys fall through to BufInsert.

## Task

Build a Node.js probe at `TI-84_Plus_CE/probe-phase184-key-classifier.mjs` that:

1. **Static disassembly of 0x07F7BD**: Decode instructions starting at 0x07F7BD for at least 60 instructions or until you hit a RET. Use the ez80-decoder.js to decode. Show the full disassembly.

2. **Dynamic trace with A=0x9A (key '2')**: Execute from 0x07F7BD with:
   - A = 0x9A (the scan code for digit 2, per keyboard-matrix.md: group 3 bit 2 → scanCode = (3<<4)|2 = 0x32... actually check keyboard-matrix.md for the correct value)
   - Actually, check what value A has when the home handler calls 0x07F7BD. The home handler gets a key code somehow. Look at `TI-84_Plus_CE/keyboard-matrix.md` for the key code format.
   - SP = 0xD1A860, IX = 0xD1A860
   - Trace up to 200 steps or until RET
   - Log: PC, instruction tag, A register at each step, and the final A value on return

3. **Test multiple key codes**: Run the classifier with at least 3 different key codes:
   - A digit key (e.g., '2')
   - The ENTER key (0x09 per keyboard-matrix.md)
   - The STAT key
   Report what A value is returned for each.
