# Task: Disassemble 0x04C990 — the NEW cxCurApp zeroing site

## Context
In the TI-84 Plus CE ROM transpilation project, we're tracing why CoorMon (the OS main event loop at 0x08C331) fails to dispatch keystrokes to the home-screen handler. Session 92 found that after manually seeding cxMain=0x058241 and cxCurApp=0x40, the RAM CLEAR at 0x001881 was prevented (major progress), but cxCurApp gets zeroed at step 18296 by code at **PC=0x04C990**. This is a different zeroing mechanism than the 0x001881 cold-restart path.

## What to do

1. **Read the ROM bytes** from `TI-84_Plus_CE/ROM.rom` at offset 0x04C950 through 0x04CA50 (256 bytes centered on 0x04C990). Use the eZ80 decoder at `TI-84_Plus_CE/ez80-decoder.js` to disassemble them.

2. **Create a probe script** `TI-84_Plus_CE/probe-phase25ah-disasm-04C990.mjs` that:
   - Reads ROM.rom
   - Decodes instructions from 0x04C950 to 0x04CA50
   - Prints each instruction with address, hex bytes, and mnemonic
   - Identifies any writes to 0xD007E0 (cxCurApp) or 0xD007CA (cxMain) — these are `LD (0xD007E0),A` or `LD (0xD007CA),HL` patterns
   - Cross-references CALL/JP targets against known addresses:
     - 0x08C331 = CoorMon
     - 0x08C72F = CoorMon dispatch sub
     - 0x08C79F = NewContext
     - 0x08C7AD = NewContext0
     - 0x058241 = home-screen handler
     - 0x099914 = ParseInp
     - 0x08238A = CreateReal
     - 0x001881 = RAM CLEAR
   - Also check the 5 known cxCurApp read sites: 0x08C408, 0x08C4C7, 0x08C59C, 0x08C5C8, 0x08C5E7

3. **Create a report** `TI-84_Plus_CE/phase25ah-disasm-04C990-report.md` summarizing:
   - What routine 0x04C990 is part of (function boundaries — look for RET instructions)
   - Why it zeros cxCurApp (deliberate context-switch? error path? cleanup?)
   - What conditions lead to this code path
   - Any conditional branches that could be avoided with proper OS state

## Reference files
- `TI-84_Plus_CE/ez80-decoder.js` — instruction decoder (import and use `decodeInstruction(rom, offset)`)
- `TI-84_Plus_CE/cpu-runtime.js` — runtime reference
- `TI-84_Plus_CE/references/ti84pceg.inc` — OS equates for cross-referencing

## Pattern for the probe
```javascript
import { readFileSync } from 'fs';
import { decodeInstruction } from './ez80-decoder.js';

const rom = readFileSync('TI-84_Plus_CE/ROM.rom');
const startAddr = 0x04C950;
const endAddr = 0x04CA50;

let pc = startAddr;
while (pc < endAddr) {
    const result = decodeInstruction(rom, pc);
    // result has .mnemonic, .length, .bytes
    const hexBytes = Array.from(rom.slice(pc, pc + result.length)).map(b => b.toString(16).padStart(2,'0')).join(' ');
    console.log(`0x${pc.toString(16).padStart(6,'0')}: ${hexBytes.padEnd(20)} ${result.mnemonic}`);
    pc += result.length;
}
```
